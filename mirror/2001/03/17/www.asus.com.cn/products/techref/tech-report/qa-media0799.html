July & August Multimedia Technical Column

Question:
When using the AGP-V3800TNT2 card, a Creative SB Live and a Pentium III CPU in the system, it crashes and hangs while running any D3D game or application, but if I use a Pentium II CPU instead, D3D game or application can be run without problems. All TNT2 chipset based VGA cards have the same problem. What is wrong?

Answer:
This problem happens on all TNT2 based VGA card system with PIII processors when DirectX6.1 is also installed in the system. If Pentium III, RivaTNT2 (sometimes Voodoo3 also), SB Live are used together, the DirectX 6.1 (optimized for Pentium III ISSE instructions) installation will overwrite VMCPD.VXD in c:\windows\system\vmm32 because it detects new ISSE instructions of Pentium III, but the overwritten version does not work properly.

You can extract the old version VMCPD.VXD from Win98 CD (4.10.1998) to c:\windows\system\vmm32, then reboot the system to solve this problem. The file can be found in Win98_45.cab (English version of Win98), you can use the cabviewer from MS Powertoys or Winzip7.0 to extract it.

Question:
When I use AGP-V2740 with the released driver, v27r320b.exe, if I run Microsoft NetMeeting or other capture software, it cannot detect the Video-in device, but there is no problem with ASUS Live2740 software.

Answer:
It is because old version of Video for Windows driver supports Video Decoder BT82p, but does not support BT827. You can upgrade to new version of display driver to solve this problem. The latest beta driver can be downloaded from:
ftp://ftp.asus.com.tw/pub/ASUS/beta/vga

Question:
After I install AGP-V3200 driver in Win98 Second Edition, the system cannot be properly shut down.

Answer:
The V3200 shutdown problem in Windows98 SE can be solved by modifying the settings in the device manager. It is due to the compatibility problem between the O/S and 3Dfx driver. Please alter the respective setting in the device manger by following the procedures listed below to fix the problem.
  1. Right click on "My Computer"
  2. Select "Properties" from the pop-up menu
  3. Select the "Device Manager" tab of System Properties
  4. Tick on "View Devices by type" and open "System Devices"
  5. Highlight "PCI bus" and click on the "Properties" button
  6. Select the "IRQ Steering" tab of PCI Bus Properties
  7. Un-check "Use IRQ Steering"
  8. Click the OK button of the dialog. The system will prompt you to restart your computer. Press the Yes button to restart it. System would halt again this time but not next time.

As soon as I launch SHOGO game in Win98 with AGP-V3400/V3800 series cards, it will return back to the Windows immediately, and I cannot play the game.

Question:
When I enable Stereoscopic Mode to play 3D games with AGP-V3400/V3800 series cards, the screen turns to black. What is wrong?

Answer:
To support stereoscopic mode with 3D VR glasses, the monitor needs to support Interlaced mode at 120Hz, that is, 60Hz for each individual eye. Please check if your monitor can support 120Hz of Interlaced mode, or you can upgrade the driver to rev. 2.08 beta 3 or later. The driver can be downloaded from:
http://www.asus.com.tw/Products/Addon/Vga/asuse/enthusiast_hottestdrv.htm

The new beta driver has been added with the following options for VR Display Refresh Rate in Advanced -> 3D/VR:

100Hz : 800 x 600 or above
112Hz : 640 x 480 or above
120Hz : 640 x 480 or above
140Hz : 640 x 480 or above
150Hz : 640 x 480 or above

Note: The upper limit of the resolution has to be adjusted according to the monitor specification. After you adjust the setting, please enter Display Properties -> Settings, click Advanced, and enter Color -> D3D/Game, and press Preview to test the setting to ensure it can work properly.

ASUS V3800 driver rev. 1.91 or later has been added with OSD (On Screen Display) feature, and you can press the hot key (Ctrl+Alt+O) to adjust the setting while playing the game.

Question:
When running AGP-V3400 or AGP-V3800 series cards on ALi chipset based M/B, the system hangs up when I play 3D games in Win9x. What is wrong?

Answer:
This is a compatibility issue between ALi chipset and NVIDIA driver. Please upgrade the ALi's AGP driver to rev. 1.60 to fix this problem. The driver can be downloaded from:
ftp://ftp.asus.com.tw/pub/ASUS/beta/mb/driver/

Note: Please use driver rev. 1.91 or later with ALi's AGP driver rev. 1.60.

Question:
When I upgrade the V3800 driver from rev. 1.00 to rev. 1.91, I can not remove the old driver from Control Panel --> Add/Remove Programs, and it reports "You do not use ASUS Installer to install ASUS display driver". What is wrong?

Answer:
It is due to some mistakes implemented in the file of removal program. Please load the new driver rev. 1.91 or later to overwrite the old driver first, and execute the remove process again from Control Panel --> Add/Remove Programs.
