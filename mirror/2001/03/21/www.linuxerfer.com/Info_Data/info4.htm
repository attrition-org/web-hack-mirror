  HTTP Server 응답 코드

HTTP 서버 응답의 첫번째 행은 클라이언트 요청의 성공 여부와 그에 대한 이유를 표시한다. 그 상태는 숫자 3개로 된 서버 응답 코드(상태 코드라고도 하는 )와 설명 메시지로 주어진다.

상태 코드는 일반적으로 웹 서버에 의해 생성되지만 서버가 미리 준비된 헤더를 무시하고 자체적으로 제공하는 것도 가능하다.

상태코드는 다음과 같이 분류된다.

코드 범위   응답 의미
100-199
200-299
300-399
400-499
500-599

  서버 에러

서버가 필요에 따라 자체 코드를 정의할 수도 있지만 HTTP는 특정 코드 몇 개만을 각 영역에 정의한다. 클라이언트가 인식 불가능한 코드를 받으면 그 코드 번호의 범위내에서 기본 의미를 이해할 수 있어야 한다. 대부분의 웹 브라우저들 은 100-, 200-, 300- 영역에 있는 코드들은 조용히 지나치지만 400-과 500- 영역 에 있는 일부 에러 코드들은 사용자에게 보고되는 것이 일반적이다 (예를 들면, "404 Not Found").


참고 정보

100-199 범위의 응답은 참고 정보로 클라이언트의 요청이 받아들여져 처리 중에 있 음을 표시한다.

  • 100 Continue
    요청의 시작 부분이 받아들여졌으며 클라이언트는 그 요청을 계속할 수 있다.

  • 101 Switching Protocols
    Upgrade 헤더 필드에 지정된 프로토콜로 전환하려는 클라이언트의 요청을 서버가 받아들인다.


클라이언트 요청 성공

200-299 범위의 응답은 클라이언트의 요청이 성공적이었음을 뜻한다.

  • 200 OK
    클라이언트의 요청이 성공적이었으며 서버 응답은 요청 데이터를 포함한다.

  • 201 Created
    이 상태 코드는 새로운 URI의 생성시에 이용된다. 이 결과 코드와 함께 새로운 데이터가 위치할 장소를 지정하기 위해 Location 헤더가 서버에 의해 주어진다.

  • 202 Accepted
    요청은 받아들여졌으나 즉시 동작이 수행되지는 않는다. 트랜잭션에 관한 더 자세한 정보가 서버 응답의 개체 본체를 통해 주어질 수도 있다. 받아들일 당시에는 정당한 요청처럼 보였을 수도 있지만 서버가 요청을 실제로 승인하리라는 보장은 없다.

  • 203 Non-Authoritative Information
    개체 헤더 내의 정보가 원래 서버가 아니라 로컬 서버나 제3의 서버로부터 온 것이다.

  • 204 No Content
    응답에 상태 코드와 헤더는 주어지지만 개체 본체는 주어지지 않는다. 브라우저는 응답을 받아도 문서를 갱신해서는 안된다. 사용자가 이미지에서 의미없거나 비어 있는 공간을 클릭할 때에 이미지맵 핸들러가 리턴하기에 유용한 코드이다.

  • 205 Reset Content
    브라우저는 추가 입력을 위해 이 트랜잭션에서 사용된 폼을 삭제해야 한다. 데이터 입력 CGI 애플리케이션에 적당하다.

  • 206 Partial Content
    서버는 요청된 크기의 부분 데이터를 리턴하는 중이다. Range 헤더 지정 요청에 응답하는데 이용된다. 서버는 Content-Range 헤더와 함께 응답에 포함된 범위를 지정해야 한다.


방향 재지정

300-399 범위의 응답 코드는 요청이 수행되지 않았으며 요청의 성공을 위해서 클라이언트가 취해야 할 동작이 있음을 나타낸다.

  • 300 Multiple Choices
    요청된 URI는 둘 이상의 자원을 참조한다. 예를 들어, URI는 여러 언어로 변환되었던 문서를 참조한다. 서버에 의해 리턴되는 이 개체 본체는 정확한 자원의 선택 방법에 대한 더 자세한 데이터의 목록을 포함할 것이다.

  • 301 Moved Permanently
    요청된 URI는 더 이상 서버에 의해 사용되지 않으며 요청에서 지정한 연산은 수행되지 않았다. 요청한 문서의 새로운 위치는 Location 헤더에서 지정한다. 이 문서에 대한 앞으로의 요청들은 새로운 URI을 사용해야 한다.

  • 302 Moved Temporarily
    요청한 URI가 임시로만 이동하였다. Location 헤더는 새로운 위치를 가리킨다. 이 상태 코드를 받은 직후, 클라이언트는 새로운 URI를 이용하여 요청을 해석하여야 한다. 그러나 앞으로의 모든 요청들에 대해서는 예전 URI가 이용되어야 한다.

  • 303 See Other
    요청된 URI는 다른 URI(Location 헤더에 지정한)에서 발견될 수 있으며 그 자원에 대한 GET에 의해 구할 수 있다.

  • 304 Not Modified
    URI가 특정 날짜 이후로 수정된 적이 없을 때 If-Modified-Since 헤더에 들어가는 응답 코드이다. 개체 본체는 전송되지 않으며 클라이언트는 자신의 로컬 복사본을 이용해야 한다.

  • 305 Use Proxy
    요청한 URI는 Location 헤더의 프록시를 통해 접근되어야 한다.


클라이어트 요청 불완전

400-499 범위의 응답은 참고 정보로 클라이언트의 요청이 받아들여져 처리 중에 있 음을 표시한다.

  • 400 Bad Request
    응답 코드는 서버가 클라이언트 요청에서 문법 에러를 탐지하였음을 표시한다.

  • 401 Unauthorized
    결과 코드가 WWW-Authenticate 헤더와 함께 주어져서 요청에 적절한 인증이 결여되어 있으며 이 URI가 다시 요청될 때에는 적절한 인증이 제공되어야함을 표시한다.

  • 402 Payment Required
    이 코드는 HTTP에서 아직 구현되지 않았다.

  • 403 Forbidden
    서버가 클라이언트에게 알리고 싶지 않은(혹은 알릴 방법이 없는) 이유로 요청이 거부되었다.

  • 404 Not Found
    지정한 URI에 문서가 존재하지 않는다.

  • 405 Method Not Allowed
    이 코드는 Allow 헤더와 함께 주어지며 클라이언트가 사용한 메소드는 이 URI에서는 지원되지 않음을 표시한다.

  • 406 Not Acceptable
    클라이언트가 지정한 URI는 존재하지만 클라이언트가 원하는 형식은 아니다. 이 코드와 함께 서버는 Content-Language, Content-Encoding, Content-Type 헤더를 제공한다.

  • 407 Proxy Authentication Required
    프록시 서버는 요청을 전달하기 전에 승인 절차를 거쳐야 한다. Proxy-Authenticate 헤더와 함께 사용된다.

  • 408 Request Time-out
    이 응답 코드는 클라이언트가 정해진 시간(보통은 서버 구성에서 지정)내에 완전한 요청을 생성하지 않았으므로 서버는 네트워크 연결을 종료한다는 의미이다.

  • 409 Conflict
    이 코드는 요청이 다른 요청이나 서버 구성과 충돌했음을 나타낸다. 충돌에 관한 정보는 응답의 데이터 부분으로 리턴된다.

  • 410 Gone
    이 코드는 요청된 URI가 더 이상 존재하지 않으며 그 서버로부터 영구히 제거되었음을 의미한다.

  • 411 Length Required
    Content-Length 헤더가 없는 요청은 받아들이지 않는다.

  • 412 Precondition Failed
    요청에서 하나 이상의 If... 헤더에 의해 지정된 조건이 false 값을 갖는다.

  • 413 Request Entity Too Large
    개체 본체가 너무 크기 때문에 서버는 요청을 처리하지 않는다.

  • 414 Request-URI Too Long
    요청 URI가 너무 크기 때문에 서버는 요청을 처리하지 않는다.

  • 415 Unsupported Media Type
    개체 본체가 지원되지 않는 형식이기에 서버는 요청을 처리하지 않는다.


서버 에러

500-599 범위의 응답 코드는 서버가 에러를 만났으며 클라이언트의 요청을 수행할 수 없을지도 모른다는 의미이다.

  • 500 Internal Server Error
    이 코드는 서버의 일부분이 멈추었거나 구성 에러를 만났다는 표시이다.

  • 501 Not Implemented
    이 코드는 서버에 의해 수행될 수 없는 동작을 클라이언트가 요청했음을 나타낸다.

  • 502 Bad Gateway
    이 코드는 서버(또는, 프록시)가 다른 서버(또는, 프록시)로부터 유효하지 않은 응답을 받았다는 의미이다.
  • 503 Service Unavailable
    이 코드는 서비스가 일시적으로 이용불능이나 나중에는 회복될 것임을 표시한다. 서버가 언제 서비스가 다시 이용 가능할지를 안다면 Retry-After 헤더가 제공될 수도 있다.

  • 504 Gateway Time-out
    이 응답은 408(Request Time-out)과 동일하나 게이트웨이나 프록시의 정해진 시간이 종료되었음을 의미한다.

  • 505 HTTP Version not supported
    요청에서 이용된 HTTP 프로토콜 버전을 서버가 지원하지 않는다.
