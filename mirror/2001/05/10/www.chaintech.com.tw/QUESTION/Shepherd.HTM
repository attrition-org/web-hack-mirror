Shepherd Q&A

  • Shepherd download options
  • Shepherd Development History

 

  • What is Shepherd ?
    Shepherd is a free hardware monitoring tool running on Windows 95/NT which is able to display data offered by the BIOS DMI (system configuration information) and onboard hardware monitoring (CPU temperature, core voltage, etc.)

  • Which Chaintech boards are supported by Shepherd ?
    The regular Shepherd updates can be used on all Intel TX/LX/BX based mainboards which have hardware monitoring onboard, either directly on the PCB or through FAN78/II. The non-Intel Socket 7 mainboards that have the ability to use FAN78 can use only v1.x of Shepherd.

  • How to install and launch Shepherd ?
    To install Shepherd, download the program and unpack the downloaded files in a directory on your disk. Then run the SETUP.EXE program. After successful installation, the Shepherd folder will be added to your Start menu. Now you can launch Shepherd from your Start menu.

  • Compatibility and other issues when running Shepherd :

  • Do not use Shepherd if other hardware management software, like Intel's LDCM, is being used. Because both programs will poll the same I/O address(es) to retrieve data from the onboard monitoring hardwaredress the system will crash when doing so

  • Do not use Shepherd if ACPI is enabled !

  • When running Shepherd on the CT-6BTM board, the CPU and case temperature is sometimes shown reversed. This is due to a minor difference in the layout between two versions of the CT-6BTM. Shepherd is not able to distinguish between these two versions and will thus show these temperature values reversed on some boards. Shepherd v2.3 and later will add a reverse function to overcome this problem.
  • Information about the system configuration such as the CPU, busspeed, etc. is often wrongly represented due to the fact that the DMI information in the BIOS is not correct. For example, when overclocking a 66MHz CPU to 100MHz on the CT-6BTM, Shepherd will still show the CPU busspeed as being 66MHz. Please note that this system information is gathered through the BIOS data and not through actual tests on the board.
  • Shepherd needs at least 256 colors to run. When using 16 color mode Shepherd will not function.

Shepherd is written and constantly updated by Chaintech's own software engineers. We appreciate your comments and problem reports. You can send your ideas and problem reports to our e-mailbox with subject title Shepherd Report. Please make sure to use the latest version of Shepherd (see download area) and give a description of your configuration (especially mainboard model and BIOS datecode). We will use your comments and problem reports to update and improve Shepherd. Please note that we will NOT react to your mails other than to confirm their reception.
