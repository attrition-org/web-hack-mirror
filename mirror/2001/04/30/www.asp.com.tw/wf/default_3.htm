1. 使用輕鬆

● Web-based流程控管，Internet嘛也通

流程於Internet網際網路和intranet內部網路都通行無阻。

使用瀏覽器做為流程表單的使用者介面，大幅降低學習成本。

使用瀏覽器透過網路即可完成表單製作、建立表單流程等工作，不須跑到伺服

器操作。

維護容易，版本更新只要更新伺服器軟體即可，不須更新所有用戶端的電腦。

表單檔案小，用戶端執行速度快。 自動發E-mail通知下一關的處理者，或由瀏

覽器得知。

● 表單處理流程看得見，追蹤容易

填寫表單傳送時，可看到於表單流程圖的傳送動畫，增加使用意願，降低導入

阻力 ，追蹤稽催容易。

可由表單流程圖看到流程進行處理狀況，於圖上按下滑鼠即可輕鬆發E-mail催促

處理者。

● 於瀏覽器處理表單，流程不中斷

待處理者將收到E-mail，按下超連結即可於瀏覽器上批示或提供意見。

或者於使用者登錄後，即可於瀏覽器看到待處理與被通知的表單直接處理。

不依賴E-mail傳送流程，不會因E-mail傳送不到或誤刪除而中斷了表單流程。

● 中文全文檢索功能，快速搜尋資料

提供中文全文檢索模組，可以輸入關鍵字搜尋到表單內的資料。可以搜尋到

Word、Excel、HTML、TEXT等格式的附件資料。

● 使用數位簽名與SSL確保安全

每個表單可以是否選擇使用SSL加密以確保資料傳輸的安全。

數位簽名（選購）確保不可否認性與資料傳輸安全性
● 暫無E-Mail Server環境也可使用

於無E-Mail Server環境，只要於每台 Client 電腦安裝一個 [WebFlow表單通知] 軟

體，即可主動通知是否有待處理或通知的表單。 即使在暫時沒有E-mail的環境

下也可使用。

2. 表單製作簡單

● 使用Word、FrontPage輕鬆完成表單製作

表單為一般HTML（或ASP）格式，只要會製作網頁，就會製作表單，可以使用

Word、FrontPage、或Visual InterDev等製作。

不用學習特殊的表單開發工具，免受限於其所未提供的功能。

表單可使用各種ActiveX、Java等開放豐富的網頁設計資源。

● 一個圖檔，就是一個電子表單

可以將一個圖檔，使用「挖洞法」於瀏覽器拖拉待填寫的欄位，即可製作完成

一個電子表單。

圖檔可由電腦檔案儲存成圖檔、或由掃瞄器將現有表單掃瞄進電腦。

支援中文直式表單，中文直立垂直顯示。

● 製作精靈輕鬆完成表單和流程製作

提供製作精靈，五步驟輕鬆完成表單和流程製作。

可使用Word、或FrontPage、asp製作一個HTML檔案，經過「自動偵測欄位」功

能，即可完成表單製作，免做設定。

可設定表單流程開始時和流程當中，於圖形設定表單各欄位是否可以變更。

● 自動建立資料表功能

自動建立查詢、分類資料表(免寫程式)，提供關鍵字全文檢索與分類查詢、樞紐

分析功能。可經自訂之既定流程新增、修訂、作廢表單資料。

● 資料表精靈功能

準備一個圖檔、或一個HTML檔案(可使用Word、或FrontPage、asp製作)，經過「

挖洞法」、「自動偵測欄位」功能，即可自動建立查詢、分類資料表(免寫程式

)、樞紐分析，可新增、修改、刪除。

● 文件管理功能

流程結束時新增資料到資料表後，使用者可隨時調閱並修訂或作廢表單資料。

                                

3. 流程規劃容易

● 圖形化流程製作，輕鬆完成流程規劃

透過Internet或intranet於瀏覽器拖拉（Drag and Drop）圖形化的任務編組與上一級

主管、或關係人，即可輕鬆建立表單流程圖。 任務編組方式，不需輸入實際組

織，只需規劃將會於流程用到的群組，人員異動不需變更表單流程。群組可包括

群組。

● 多種取得上一級主管的方式

上一級主管除了取決於組織圖外，也可設定取決於Exchange Server Organization、

或取決於自訂資料庫。

● 多種關係人

可自定關係人為上一級主管、職務代理人、助理、老師、班長、或家長。 只要

準備一個Access或Excel檔即可 匯入關係人。

● 組織圖

於瀏覽器建立組織圖，以設定上一級主管和權限。 只要準備一個Access或

Excel檔即可 匯入組織圖。

● 上一級主管有多人時，使用者可自己選擇

上一級主管取決於組織圖或自訂資料庫時，如一人身兼數職，造成上一級主管有

多人時，使用者可由畫面自己選擇正確的上一級主管。

● 多種流程處理方式

於表單流程圖設定串簽、會簽、條件式流程等處理方式：

串簽：一關處理完後送到下一關。

會簽：平行單位都處理完後才送到下一關。

條件式流程：表單某些欄位或申請者、處理者的名稱/職稱/部門等符合所設定的

條件時，臨時改變跑到那一關處理。

可依照欄位簡易條件設定、或執行自訂程式判斷。

自訂程式可設定下一個處理者為某一個人、或某一群組。條件處理可設定為申請

者的名稱/職稱/部門。

● 臨時改變下一關處理者的功能

包括加簽、退簽、跳簽、變簽、手動分派、自定流程等方式：

加簽：可臨時改變送件到某人加簽後，再繼續跑下一關流程。

退簽：若不同意可臨時改變退件至任一關，繼續跑流程。

跳簽：若同意可臨時改變送件到任一關或任一人。

變簽：可於表單欄位、或條件流程規劃中指定下一個處理者為某一個人。

手動分派：臨時手動分派下一關的處理者為下一關群組或所有人中的某一個人。

自訂流程：臨時自訂流程。

● 依照表單內容動態改變流程

申請者/處理者於表單可以自行指定下一個處理者、處理群組、或上一級主管，

和填寫多個通知人(Cc)、主旨。處理者、或處理群組可以指定為多個。
● 副本通知功能

每關處理後可副本（Cc）通知處理者關係人、或群組。

流程進行狀況可設定是否以E-Mail通知申請者。

● 所有人或任一人處理選擇

各關處理可設定為群組所有人都處理完才繼續跑流程，或只須任何一人處理即可

繼續跑流程。

● 附件功能

可加附件。流程中每一關可設定處理者是否可以新增、刪除附件、或須有附件。

可依據不同流程，設定上載附件之檔案大小限制。

● 撤回申請功能

申請者可以撤回申請，自動發E-mail通知所有已處理者。

撤回時可執行一個自己的程式。
● 不同表單不同代理人功能

可設定某一期間不同表單由不同人代為處理。

代理人設定可選擇任何時間及可設暫停代理。

● 逾時通知功能

每一關可設處理期限，逾時處理時，可自動發E-mail通知申請者、處理者、或處

理者的關係人。 每一關可設逾時期限。

● 查詢逾時處理表單功能
可依期間及逾時天數(多少天之內)查詢未處理表單以供追蹤。
● 多個上班行事曆功能

為了讓逾時通知/處理和處理效率統計分析功能的時間計算更為準確，可依照不

同部門或個人設定不同的上班行事曆。

建立新的上班行事曆時，可以由舊行事曆修改而成。

● E-Mail通知功能

可以E-Mail通知申請者流程進行之狀況。 流程當中每一關可設定表單傳送至下一

關要不要寄發E-Mail。

可於流程中設定不發送E-Mail通知下一關處理者，可於表單設定，或於自訂條件

中設定。

● 測試模式功能

有權利規劃表單者可以模擬任何一個使用者，進行模擬測試。

所有文件和E-Mail將傳送給測試者，測試者可於 [處理追蹤] 的 [處理與通知] 直接

測試模擬。

測試模式資料不列入統計分析。

● 申請者自訂流程功能

申請者可依照表單內容，自已臨時制定流程，透過Internet或intranet於瀏覽器拖拉

（Drag and Drop）圖形化的任務編組與上一級主管，即可輕鬆建立表單流程圖。

可依表單設定是否可以由申請者自訂流程，將先顯示由規劃者所預設的流程圖，

若申請者申請過此表單，將自動顯示上次的流程圖，以供申請者於瀏覽器拖拉修

改。

● 批次處理功能

處理者可以一次選取多筆表單進行處理，進入批次處理模式，送信之後將暫時不

出現送信之流程圖，直接載入下一筆需處理之表單。

● 快速處理功能

快速處理讓處理者不須看到表單詳細內文，即可快速傳送處理。

可依據不同表單流程，設定是否可以快速處理。

● 重新分派功能

系統管理者可於流程圖上直接將表單重新分派給其他使用者，解決表單處理狀態

停滯或延遲。

● 直接選擇處理者或群組功能
表單中可以直接由選單依照分類或關鍵字查詢來選擇多個處理者或處理群組。
● 佈告欄功能

可於表單送出時，顯示一段文字或圖片(佈告欄)。如有多個佈告欄將採隨機方式

顯示。

 

4. 與現有資料庫軟體整合快速

● 快速將現有軟體賦予流程控管功能

WebFlow使用您現有軟體的資料庫資料與變數與現有軟體做溝通，您可以不用修

改現有軟體，您可將WebFlow當成黑箱，免API，即可快速將現有軟體賦予流程

控管的功能。

● 可將ASP程式當成表單，彈性大不受限

可將ASP程式當成一個表單，於流程當中動態地整合現有軟體資料庫、或顯示最

新的資料。

● 只設定免寫程式，即可與現有資料庫整合

可讀取現有資料庫的資料來預設流程開始時表單各欄位的值，並可設定表單流程

結束時，新增一筆資料到現有資料庫。

您可從現有SQL Server、Oracle、Sybase、Access、dBase、Excel等支援ODBC的資

料庫資料，設定為表單各欄位的預設值。

可於表單流程開始、流程當中、或流程結束時，執行一個ASP程式檔案，只須幾

行程式即可存取現有軟體所使用的資料庫資料，以整合現有軟體。

表單各欄位的值儲存為變數，方便於流程中與現有軟體的資料庫作整合。

將欄位值與資料庫資料比較結果，設定變數值即可改變流程。

● 可由程式或於流程中來觸發流程

由VB、VC、ASP等程式觸發流程。

可於流程中、或流程結束時自動啟動觸發其他的流程。 觸發流程可含附件。

● 使用變數與外部程式溝通，

流程當中可由變數取得或設定表單欄位的值、關名、 同意/不同意、意見、使用

者、申請者相關資訊、下一關處理者、上載的附件資訊、錯誤訊息。

● 提供多種函數與程式配合：

提供多種函數，提供包括取得使用者資料、 群組成員、計算某段期間行事曆的

上班時天數、設定代理人、啟動或暫停代理人動作、匯入使用者資料表等功能。

5. 管理零負擔

● 表單、流程變更版本控制

表單、流程變更後記錄版本，可註明變更原因與設定實施日期。表單或流程變更

後，原流程中的表單繼續使用舊表單與跑舊流程。實施日期後自動啟用新的表單

或流程。由版本記錄中，可查到各版本的表單與流程、變更日期、變更者、變更

原因、與實施日期。

● 流程資料備份管理

可備份(或刪除)流程資料，可依表單或處理狀況選擇備份部份，並可匯入備份過

的資料。

●與Windows NT、Exchange Server整合

可使用Windows NT使用者帳號登入做為安全驗證。

可設定有Exchange Server Mailbox 之使用者即可使用本系統，不須於本系統手動

建立使用者資料。

上一級主管可取決於Exchange Server Organization。 於瀏覽器收發郵件、排定行

事曆、連絡人、工作、記事等功能。

於瀏覽器新增Windows NT、Exchange Server使用者。

可以匯入Windows NT、Exchange Server的使用者帳戶資料。

只要準備一個Access或Excel檔即可整批新增或刪除。

● 權限管理功能實用，複雜組織也適用

個人化表單內容（讀取權限），可以規劃那個表單分類屬於那個群組專用，或所

有人都可以申請使用。

多人各自維護表單（規劃權限），可以規劃那個表單分類屬於那個群組才可製作

表單、建立表單流程圖。

各自管理組員（管理權限），可以規劃那個群組可以「管理」那個群組的組員。

● 流程統計樞紐分析圖表，透視流程瓶頸

用戶端安裝Office 2000，即可於瀏覽器查看流程統計樞紐分析圖表，提供針對表

單、人員、部門分析申請數量、處理數量、處理效率的長條圖、折線圖圖表。

分析統計資料為EXCEL格式，方便移做它用。

● 隨裝即用，免東設西設

安裝十分容易，安裝完即可使用，不須到Exchange Server、OutLook、資料庫、

Web Server、ODBC東設西設裝半天。使用者電腦免安裝其他軟體。

● 多層式Web架構，執行效能高，擴充易

採用Transaction Server多層式架構，Exchange Server、和資料庫可與Web Server同

一台PC，或隨成長需要裝於另外一台PC，多人上網共享資源。

● Microsoft Windows 2000 相容

支援更穩定更快速的Microsoft Windows 2000 Server和現有 Windows NT Server

4.0作業系統。

● Transaction功能確保流程資料與您資料的一致性

於流程當中或流程結束時，可以更新您的資料庫，若無Transaction功能將無法確

保流程資料與您資料的一致性。

譬如流程當中或流程結束時，若流程跑失敗，可能誤更新您的資料庫；或者流程

跑成功但是更新您的資料庫卻失敗。

使用Transaction功能時，當傳送到下一關時若也更新您的資料庫時，當任何一個

資料更新失敗時，已經誤更新的資料庫資料將自動回覆成原值。

可依據不同表單流程，設定是否使用Transaction功能。 您必須使用譬如SQL

Server、Oracle等支援MTS Transaction功能的資料庫，才可使用Transaction功能。

● 錯誤記錄功能，分析意外錯誤原因
執行當中若發生錯誤，記錄(log)於 \webflow\wflogfile.txt 檔案當中。
