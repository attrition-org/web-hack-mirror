1. 資料庫化的網站

甚麼是「資料庫化的網站」？

於上一期當中，談到目前Internet上大部份的網站，仍然提供「靜態」（static）的網頁內容，所謂「靜態」的網站，指的就是網站的網頁內容是「固定不變」地。而相對地所謂「動態」的網站，所指的是「內容會變」、「自動更新」、和「互動性」的網站。

將原本「靜態」的網站，變成一個「動態」的網站，方法為將網站「asp化」（採用於IIS網站伺服器執行的asp碼，使用程式控制，於伺服器端動態地產生網頁內容）、「動態HTML化」（採用4.0版IE和Netscape瀏覽器所提供的動態HTML，於瀏覽器端動態地改變網頁內容）、以及將網站「資料庫化」。

甚麼是「資料庫化的網站」？所謂將網站「資料庫化」，指的是使用「資料庫」管理整個網站，只要更新資料庫的內容，網站的內容將自動更新。

將網站「資料庫化」的好處在那裡？

採用資料庫，將網站「資料庫化」的好處在那裡？好處如下：

  • 自動更新網頁：只要更新資料庫的資料，網頁內容會自動更新。

  • 可以搜尋：將網站的內容儲存於資料庫當中，可以搜尋網站的資料。

  • 各種Web資料庫應用。

讓我們看個例子。

「新消息」真的是「新的消息」嗎？

當連上網際網路瀏覽各網站時，偶而會看到「過期的新消息」，甚至去年的消息仍然列為「新消息」，這時候您就可以寫封信給站長，來嘲笑其新消息「過期」了。

其實這不是站長的錯，日期一天天的過，總不能要求站長每天都去檢查每一則「新消息」是否過期。

但是如果站長將網站「資料庫化」，使用「資料庫」來管理網站的「新消息」，只要更新資料庫的內容，網站的「新消息」將自動更新。

譬如筆者網站「www.asp.com.tw」中的「新消息」部份，每一則「新消息」都於「資料庫」設定其「開始日期」和「結束日期」。當使用者瀏覽此網頁時，將自動比較當天日期是否介於資料庫中「開始日期」和「結束日期」之間，只將符合此條件的「新消息」顯示出來，顯示出來的「新消息」保證是「新的消息」。

加入「新消息」時，只要於資料庫新增一筆資料，於資料庫設定「新消息」的「開始日期」和「結束日期」即可，如此一來，網站的「新消息」內容將自動地更新，「新消息」永遠是「新的消息」，站長可以安心地睡個好覺，不用擔心半夜有訪客會寫信來嘲笑「新消息」過期了。

下次您看到「過期的新消息」時，仍然可以寫封信給站長，不是嘲笑其新消息「過期」了，而是嘲笑其網站仍未「資料庫化」。

開發各種Web資料庫的應用

將網站「資料庫化」的另一個好處，就是可以開發出各種Web資料庫的應用。

使用者使用瀏覽器，透過網際網路或intranet內部網路，來存取Web資料庫的資料，可以使用於學校教學、醫院、商業、銀行、股市、和運輸旅遊等各種Web資料庫的應用，譬如銀行餘額線上查詢、線上機票訂位、線上醫院預約掛號、網路下單、線上股市買賣交易、線上學校註冊選課、線上擇友、線上購物等各種應用。

筆者準備一個展示各種Web資料庫設計實例的網站，網址為「www.asp.com.tw」，歡迎讀者光臨指教。

將目前各種網路上的應用軟體改變為「Web資料庫」的應用，使用「瀏覽器」做為使用者介面，是必然的趨勢。

以往每個使用者都必須去學習每一種應用軟體的操作方式。Web資料庫應用使用「瀏覽器」做為使用者介面，對於使用者的好處是只要學會瀏覽器的操作方式，譬如看到有超連結的部份就按下去，如此可以操作各種軟體應用，大幅降低訓練的成本。

以往每當修改一個bug時，就須為所有使用者去更新一個新的版本。Web資料庫應用對於設計者的好處是，新版本的軟體只要更新一次網站的軟體就可以了。

基本上，所有的網路上的應用軟體，都可以改變為Web資料庫的應用，使用「瀏覽器」做為使用者介面。讓我們趕快來學會如何設計一個Web資料庫的應用。

　

2. 存取Web資料庫的解決方案

存取Web資料庫的解決方案

如何讓使用者於瀏覽器畫面中，透過Internet、或Intranet，查詢Web資料庫的資料，甚至輸入、更新、和刪除網站伺服器之資料庫資訊呢？

存取Web資料庫的解決方案有數種，如下：

  1. CGI（Common Gateway Interface）：傳統的方式。

  2. IDC（Internet Database Connector，Internet資料庫連接頭）：簡易的Web資料庫存取解決方案。

  3. ADO（ActiveX Data Object，ActiveX資料物件）：完整的Web資料庫存取解決方案。

  4. RDS（Remote Data Service）：在用戶端的瀏覽器上運作資料查詢動作的頻繁的資料查詢之解決方案。

簡介如下：

CGI

CGI（Common Gateway Interface）的傳統方式，當於瀏覽器填好表單（form）的輸入資料，提出HTTP要求，要求於網站伺服器執行一個表單所設定的可執行應用程式。網站伺服器收到要求執行應用程式後，CGI程式分析表單（form）的輸入資料，存取Web資料庫，將查詢執行的結果以HTML的格式傳送給瀏覽器。

使用CGI方式存取Web資料庫，有很多的缺點，譬如不易開發、變更修改成本高、功能有限、不易偵錯、執行速度慢等，而且不具備交易（transaction）的功能。

於Windwos NT上，CGI將被ASP所取代，是必然的趨勢。。

IDC（Internet Database Connector）

Windows NT Server內含Internet Information Server（簡稱IIS） 2.0，提供Internet Database Connector（Internet資料庫連接頭，簡稱IDC）的技術，讓您建立提供資料庫資訊的網頁內容，於網頁畫面執行Structured Query Language（結構化查詢語言，簡稱SQL）指令，讓使用者於瀏覽器畫面中查詢、輸入、更新、和刪除網站伺服器之資料庫資訊。

IDC特點如下：

  1. 內含於NT Server的Internet Information Server 2.0版本當中。

  2. 設計簡易，只要準備兩個檔案，即可於使用者的瀏覽器，存取網站伺服器之資料庫資訊。不須Compile，直接執行。

  3. 支援多重查詢（Multiple Query）的功能，即可同時下多個SQL指令。

IDC將被ADO（ActiveX Data Object）所取代，也是必然的趨勢。。

ADO（ActiveX Data Object）

存取資料庫，可採用Active Server Pages（簡稱ASP）內建的一個ActiveX Server component，叫做Database Access component，使用ActiveX Data Object（簡稱ADO）的技術，讓您可以與Active Server Pages結合，以提供存取資料庫的功能。

ActiveX Data Object（ADO）是Web資料庫最佳的選擇。

ActiveX Data Object（簡稱ADO）是甚麼東西呢？ActiveX Data Object的技術，讓您可以與Active Server Pages（簡稱ASP）結合，以建立提供資料庫資訊的網頁內容，於網頁畫面執行Structured Query Language（結構化查詢語言，簡稱SQL）指令，讓使用者於瀏覽器畫面中輸入、更新、和刪除網站伺服器之資料庫資訊。

當於用戶端的瀏覽器填好表單（form）的輸入資料，按下「Submit」按鈕後，經過Internet、Intranet傳送HTTP訊息到網站伺服器，請求於網站伺服器執行一個表單所指定的Active Server Pages程式碼（附檔名.asp的檔案）。

一個.asp檔案是一個文字檔，包括：HTML標記（tags）、VBScript或Jscript語言的程式碼、ASP語法、和Structured Query Language（結構化查詢語言，簡稱SQL）指令。

IIS 3.0/4.0網站伺服器執行.asp檔案，透過ODBC驅動程式，連接到支援ODBC的資料庫，執行SQL指令，將執行的結果以HTML碼的格式傳送給瀏覽器。

　

ADO使用RecordSets物件，做為資料的主要介面。

ADO可使用VBScript、JavaScript語言來控制資料庫的存取，與查詢結果的輸出顯示畫面。

ADO主要的優點，為容易使用、快速、消耗少量的記憶體，和占用少量的磁碟空間。

ADO提供以下的特點，以建造主從式（client-server）和web-based的運用，包含：

  • 可以獨立地創造物件（Independently-created objects）。

  • 批次更新（Batch updating）。

  • 以in/out參數與返回值，支援儲存的程序（stored procedures）。

  • 多種游標模式（cursor type）。

  • 先進的recordset暫存記憶體管理（cache management）。

  • 可限制傳回的查詢筆數。

  • 提供由儲存的程序（stored procedures）或批次SQL指令，傳回多組recordsets的功能。

於伺服器端所須的ADO執行環境，如下：

  • Windows NT Server安裝Internet Information Server（IIS）3.0（含）以上版本。

  • Windows 95/98安裝Personal Web Server（PWS）4.0版本。

於用戶端所須的ADO執行環境，只須一般的瀏覽器即可。

ADO的相關資料，可於以下的網址取得，如下：

http://www.microsoft.com/ado

http://www.microsoft.com/iis

http://www.microsoft.com/intdev

RDS（Remote Data Service）

RDS（Remote Data Service），以前叫做ADC（Advanced Data Connector，先進資料連接頭），提供一個資料處理「AdvancedDataControl」的ActiveX Control，以存取ODBC（Open Database Connectivity）的資料庫。

一般的網站資料處理技術，譬如ADO、IDC等，資料查詢的動作，是在伺服端的網站伺服器運作的，每次查詢資料的要求，都須由用戶端發起送到伺服端，伺服器運作查詢資料庫後，再傳回用戶端的瀏覽器顯示出來。這一來一往，對頻繁的資料查詢動作，效率較差。

RDS與其他網站資料處理技術，有一個很大的不同點，就是RDS的資料查詢動作，是在用戶端的瀏覽器上運作的。RDS將伺服器資料庫可高達數千筆的資料，先下載到用戶端，在用戶端的瀏覽器直接做頻繁的資料查詢動作，大部份時間不須再與伺服器連線，效率較佳。這個資料查詢就是由安裝於用戶端的「ADC ActiveX Control」處理的。

RDS的特點，如下：

  1. 於用戶端執行資料查詢：將資料庫的大量資料暫存於用戶端（前端），直接於用戶端的瀏覽器查詢資料，節省前端（用戶端）和後端（伺服端）之間資料往返所須的時間。適合於Internet、Intranet、或LAN上，做頻繁的資料庫資料存取查詢動作。

  2. 是一個ActiveX Control：由網站伺服器自動下載和安裝ADC的ActiveX Control於用戶端，無須另行安裝或設定。用戶端須使用能支援ActiveX的瀏覽器，譬如IE 3.0/4.0等，才能存取網站伺服器的資料庫資料。由於是一個ActiveX Control，可使用VBScript、JavaScript語言來控制此ADC的ActiveX Control，以及可與其他的ActiveX Control配合使用。

可連接各種支援ODBC的資料庫

ADO或RDS透過各種ODBC的驅動程式，可連接多種的資料庫，包括Access、Excel、FoxPro、dBase、SQL Server、Oracle、Informix等支援ODBC的資料庫。

　

　

　

　

3. ADO資料庫設計要訣

只要幾個步驟，就可以應用ADO的技術，於網站伺服器執行存取查詢資料庫的動作。

第一步驟 設定所要連結的資料庫

若要存取NT網站的資料庫，如何讓程式連接到正確的資料庫呢？

基本上，有兩種方法：

  1. 於 [控制台] 的 [ODBC] 中，新建一個系統資料來源，包括資料庫名稱（使用者資料來源名稱）、資料庫所用的驅動程式、資料庫檔案等。於ODBC設定的「資料來源名稱」後，使用「Server.CreateObject」建立連線的物件，並使用「Open」開啟待存取查詢資料庫，程式如下：

    2. Set Conn = Server.CreateObject("ADODB.Connection")

    Conn.Open "資料庫名稱"

    資料庫名稱即為於ODBC設定的「資料來源名稱」。

  2. 第一個方法有時候用起來並不方便，譬如當網站伺服器不在身旁、或向ISP租用虛擬主機時，這時候並不方便跑到網站伺服器的 [ODBC] 中去新建一個系統資料來源。於是ASP提供了一個直接於ASP程式碼中指定所要連結的資料庫的方法，使用「Server.CreateObject」建立連線的物件後，使用「Open」直接連結到待存取查詢資料庫，對於Access資料庫而言，程式如下：

Set Conn = Server.CreateObject("ADODB.Connection")

conn.Open "DBQ="& Server.MapPath("資料庫檔案名稱") &";Driver={Microsoft Access Driver (*.mdb)};DriverId=25;FIL=MS Access;"

其中Server.MapPath("資料庫檔案名稱") 取得資料庫的絕對位址。

對於SQL Server資料庫而言，程式如下：

Set Conn = Server.CreateObject("ADODB.Connection")

conn.Open "DBQ=資料庫檔案名稱" &";Driver={SQL Server;Server=(Local);UID=sa;PWD="

第二步驟 執行SQL指令

當兩個使用者同時瀏覽同一個網頁時，於第一個使用者已經開啟資料庫尚未關閉前，若第二個使用者也要開啟相同的資料庫時，將產生錯誤的訊息，要如何解決呢？

解決之道為使用ADO的Recordset（記錄集合）物件。將資料庫的資料先儲存到Recordset當中，就可以讓多位使用者同時存取同一個資料庫。

使用Recordset物件前先定義如下：

Set rs1 = Server.CreateObject("ADODB.Recordset")

第二步驟設定SQL指令，即可開始執行存取查詢資料庫的動作。如下，其中Conn為第一步驟所定義的名稱：

RS.Open SQL指令, Conn, 1, 3

第三步驟

第三步驟，使用如下RecordSets物件的指令，顯示查詢的結果，其中RS為第二步驟所定義的名稱：

  • RS.Fields.Count：RecordSets的欄位數。

  • RS(i).Name：第i（指標）個欄位名稱，i由0算起到RS.Fields.Count-1。

  • RS(i)：讀取第i（指標）個欄位的資料，i由0算起到RS.Fields.Count-1。

  • RS(“欄位名稱”)：讀取指定欄位的資料。

  • RS.EOF：是（True）否（False）已指到最末筆。

  • RS.MoveNext：將指標移到下一筆。

  • RS.MovePrev：將指標移到上一筆。

  • RS.MoveFirst：將指標移到第一筆。

  • RS.MoveLast：將指標移到最末筆。

  • RS.GetRows：將查詢結果存於陣列當中，如v=RS.GetRows(100)，再讀取陣列v(col,row)的值即為查詢的資料。

  • RS.Properties.Count：得知ADO的Connection或Resultset之屬性總數。

  • RS.Properties(Item).Name：得知ADO的Connection或Resultset之屬性名稱。

  • RS.Properties(Item)：得知ADO的Connection或Resultset之屬性值。

最後，顯示後使用如下的指令，關閉資料庫：

RS.Close

Conn.Close

ASP程式碼，如下：

<%

Set Conn = Server.CreateObject("ADODB.Connection")

Conn.Open "BookSamp

　

Set RS = Server.CreateObject("ADODB.Recordset")

RS.Open "SELECT * FROM Product", Conn, 1, 3

%>

將查詢的結果顯示於一個表格的程式碼，如下：

<TABLE BORDER=1>

　

<TR>

　

<% For i = 0 to RS.Fields.Count - 1 %>

<TD><B><% = RS(i).Name %></B></TD>

<% Next %>

　

</TR>

　

<% Do While Not RS.EOF %>

<TR>

<% For i = 0 to RS.Fields.Count - 1 %>

<TD VALIGN=TOP><% = RS(i) %></TD>

<% Next %>

</TR>

<%

RS.MoveNext

Loop

RS.Close

Conn.Close

%>

</TABLE>

查詢資料庫的SQL指令

若要查詢資料庫的資料，係使用SELECT的SQL指令。

SELECT的基本語法，如下：

SELECT 欄位名稱#1, 欄位名稱#2, ...

FROM 資料表名稱#1, 資料表名稱#2, ...

WHERE 條件式

ORDER BY 欄位名稱#1, 欄位名稱#2, ...

  • SELECT：所接的欄位名稱為待查詢的欄位，各欄位名稱之間以逗點分隔之。

  • FROM：所接的資料表名稱為待查詢的資料庫資料表（table）名稱，各資料表名稱之間以逗點分隔之。

  • WHERE：所接的條件式為設定查詢條件式。

  • ORDER BY：所接的欄位名稱為排序的欄位，將查詢的資料加予排序。指定多個欄位時，先以欄位名稱#1排序，若資料相同再依欄位名稱#2排序，...，各欄位名稱之間以逗點分隔之。

增加資料庫資料的方法

若要新增資料，首先於開啟資料庫後，使用AddNew方法，設定各欄位的資料後，最後使用Update方法，如下：

RS.AddNew

RS(“欄位名稱#1”).VALUE = 欄位#1資料

RS(“欄位名稱#2”).VALUE = 欄位#2資料

…

RS.Update

<%

修改資料庫資料的方法

RS(“欄位名稱#1”).VALUE = 欄位#1資料

…

RS.Update

<%

Set conn1 = Server.CreateObject("ADODB.Connection")
