ActiveX Server Components

於Active Server Pages（簡稱ASP）當中，使用「ActiveX伺服器元件」（ActiveX Server Components，簡稱「元件」），讓您重複地使用設計好的功能（像副程式般可以一再地重複呼叫使用），製作出動態、互動的網頁內容。

「ActiveX Server Components」，以前叫做「Automation servers」，讓您將網站伺服器成為網頁應用程式的一部份。ASP內建的「元件」提供存取資料庫、讀寫伺服器檔案、輪流顯示廣告圖片、得知用戶端瀏覽器的資訊、和管理超連結表等精彩的功能。

ASP可經由「元件」來擴充增加功能，可使用Visual Basic、Java、Visual C++、COBOL等程式語言來發展。

「元件」一般由.asp檔案的ASP程式碼所呼叫使用，但是也可以由ISAPI應用程式、其他伺服器元件、或其他OLE相容的語言所呼叫。

網站「元件化」的好處

所謂將網站「元件化」，指得是於ASP當中使用「元件」，好處如下：

  • 可以重覆使用（Reuse）。

  • 執行速度增快：據筆者測試的結果，於ASP當中使用「元件」，必未使用「元件」的執行速度快約10~30%。

  • 保護原始碼：由於「元件」是compile後的.dll檔案，別人看不到您的原始碼。

ASP內建的「元件」

ASP內含數個內建的「元件」，如下：

  1. File Access component（檔案存取元件）：經由FileSystemObject物件、和TextStream物件，提供讀寫伺服器檔案的功能，常見的訪客計數器、和每次載入網頁時都給一個不同的提示或座右銘等應用，都可以經由此ActiveX伺服器元件設計出。

  2. Page Counter Component（訪客計數器元件）：記錄訪客的次數，每次有訪客瀏覽此網頁，自動將訪客計數器加一。

  3. Content Rotator Component（內容輪播器元件），讓HTML內容輪流顯示於網頁上，您可設定各內容出現的機率。

  4. Ad Rotator component（廣告輪播器元件）：提供廣告輪播器的功能，讓廣告圖片輪流顯示於網頁上，您可設定各廣告出現的機率。

  5. Browser Capabilities component（瀏覽器資訊元件）：提供用戶端使用瀏覽器的功能資訊給網站伺服器。

  6. Content Linking component（內容連結元件）：讓您很方便地使用一個文字檔，來管理超連結、上一頁、下一頁等連結功能。幾行的ASP程式碼，加上準備一個URL表的文字檔，網頁內容將自動更新，不須修改網頁的程式碼。

  7. Permission Checker Component（允許檢查元件）：讓設計者檢查使用者是否有權存取某個檔案，以便決定顯示不同的網頁內容給不同的使用者。

  8. Database Access component（資料庫存取元件）：使用ActiveX Data Object（簡稱ADO）的技術，讓您可以與Active Server Pages結合，以提供存取伺服器資料庫的功能。

除了這些「內建」的「元件」，您可以製作自己的「元件」，來符合自己特殊的需求，以增加擴充Active Server Pages的功能，可使用Visual Basic、Java、Visual C++、COBOL等程式語言來發展。或購買別人已經開發好的「元件」來使用。建議您使用瀏覽器連線上http://www.microsoft.com/iis/ 以下載新的components。

定義一個元件

要使用「元件」須先定義。定義一個元件，以產生元件的instance，定義後即可呼叫此元件的方法（methods）、或存取其屬性（properties）。

定義一個「元件」有兩個方法：

  1. 使用「CreateObject」的方法定義一個元件的方法如下：

    2. <% Set obj1 = Server.CreateObject(”專案名稱.class名稱") %>

    譬如以下定義一個Browser Capabilities component：

    <% Set bc = Server.CreateObject("MSWC.BrowserType") %>

  2. 使用<OBJECT> 標記定義一個ActiveX伺服器元件。譬如以下定義一個Ad Rotator Component：

<OBJECT RUNAT=Server ID=MyAd ROGID="MSWC.AdRotator">

</OBJECT>

一般而言，您可以於Global.asa使用<OBJECT> 標記，定義使用於一個session、或application的元件。

企業網站熱門應用精選

「企業網站熱門應用精選」，為喬篷科技公司採用微軟相關產品與技術所開發的套裝軟體，提供Microsoft SQL Server資料庫的網站完整解決方案，包括許多ASP元件，您可以使用這個產品將您的網站「元件化」，迅速提供12個常用的網站功能，而不需要懂得如何寫ASP和元件，特點如下：

1. 不須會寫程式，馬上使用：

將12個常用的網站功能，整合成一個套餐，馬上即可使用，不須寫程式。

選項部份，由瀏覽器做設定。

2. Choose on-demand：

可依據實際需要，從12個常用的網站功能，挑選所須的數個功能。

3. 動態的網頁內容：

使用資料庫管理「網站內容」，只要由瀏覽器更新資料庫，不用找人修改網頁，自動產生新的網頁內容。

4. 網頁與美工部份可自行設計：

由於使用include和Component的方式，網頁部份可自行設計，只須將所用到的網站套餐功能包含於所設計的HTML碼中。

您現有的網站可以於任何一個網頁當中，加入任何一個「企業網站熱門應用精選」所提供的網站常用新功能，只要使用記事本或FrontPage Express開啟此網頁現有的HTML碼，於想插入此新功能的地方，插入一行include的文字譬如「<!--#INCLUDE FILE="xxx.inc" -->」，再儲存成.asp檔案，即可擁有那項網站的新功能。

「企業網站熱門應用精選」所提供的12個常用的網站功能，如下：

1. 新消息（佈告欄）

2. 廣告輪播

3. 產品資訊

4. 討論區

5. FAQ

6. 檔案下載

7. 意見箱

8. 問卷調查

9. 活動報名

10. 線上徵才

11. 訂閱Email

12. 相關網站

「企業網站熱門應用精選」軟體所須的執行環境如下：

  • Windows NT Server 4.0/Windows NT Option Pack

  • SQL Server資料庫

「新消息」保證是「新的消息」嗎？

由於「企業網站熱門應用精選」使用「資料庫」來管理網站的「新消息」，只要更新資料庫的內容，網站的「新消息」將自動更新。

每一則「新消息」都設定其「開始日期」和「結束日期」，當使用者瀏覽此網頁時，將自動比較當天日期是否介於資料庫中「開始日期」和「結束日期」之間，只將符合此條件的「新消息」顯示出來，顯示出來的「新消息」保證是「新的消息」。

加入「新消息」時，只要使用瀏覽器於資料庫新增一筆資料，於資料庫設定「新消息」的「開始日期」和「結束日期」即可，如此一來，網站的「新消息」內容將自動地更新，「新消息」永遠是「新的消息」，站長可以不用擔心有訪客會寫信來嘲笑其網站的「新消息」過期了。

「企業網站熱門應用精選」於每天第一個網站訪客進來時，自動讀取「新消息」資料庫以產生新的網頁內容。每天只讀取資料庫一次，不須每次訪客進來時都讀取資料庫，可以加快多人瀏覽的速度。如果您的「新消息」是隨時變動的，當然您也可設定為「即時」更新新消息。

您可以設定是否顯示消息簡介或只顯示標題，也可超連結到詳細說明的html檔案。

如果當天沒有最新的「新消息」時，不用擔心，「企業網站熱門應用精選」將自動傳送EMAIL給維護人員以告之。

對於不同種類的「新消息」，您可以做分類，不同的分類可以設定不同的標題圖片。

將您現有的網站加入「新消息」的功能是一個很輕鬆的事，您只要使用記事本或FrontPage Express開啟html原始碼，於想插入「新消息」的地方，插入一行如下的文字，再儲存成.asp檔案，即可擁有使用「資料庫」來管理網站「新消息」的功能：

<!--#INCLUDE FILE="package/news.inc" -->
