1. 動態的網站

您的網站還是「靜態」的嗎？

目前Internet上許多的網站，仍然提供「靜態」（static）的網頁內容。

所謂「靜態」，指的就是網站的網頁內容是「固定不變」地。

當瀏覽器透過Internet的HTTP（Hypertext Transport Protocol）協定，向網站伺服器（web server）要求提供網頁的內容時，網站伺服器收到要求後，傳送原已設計好靜態的HTML的檔案資料給瀏覽器。

「傳統」的網站是屬於「靜態」的，僅僅使用標準的HTML碼，頂多加上GIF89A格式的圖片，產生幾隻小貓小狗跑來跑去的動畫效果，若要更新網頁的內容，必須手動地來更新其HTML的檔案資料。「靜態」網站的一個大缺點就是不容易維護，為了不斷更新網頁內容，必須不斷地製作新的HTML檔案資料。

「傳統」的網站伺服器並沒有做甚麼事，僅僅將使用者所要求的HTML檔案，傳送到瀏覽器而已。

甚麼是「動態的網站」？

一個「動態」的網站，網站的網頁內容是「動態」地。

所謂「動態」的網站，指的是：

  • 「內容會變」：網頁內容會依照不同時間、不同使用者，而提供不同的網頁內容。

  • 「自動更新」：網頁內容會自動更新。

  • 「互動性」：網頁內容會依照使用者的需要或選擇而改變。

將原本「靜態」的網站，變成一個「動態」的網站，方法如下：

  • 將網站「asp化」：所謂「asp化」，即採用於IIS網站伺服器執行的asp碼，使用程式控制，「動態」地產生網頁內容。

  • 將網站「資料庫化」：所謂「資料庫化」，即使用資料庫管理整個網站，只要更新資料庫的內容，網站的內容將自動更新。

  • 使用「動態HTML」：採用4.0版IE和Netscape瀏覽器所提供的「動態HTML」，於瀏覽器端「動態」地改變網頁內容。

「動態」的網站伺服器，必須執行一些程式，然後將將執行的結果輸出成標準的HTML格式檔案，傳送到使用者的瀏覽器。

將網站「 asp化」的好處在那裡？

採用於IIS網站伺服器執行的asp碼，將網站「 asp化」的好處在那裡？好處如下：

  • 動態：每次內容不同。

  • 增加互動性。

  • 設計迅速與維護方便：asp不須compile編譯，使用記事本即可編輯設計。

讓我們看個例子。譬如每次使用者光臨您的網站時，都會看到不同的內容。

　

　

2. Active Server Pages

還在使用CGI嗎？

當於瀏覽器填好表單（form）的輸入資料，提出HTTP要求時，可以要求於網站伺服器執行一個表單所設定的可執行應用程式，而非只是一個HTML的檔案。網站伺服器收到要求執行應用程式後，分析表單（form）的輸入資料，將執行的結果以HTML的格式傳送給瀏覽器。

使用CGI（Common Gateway Interface）應用程式，或使用Netscape公司的NSAPI（Netscape Server Application Programming Interface）、和微軟公司的ISAPI（Internet Server Application Programming Interface）等伺服器API撰寫的應用程式，可以提供「動態」（dynamic）的網頁內容。

CGI應用程式有一個缺點，就是不易產生，變更修改也很困難，而且由於並非整合於HTML當中，必須使用與HTML完全不同的設計過程來設計一個可執行的應用程式。

CGI的效率也是一個大問題，每一個表單（form）均須執行一個可執行檔，多人同時上線使用時，多個執行檔同時執行，將大大降低網站伺服器的執行速度。

即將取代CGI的是IIS的Active Server Pages。

Active Server Pages

Active Server Pages（動態伺服器網頁，簡稱ASP），內含於IIS 3.0、或4.0（或以後版本）當中，提供一個伺服器端（server-side）執行的scripting環境，您可以結合 HTML 網頁、ASP指令和 ActiveX 元件，以產生和執行動態、互動、高效率的網站伺服器（web server）的應用程式。

當瀏覽器對要求您網站伺服器的ASP程式碼（檔案副檔名為 .asp）時，譬如 http://www.asp.com.tw/test.asp，您網站伺服器即會「從頭到尾」讀取所要求的ASP程式碼test.asp檔案，並且開始執行，將「<%…%>」部份的ASP程式碼執行後，輸出標準的HTML碼，最後傳送此HTML碼到瀏覽器。

ASP程式控制部份，可使用VBScript、或JScript語言來設計。

您不需要擔心，是否瀏覽器無法執行您設計出來的Active Server Pages，您的網站伺服器，會自動將Active Server Pages的程式碼，解譯為標準HTML格式的網頁內容，再送到用戶端的瀏覽器上顯示出來。用戶端只要使用一般可執行HTM碼的瀏覽器，即可瀏覽Active Server Pages所設計的網頁內容。

Active Server Pages的特點

Active Server Pages（ASP）的特點是：

  1. 使用VBScript、或JScript等簡易的script語言，結合HTML碼，即可快速地完成網站的應用程式。

  2. 不須compile編譯，容易產生，不須compile編譯或link連結即可直接執行。整合於HTML當中。

  3. 使用一般文書編輯程式，如Windows的記事本，即可編輯設計。

  4. 與瀏覽器無關（Browser Independence）：用戶端只要使用一般可執行HTML碼的瀏覽器，即可瀏覽Active Server Pages所設計的網頁內容。Active Server Pages所使用的script語言（VBScript、或Jscript）係於網站伺服器（server端）執行，用戶端的瀏覽器不需要能夠執行這些script語言。

  5. Active Server Pages與任何ActiveX scripting語言相容。除了可使用VBScript、或JScript語言來設計，並可經由plug-in的方式，使用由third party所提供的其他譬如REXX、Perl、Tcl等scripting的語言。scripting引擎是處理script的COM（Component Object Model）物件。

  6. Active Server Pages的原始程式碼，不會傳到使用者的瀏覽器，因此可以保護辛辛苦苦寫出來的原始程式碼不會外漏。傳到使用者瀏覽器的為Active Server Pages執行結果的一般HTML碼。

  7. 使用server端script產生client端script，您可以使用ASP程式碼，於網站伺服器（server端）執行script語言（VBScript、或Jscript），來產生或修改於client端（用戶端的瀏覽器）執行的script語言（VBScript、或Jscript）。

  8. 物件導向（Object-oriented）。

  9. 可經由ActiveX Server components（ActiveX伺服器元件）來擴充增加功能。ActiveX Server component，可使用Visual Basic、Java、Visual C++、COBOL等程式語言來發展。

執行Active Server Pages所須環境

以下任何一種環境可以執行Active Server Pages：

  • Windows NT Server 4.0/5.0：執行IIS 3.0、或4.0（或以後版本）。

  • Windows NT Workstation 4.0/5.0：執行IIS 3.0、或4.0（或以後版本）。

  • Windows 95/98：執行Microsoft Personal Web Server（PWS）1.0a/4.0（或以後版本）。

ASP新增功能

Active Server Pages 到了IIS 4.0版本，增強了很多功能，如下：

  • Microsoft Script Debugger：偵錯Active Server Pages所使用的script語言（VBScript、或Jscript）。

  • 可執行交易（transaction）的指令：所有在一個網頁執行的工作，會全部成功或全部失敗。

  • 隔離的 ASP 應用程式：隔離ASP應用程式，可以與Web伺服器分開於不同的記憶體空間執行。好處是假若某一個應用程式當機時，就不會影響到其它應用程式和Web伺服器。也可以讓您不必停止Web伺服器，就可以停止應用程式或取消從記憶體載入元件。

  • 檔案上載：瀏覽器可以使用Posting Acceptor應用程式，將檔案上載至 Web伺服器。

  • 新的ASP元件（components）。

  • 應用程式的根目錄可以是實際目錄：以前，ASP應用程式的根目錄必須是虛擬目錄。現在，根目錄可以是Web站台內實際的目錄。可以使用 [Internet服務管理員] 來指定ASP應用程式的根目錄。

  • 支援 Internet Explorer 4.0頻道：ASP可以產生頻道定義的CDF檔案。

  • 支援 HTTP 1.1 通訊協定：Internet Information Server 和 Personal Web Server 支援 HTTP 1.1通訊協定。HTTP 1.1可以使用巨量編碼處理、和改進傳送網頁到瀏覽器時的效率。

ActiveX伺服器元件

使用Active Server Pages的「ActiveX伺服器元件」（ActiveX Server Components），讓您重複使用設計好的功能（像副程式般可以一再地呼叫使用），讓您製作出動態、互動的網頁內容。

Active Server Pages包括數個內建的「ActiveX Server Components 」，如下：

  1. File Access component（檔案存取元件）：提供讀寫伺服器檔案的功能。

  2. Page Counter Component（訪客計數器元件）：記錄訪客的次數。

  3. Content Rotator Component（內容輪播器元件），讓HTML內容輪流顯示於網頁上。

  4. Ad Rotator component（廣告輪播器元件）：提供廣告輪播器的功能。

  5. Browser Capabilities component（瀏覽器資訊元件）：提供用戶端使用瀏覽器的功能資訊給網站伺服器。

  6. Content Linking component（內容連結元件）：讓您很方便地使用一個文字檔，來管理超連結、上一頁、下一頁等連結功能。

  7. Permission Checker Component（允許檢查元件）：檢查使用者是否有權存取某個檔案。

  8. Database Access component（資料庫存取元件）：提供存取伺服器資料庫的功能。

您可自己製作符合自己特殊需要的「ActiveX伺服器元件」，來增加擴充Active Server Pages的功能。可使用Visual Basic、Java、Visual C++、COBOL等程式語言來製作。

ASP物件

Active Server Pages提供數個內建的「物件」（object），讓您直接呼叫使用：

  • Request 物件：取得用戶資訊。

  • Response物件：傳送資訊給用戶。

  • Server物件：提供存取伺服器的方法（methods）、和屬性（properties）的功能。

  • Application物件：Application的所有資訊，於一個應用程式，多個網頁之間，可以讓所有的使用者共同使用。

  • Session物件：Session的所有資訊，僅適用於一個使用者。

  • ObjectContext物件：提供Transaction交易處理功能的物件。

Application與Session物件，可以於多個網頁之間保留和使用一些共同的資訊，以發展網站的應用程式。

　

　

3. Active Server Pages語法

.asp檔案

Active Server Pages（ASP）製作成.asp附檔名的檔案，一個.asp檔案是一個文字檔，包括：

  • HTML標記（tags）：「<...>」的部份。

  • VBScript、或Jscript語言的程式碼

  • ASP語法：「<%...%>」的部份。

ASP語法

HTML標記（tags）使用「<...>」來將HTML碼包含起來，以與一般的文字區分開來，HTML碼係於使用者的瀏覽器當中執行；而ASP使用「<%...%>」來將ASP的script程式碼包含起來，ASP碼係於伺服器當中執行，執行後產生瀏覽器看得懂的HTML碼，再傳送給使用者。

譬如網站伺服器asp碼：

現在是: <% =now %>

於用戶端使用一般的瀏覽器，瀏覽執行的結果如下，顯示現在的日期時間：

1998/7/10 11:30:59 AM

其中now是VBScript的時間函數。

有趣嗎？讓我們趕快來看看另外一個ASP例子。

這個ASP例子，於用戶端使用瀏覽器，瀏覽執行的結果如下，顯示五個重覆的句子，字體愈來愈大。

網站伺服器的ASP程式碼如下：

<% for i = 1 to 5 %>

<FONT SIZE= <% = i %> COLOR=#0000FF>快速掌握Internet技術

　

</FONT><BR>

<% next %>

上例使用的是VBScript語言的迴圈指令for…next，循環5次。

Active Server Pages的原始程式碼，不會傳到使用者的瀏覽器，可以保護您寫出來的原始程式碼不會外漏。因此於用戶端使用瀏覽器，觀看到的是一般標準的HTML碼，您的網站伺服器會自動將Active Server Pages的程式碼，解譯為標準HTML格式的網頁內容，如下：

<FONT SIZE=1 COLOR=#0000FF>快速掌握Internet技術</FONT><BR>

　

<FONT SIZE=2 COLOR=#0000FF>快速掌握Internet技術</FONT><BR>

　

<FONT SIZE=3 COLOR=#0000FF>快速掌握Internet技術</FONT><BR>

　

<FONT SIZE=4 COLOR=#0000FF>快速掌握Internet技術</FONT><BR>

　

<FONT SIZE=5 COLOR=#0000FF>快速掌握Internet技術</FONT><BR>

傳送字串或HTML碼到用戶端的瀏覽器

Response.Write方法，是最常用到傳送字串或HTML碼到瀏覽器的方法。語法如下：

Response.Write 字串

以下的例子向使用者打招呼：

<%

Response.Write "歡迎光臨"

%>

於用戶端使用瀏覽器，瀏覽執行的結果，為 歡迎光臨。

傳送字串到用戶端的瀏覽器，不一定須要使用Response.Write，位於<%...%>外的部份將直接傳送字串或HTML碼到用戶端的瀏覽器，譬如以下的結果與上例相同：

<% .... %>

歡迎光臨

<% ... %>

使用<% =...%> 也將直接傳送字串或HTML碼到用戶端的瀏覽器，譬如以下的結果也與上例相同：

<% =”歡迎光臨” %>

連結到指定的URL位址

使用Response.Redirect 方法，可以連結到指定的URL位址。語法，如下：

Response.Redirect URL

譬如，自動連結到首頁：

<% Response.Redirect "homepage.asp" %>

須注意的一點是，若此.asp 網頁的內容已經傳送給使用者，再使用Response.Redirect，將發生錯誤！

#INCLUDE包括其他檔案

您可於ASP程式碼插入另外一個「被包括」的檔案，執行到此處時將先執行此「被包括」的檔案後，再繼續執行下一行的程式碼。使用的是「#INCLUDE」的語法，語法如下：

<!--#INCLUDE VIRTUAL|FILE="filename"-->

其中VIRTUAL代表虛擬的檔案位址，FILE代表絕對的檔案位址，如下：

<!--#INCLUDE VIRTUAL="/booksamp/test.inc"-->

<!--#INCLUDE FILE="test/test.inc"-->

附加檔案名稱建議使用.inc，但是也可以使用任何附加檔案名稱。

您可使用FILE和「../ 」代表上一層的檔案目錄。

「#INCLUDE」的語法，適用使用於多個網頁共同使用的部份，有點像副程式，供多個ASP程式碼呼叫使用。

讓我們看一個#INCLUDE的例子，於用戶端使用瀏覽器，瀏覽basic1.asp執行的結果，顯示所「被包括」的檔案copyrgt.asp。

網站伺服器的ASP程式碼如下：

<!--#INCLUDE FILE="copyrgt.asp"-->

copyrgt.asp檔案的ASP程式碼如下，只有幾行：

<B>

<FONT SIZE=1 COLOR=BLUE FACE="Verdana, Arial, Helvetica">

&copy; 1998 ABC Company All rights reserved.

</FONT>

</B>

「被包括」的HTML檔案，可以包括任何HTML標記，譬如圖片與超連結。

「被包括」的#INCLUDE 檔案，也可再包括其他「被包括」的#INCLUDE 檔案。

好玩嗎？

ASP取得表單（form）輸入資料的方法

將輸入的資料傳送到伺服器的方法，係使用HTML標記<FORM>…</FORM>的「表單」（form），於用戶端瀏覽器輸入資料，按下「SUBMIT」按鈕後，將輸入的資料傳送到伺服器加予處理。由表單（form）的METHOD得知，用戶端瀏覽器可以用下面兩種方法，將輸入的資料傳給網站的伺服器：

  1. GET：用戶端將資料附加到 URL 後，格式為「?欄位1=輸入資料1&欄位2=輸入資料2&... 」，再將它送到伺服器。譬如ACTION設為http://www.abc.com，欄位Name的輸入資料為Jack，欄位Age的輸入資料為15，則使用GET方法，傳給網站伺服器資料為http://www.abc.com?Name=Jack&Age=15。

  2. POST：用戶端利用HTTP訊息資料欄位將資料傳送到伺服器。

ASP取得表單（form）輸入資料的方法，係使用Active Server Pages一個內建的「物件」（object），叫做Request，依據GET、POST方法而異：

GET：

使用「輸入資料= Request.QueryString("欄位名稱")」或「Request. ("欄位名稱")」，將附加於URL後的資料取出，以取得此欄的輸入資料。

　

POST：

使用「輸入資料=Request.Form("欄位名稱")」或「Request. ("欄位名稱")」，讀取HTTP 訊息資料欄位，以取得此欄的輸入資料。

無論GET或 POST方式，都可以使用「Request. ("欄位名稱")」取得此欄的輸入資料。

Cookies存取用戶端檔案

為了預防病毒於Internet上到處漫延，WWW禁止於用戶端的硬碟上做讀寫存取檔案的動作。但是使用Cookies，可讓您於用戶端的硬碟上儲存一些資訊，譬如當來訪者輸入其姓名等資料時，使用Cookies將之儲存於來訪者用戶端的硬碟上，下次這位來訪者再度瀏覽此網站時，即可讀取此Cookies直接得知來訪者的身份。

ASP存取讀寫Cookies的方式，係使用Active Server Pages兩個內建的「物件」（object），叫做Response和Request，十分簡單，如下：

寫入Cookies：

Response.Cookies("待寫入的Cookies名稱") = "待寫入資料"

　

讀取Cookies：

讀取資料 = Request.Cookies("待讀取的Cookies名稱")

Microsoft Script Debugger

IIS 4.0所包括的Microsoft Script Debugger（Script偵錯工具），提供Script程式的偵錯功能。您可以使用 Microsoft Script偵錯工具來進行使用VBScript、JScript編寫的 Script程式，以及Java applets、beans 和 ActiveX 元件的偵錯工作。

有的Script程式在用戶端瀏覽器執行，有的Script程式（<%…%>中的部份）在伺服器端執行。

Microsoft Script Debugger，可以偵錯用戶端執行的Script 程式，和伺服器端執行Script程式：

  • 在用戶端瀏覽器執行的Script程式碼：於用戶端的瀏覽器當中執行，包括於標準HTML碼中的VBScript、Jscript部份。在瀏覽器載入此HTML碼，或譬如按下按鈕觸發事件時，將執行此包括Script程式的HTML碼。用戶端瀏覽器執行的Script程式，主要用於HTML表單輸入的基本檢查等。

  • 在伺服器端執行的Script程式碼：於IIS伺服器端當中執行，包括於.asp 檔案（Active Server Page）中。先於IIS伺服器執行，執行結果產生標準的HTML碼，再傳送到用戶端瀏覽器。伺服器端執行的Script程式，主要用於多個網頁之間的連結， HTML表單輸入的處理，以及存取伺服器資料庫的資料等。

Microsoft Script Debugger提供以下的除錯功能：

  • 設定中斷點

  • 逐步追蹤Script程式。

  • 設定書籤。

  • 檢視呼叫堆疊。

  • 檢視和更改變數值。

  • 執行Script指令。

結論

趕快採用於IIS網站伺服器執行的asp碼，將您的網站「 asp化」，變成一個「動態」的網站，以增加互動性，以及享受設計迅速與維護方便的好處。若您仍然使用CGI，建議您轉移到ASP來發展。
