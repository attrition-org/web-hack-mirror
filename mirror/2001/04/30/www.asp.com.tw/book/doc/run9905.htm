Windows NT Option Pack

自從Microsoft公司推出「Windows NT Option Pack」中文版軟體，將Microsoft開發的大部份Web伺服器都隨著Windows NT免費附送後，整個Web市場的趨勢，正如筆者所感受到的，Web應用非常地熱門，Windows NT正取代UNIX成為主要的Web伺服器，ASP（Active Server Pages）也迅速取代CGI。

Windows NT Option Pack包括主要的IIS 4.0（網站伺服器），以及Index Server（中文全文檢索）、Certificate Server（SSL與安全認證）、Transaction Server（多層式Web架構）、Site Server Express（使用分析）、Message Queue Server（非同步處理）等軟體。

其中Windows NT Option Pack的IIS 4.0和Transaction Server，確立了多層式（multi-tier）Web架構平台，也促成筆者所觀察到網站的最新發展趨勢。

筆者藉著於微軟公司當講師的機會，累積超過2000人次IIS/ASP研討會的教學相長寶貴經驗，將研究心得整理出十幾本網站設計的電腦書籍，並且於喬篷科技公司累積幾十個ASP網站資料庫設計專案的實務經驗，本文將以筆者參予設計『網站熱門應用精選』軟體的實例，介紹筆者所觀察到網站的最新發展趨勢---網站10化：

從需求面歸納網站的趨勢

您是否注意到網站的最新發展趨勢呢？

讓我們從客戶、和管理的需求面，來歸納出甚麼是網站的最新發展趨勢。

客戶的需求為「最新的資訊隨手可得」，而管理的需求為「管理零負擔」。

最新的資訊隨手可得

首先從客戶需求面來看，訪客上您的網站，要得就是所謂「最新的資訊隨手可得」。既然說「最新的資訊」，「過期的資訊」就不要放在網站上。

而所謂「隨手可得」，換句話說，就是訪客可以自行「搜尋」網站的內容。

要做到「最新的資訊」和「隨手可得」，唯一可以做到的就是使用「資料庫」管理整個網站，設定每一筆資訊的有效期間，網站的內容將永遠是「最新的資訊」，並可提供分類和關鍵字的資料庫「搜尋」功能。

慢慢地，訪客更希望各人可以看到不同的內容，即所謂「個人化」的網站內容。

管理零負擔

接著從管理需求面來看，網站維護者要得就是所謂「管理零負擔」。既然說「管理零負擔」，就不要天天花許多的時間在修改網站內容，最好是「網頁內容自動更新」。

「網頁內容自動更新」，唯一可以做到的就是將網站「資料庫化」，使用「資料庫」管理整個網站，只要更新資料庫的內容，網站的內容將自動更新。

網站10化

從以上客戶、和管理的需求面，就可以歸納出甚麼是網站的最新發展趨勢，筆者稱之為「網站10化」：

1. 應用軟體「Web-based化」

2. 網站內容「動態化」

3. 網站「ASP化」

4. 網站「資料庫化」

5. 網站「元件化」

6. 網站內容「個人化」

7. 網站維護「多人化」

8. 網站「電子商務化」

9. 流程「數位神經化」

10. 網站設計「自助餐化」

 

1. 應用軟體「Web-based化」

Web-based應用軟體

將目前各種網路上的應用軟體改變為「Web-based資料庫」的應用，使用「瀏覽器」做為使用者介面，是必然的趨勢。

為什麼呢？

第一個原因是操作成本降低。以往每個使用者都必須去學習每一種應用軟體的操作方式，而Web-based資料庫應用使用「瀏覽器」做為使用者介面，對於使用者的好處是只要學會瀏覽器的操作方式，譬如看到有超連結的部份就按下去，如此可以操作各種軟體應用，大幅降低訓練的成本。

第二個原因是維護成本降低。以往每當修改一個bug或修改功能時，就須為所有使用者去更新一個新的版本，當有100個使用時，就必須到100台電腦去更新版本。Web-based資料庫應用對於設計者的好處是，新版本的軟體只要更新一次網站的軟體就可以了。

基本上，所有的網路上的應用軟體，都可以改變為Web資料庫的應用，使用「瀏覽器」做為使用者介面。

2. 網站內容「動態化」

靜態的網頁內容

您的網站好久沒有訪客光臨嗎？您的網站還是「靜態」的嗎？是否訪客早已厭倦不變的靜態網頁內容呢？

目前Internet上許多的網站，仍然提供「靜態」（static）的網頁內容。所謂「靜態」，指的就是網站的網頁內容是「固定不變」地。

當瀏覽器透過Internet的HTTP（Hypertext Transport Protocol）協定，向網站伺服器（web server）要求提供網頁的內容時，網站伺服器收到要求後，傳送原已設計好靜態的HTML的檔案資料給瀏覽器

「傳統」的網站是屬於「靜態」的，僅僅使用標準的HTML碼，頂多加上GIF89A格式的圖片，產生幾隻小貓小狗跑來跑去的動畫效果，若要更新網頁的內容，必須手動地來更新其HTML的檔案資料。

「靜態」網站的一個大缺點就是不容易維護，為了不斷更新網頁內容，必須不斷地製作新的HTML檔案資料。「傳統」的網站伺服器並沒有做甚麼事，僅僅將使用者所要求的HTML檔案，傳送到瀏覽器而已。

甚麼是「動態的網站」？

一個「動態」的網站，網站的網頁內容是「動態」地。所謂「動態」的網站，指的是：

-         「內容會變」：網頁內容會依照不同時間、不同使用者，而提供不同的網頁內容。

-         「自動更新」：網頁內容會自動更新。

-         「互動性」：網頁內容會依照使用者的需要或選擇而改變。

3. 網站「ASP化」

網站「ASP化」

「動態」的網站伺服器，必須執行一些程式，然後將執行的結果輸出成標準的HTML格式檔案，傳送到使用者的瀏覽器。

所謂將網站「ASP化」，就是於IIS或PWS網站伺服器執行ASP程式，使用程式來控制網站。ASP（Active Server Pages）為於IIS或PWS網站伺服器執行的script語言，使用VBScript或JScript來撰寫程式。

將網站「ASP化」的好處在那裡？好處如下：

-          以程式控制網站：每次內容不同。

-          個人化：因人因時內容不同。

-          互動：增加互動性。

-          設計迅速維護方便：ASP不須compile編譯，使用記事本即可編輯設計。

還在使用CGI嗎？

依照筆者的測試，ASP程式執行的效率比CGI快好幾倍。如果您還在使用CGI開發Web應用程式，趕快轉到ASP程式吧！ASP程式不僅執行較CGI快，開發速度與維護方便都比CGI好上多多！

於Windows的平台上，ASP程式取代CGI是必然的趨勢。

 

4. 網站「資料庫化」

資料庫化的網站

甚麼是「資料庫化的網站」？所謂將網站「資料庫化」，指的是使用「資料庫」管理整個網站，只要更新資料庫的內容，網站的內容將自動更新。

將網站「資料庫化」的好處在那裡？

採用資料庫，將網站「資料庫化」的好處在那裡？好處如下：

-          訪客自由搜尋：將網站的內容儲存於資料庫當中，可以搜尋網站的資料。

-          網頁自動更新：只要更新資料庫的資料，網頁內容會自動更新。

-          各種網站資料庫應用。

-          管理方便：直接使用瀏覽器管理與維護。

將網站「資料庫化」的第一個好處，就是使用「資料庫」管理整個網站，只要更新資料庫的內容，網站的內容將自動更新。

每一則「新消息」和「廣告輪播」圖片都設定其「開始日期」和「結束日期」。當使用者瀏覽首頁時，將自動比較當天日期是否介於資料庫中「開始日期」和「結束日期」之間，只將符合此條件的「新消息」和「廣告輪播」圖片顯示出來。

譬如「新消息」、「輪播廣告圖片」、「檔案下載」、「相關網站」等常見的網站功能，就可以使用「資料庫」管理整個網站的內容，自動更新網頁。

5. 網站「元件化」

網站「元件化」

所謂將網站「元件化」，指的是使用Visual Basic或Visual C++等語言來開發元件（component），再使用ASP（Active Server Pages）來呼叫所撰寫的元件。元件（component），為ActiveX DLL，副檔名為.dll。

將網站「元件化」的好處在那裡？好處如下：

-          可以重覆使用。

-          執行速度增快10~30％。

-          保護原始碼。

-          多人使用資源共享。

元件就像一個零件一般，第一個好處就是可以重覆使用。

由於元件是經過compile過的，執行速度依據筆者的測試結果，大概可以增快10~30%。

也由於元件是經過compile過的，所以可以保護您的原始碼。不像ASP（Active Server Pages），原始碼可以一覽無遺。

撰寫的元件，使用MTS（Microsoft Transaction Server）來管理，就可以享受到MTS多人使用時共享資源的功能，多人使用時效率將提高。採用Transaction Server多層式Web架構，還有一個好處，就是資料庫可與Web Server同一台PC或隨成長需要裝於另外一台PC，架構可以隨需要而成長。

6. 網站內容「個人化」

各人可以看到不同的內容

所謂「個人化」的網站內容，即各訪客可以看到不同的內容，方法為將使用者分為許多的「群組」，各群組可以看到不同的網站內容。

『網站熱門應用精選』軟體可以規劃那個分類屬於那個群組專用，或所有人都可以讀取，譬如二年三班有專用的討論區，也可以參予全校的討論區。

7. 網站維護「多人化」

各自維護各自的網頁內容

「管理零負擔」的另外一個需求，就是各部門各自維護各自的網頁內容。

以往網站內容要做更新，必須找負責網站維護的站長來做，但是常常站長忙著其他事情，讓急著發佈新消息的部門急著跳腳，現在的趨勢為各部門各維護其相關的網頁內容。

以筆者參予設計『網站熱門應用精選』軟體為例，可以規劃那個分類屬於那個群組才能修改維護，譬如負責佈告欄研討會分類的人可以自行修改此分類的資料。

「管理零負擔」還有一個需求，就是如何將客戶於網站輸入的資料做統計報表，譬如統計客戶的意見等，最好就是將客戶於網站輸入的資料與Office整合。

『網站熱門應用精選』軟體可將意見、問卷調查統計結果、活動錄取名單、訂閱者名單等，儲存成Excel格式檔案，由瀏覽器瀏覽或下載。

前面談到「個人化」的網站內容，各「群組」的成員是由誰來管理呢？各「群組」的成員最好是由各群組的管理者各自來管理。

『網站熱門應用精選』可以規劃那個群組可以「管理」那個群組的組員，譬如二年三班老師群組可以管理二年三班的學生群組。

另外，從 [檔案總管] 拖曳檔案到瀏覽器即可上傳佈告欄、資訊、廣告圖片、檔案下載等所須檔案到網站。可規劃那個分類屬於那個群組才能上傳檔案。

各群組的組員可以於網路線上申請後自動加入，也可以設定成必須經過各群組管理者的核准。

8. 網站「電子商務化」

網站加上線上購物等電子商務的功能逐漸成為網站的趨勢。

線上購物不一定需要使用信用卡付款，譬如筆者公司所設計的雅芳線上購物網站，提供給數萬名雅芳小姐網路線上訂購化妝品等商品的服務，這個網站不僅僅是接受定單而已，挑戰之處為會配合20天一期的促銷活動，譬如買三送一、滿1000元送香水等，自動計算價格和贈品，並且可以查詢業績。

9. 流程「數位神經化」

逐漸地，intranet內部網路使用流程控制管理的軟體，來大大協助內部流程的管理工作，讓公司「數位神經化」，使用個人電腦和網路做管理與決策。

流程控制管理的軟體，目前最大的問題是表單十分難以設計，安裝十分麻煩，必須東設西設，而且不是「Web-based」的應用，在瀏覽器執行都有些問題。

『網站熱門應用精選』的意見箱傳送處理功能，訪客於意見箱留下意見後，自動傳送E-mail給負責人，可於E-mail中處理和回覆。

10. 網站設計「自助餐化」

自助餐化

您是否花了許多金錢和人才找人或自己寫程設計建構一個動態的網站呢？

網站有許多常用的功能，並不需要自己重覆開發，您可以由常用的功能中挑選目前所需求的功能，只有特殊需求的部份需要自行開發。這些常用的功能就像煮好的「自助餐」一般，自由挑選喜歡吃的，只有特殊口味的才需要自己煮。

以筆者參予設計『網站熱門應用精選』軟體為例，將12個常用的網站功能，整合成一個套裝軟體，馬上即可使用，不須寫程式。安裝非常容易，不須東設西設。可依據實際需要，挑選所須的數個網站功能。

與既有網頁輕鬆整合

『網站熱門應用精選』軟體，也提供與既有網頁輕鬆整合的功能。只要於您現有的網頁當中，插入一行HTML標記，即可擁有一項新的動態網站功能。

自定外觀顏色欄位

值得一提的是，您可以自行規劃欄位名稱與顯示欄位，可以規劃那些欄位要顯示、前加圖片、顯示方式、顯示顏色、顯示欄位名稱，顯示顏色可由色盤挑選。

網站熱門應用精選

企業/校園/政府「網站熱門應用精選」2.0專業版，為筆者公司喬篷科技採用多層式Web架構和SQL Server等微軟相關Web產品與技術所開發的套裝軟體，將數十個專案開發中常用的12個網站功能，整合成一個套裝軟體，不須寫程式，於瀏覽器做設定，即可快速建構動態的網站。

使用資料庫管理「網站內容」，只要由瀏覽器更新資料庫，不用找人修改網頁，自動產生新的動態網頁內容。

並提供多人各自維護網站（修改權限）、會員專屬網站內容（讀取權限）、各自管理組員（管理權限）、自定外觀顏色欄位、檔案上載、意見箱傳送處理等新功能。
