IIS網站管理方法

四種IIS網站管理方法

IIS網站的的管理方法，有以下四種：

  1. Internet 服務管理員：您可使用IIS的「Internet 服務管理員」，來管理區域網路或 Internet 上的Web站台或FTP站台。

  2. Internet 服務管理員 （HTML）：IIS 也提供有瀏覽器適用的「Internet 服務管理員 （HTML）」，可透過瀏覽器管理Intranet 或 Internet 上的站台。

  3. ASP（Active Server Pages）：使用ASP程式來呼叫「IIS Admin Objects」，可用來透過瀏覽器，來管理IIS網站。

  4. WSH（Windows Scripting Host）：從 Windows 下或DOS指令行下執行一個.vbs的檔案，來管理IIS網站。WSH可以呼叫「IIS Admin Objects」來管理IIS網站。

站台的操作員可以使用瀏覽器，於伺服器的電腦、或於遠端，對Web站台或FTP站台，做新增、刪除、重新命名、啟動、停止、暫停、繼續等控制，以及對內容做設定，和做備份設定等動作。

　

遠端管理站台

Windows NT 管理員或Web 站台操作員，可以透過Internet或Intranet，使用瀏覽器於遠端來管理站台。即遠端存取「Internet 服務管理員 （HTML）」。

方法為Windows NT 管理員或Web 站台操作員，透過Internet或Intranet，啟動瀏覽器，並輸入Web 站台的網域名稱及指定的連接埠編號<port number>，並於其後加上 /iisadmin/。例如，http://www.asp.com.tw:<port number>/iisadmin/。一旦連上站台，將要求殊輸入使用者名稱與密碼，才可以繼續登入。

Windows NT 管理員須指定連接埠編號<port number>，連接埠編號可在 [系統管理 Web 站台] 中[內容找到。

當安裝 IIS 時，將指定給這個 Web 站台一個隨機選取 2,000 到 9,999 之間的連接埠編號。

　

IIS Admin Objects

使用IIS Admin Objects來管理IIS網站，可以做到的功能譬如：

  • 啟動、暫停、繼續、或停止一個Web站台、FTP站台、伺服器。

  • 建立站台、建立虛擬目錄。

  • 尋找站台。

改變站台或虛擬目錄的屬性設定。譬如設定存取權限、設定最大頻寬限制等。

您可以使用ASP（Active Server Pages）、或WSH（Windows Scripting Host），來呼叫「IIS Admin Objects」，以管理IIS網站。

另外，使用CreateObject("MTSAdmin.Catalog.1") 與MTS（Microsoft Transaction Server）溝通，可以做到的功能譬如：

  • 安裝和解除安裝元件（DLL）。

  • 安裝和解除安裝套件。

　

Metabase

IIS 4.0之管理IIS 的設定值，儲存於一個叫做metabase的階層式資料庫。您可在電腦、Web 站台、虛擬目錄、目錄及檔案等層次，藉由修改 metabase的值來改變伺服器的設定。

直接修改 metabase的值是十分危險的，若將metabase 中的內容設定錯誤，可能造成Web 站台或 FTP 站台發生問題，甚至可能損毀Web 站台或 FTP 站台的設定。您應在無法調整使用者介面下才編輯 metabase 內容，且當您直接編輯 metabase 時需非常小心。

　

Windows Scripting Host

IIS提供於DOS指令行下或Windows 下，使用VBScript及 JavaScript 指令設定IIS伺服器的方式，叫做Windows Scripting Host（WSH），將設定值儲存於Metabase當中。

Windows Scripting Host ，是適用於32 位元 Windows 平台的指令主機。Windows Scripting Host ，提供VBScript及 JavaScript 指令的執行引擎。

Windows 作業系統，於MS-DOS 指令行下所能支援的scripting語言，以前只提供批次檔。現在，Windows Scripting Host 讓您可以於MS-DOS 指令行下，執行威力強大的VBScript 及 JScript 指令碼。

使用Windows Scripting Host 設定IIS時，可以從 Windows 下執行或DOS指令行下執行。

當您準備好設定IIS的VBScript 或 JScript 指令碼後，使用 .vbs的 副檔名。直接於 [檔案總管] 中按兩下即可執行此 .vbs 檔，將自動啟動wscript.exe以執行之。您也可以從DOS指令行下，使用cscript.exe來執行.vbs 檔的VBScript 或 JScript 指令碼。

Windows Scripting Host（WSH）提供以下數個物件：

  • WScript.Network：與網路溝通。

  • WScript.Shell ：與Shell溝通。

  • Excel.Application ：與Excel溝通。

WScript.Network 用來與網路溝通，可以做到的功能譬如：

  • 取得網路的資訊。

WScript.Shell 用來與Shell溝通，可以做到的功能譬如：

  • 於桌面上建立捷徑。

  • 建立及刪除登錄機碼（registry）。

Excel.Application 用來與Excel溝通，可以做到的功能譬如：

  • 在Excel中顯示WSH內容。

  • 用WSH存取Excel顯示圖表。

　

DOS指令行WSH語法

首先，請確定安裝Windows NT Option Pack中的Windows Scripting Host。

譬如，若要執行test.vbs，於 [開始] 按鈕的 [程式集] 中執行 [命令提示字元]（MS-DOS 提示），於DOS指令行下執行下列指令：

C:\> cscript test.vbs

cscript.exe程式，使用於DOS指令行下，執行.vbs 檔語法如下：

cscript 指令檔名稱 [/指令參數...] [//主機參數...]

  • 指令檔名稱：VBScript 或 JScript 指令碼檔案名稱，例如test.vbs。

  • 指令參數：傳遞給指令檔的參數，指令參數前面有一個斜線 （/）記號。

  • 主機參數：Windows Scripting Host 選項，主機參數前面有兩個斜線 （//）記號。

Cscript.exe 的主機參數如下：

  • //I：互動模式 （預設值），與//B相反，在螢幕上顯示指令錯誤及使用者提示。

  • //B：批次模式，與//I相反，在螢幕上不顯示指令錯誤及使用者提示。

  • //T:nn：指令可執行的最多時間 （單位：秒）。當執行時間超過所設定值時，CSCRIPT 會用中斷處理。

  • //logo： 在執行時顯示執行標題 （預設值），與//nologo 相反。

  • //nologo： 在執行時不顯示執行標題，與//logo 相反。

  • //H:Cscript 或 Wscript： 讓 Cscript.exe 或 Wscript.exe設定為執行指令的預設程式。預設值為Wscript.exe 。

  • //S：儲存目前指令行選項。

  • //?：顯示指令用法。

本文將舉數個範例，讓您迅速學會於DOS指令行下：

  • 讀取和設定IIS，學會Windows Scripting Host 基本指令。

  • 設定存取權限。

  • 設定最大頻寬限制。

  • 讀取各種屬性（properties）值。

  • 使用各種方法（methods），讓您可以於DOS指令行下啟動、繼續、暫停、和停止一個站台。

  • 建立虛擬目錄。

趕快開始吧！

　

IIS Admin Objects管理IIS網站

本單元介紹如何使用IIS Admin Objects來管理IIS網站，可以透過瀏覽器使用ASP、或於MS-DOS 指令行和於Windows環境下使用WSH，呼叫IIS Admin Objects，來控制管理一個網站。譬如：啟動、暫停、繼續、或停止一個Web站台、FTP站台、伺服器，或建立一個新的站台、建立一個新的虛擬目錄、尋找站台，以及改變站台或虛擬目錄的屬性設定，譬如設定存取權限、設定最大頻寬限制等。

　

　

IIS Admin Objects

首先先定義待讀取或設定的電腦、Web 站台、虛擬目錄、目錄或檔案等路徑，語法如下，其中IisObj為定義的路徑物件名稱：

Set IIsObj = GetObject("IIS://電腦名稱/服務/站台編號/目錄")

其中路徑「IIS://電腦名稱/服務/站台編號/目錄」，語法如下：

  • 電腦名稱：為IIS伺服器的電腦名稱，若為本機可以LocalHost代表。

  • 服務：Web站台為W3SVC，FTP站台為MSFTPSVC，

  • 站台編號：1代表第一個站台。

  • 目錄：虛擬目錄，Root表示根目錄。

譬如：

  • IIS://LocalHost：表示為整個IIS伺服器。

  • IIS://LocalHost/W3SVC/1：表示「預設的 Web 站台」。

  • IIS://LocalHost/W3SVC/1/Root：表示「預設的 Web 站台」的主目錄。

  • IIS://LocalHost/MSFTPSVC/1：表示為「預設的 FTP站台」。

  • IIS://LocalHost/SMTPSVC/1：表示為「Default SMTP Site」。

  • IIS://LocalHost/NNTPSVC/1：表示為「Default NNTP Site」。

定義位址後即可以使用「GET」指令讀取屬性（properties）值，語法如下，其中IIsObj為上面所定義的位址物件名稱：

xxx = IIsObj.Get("屬性名稱")

譬如「a = IIsObj.Get("AccessRead")」，表示讀取AccessRead 屬性的設定值，儲存到a的變數當中。

若要設定IIS，可以使用「PUT」指令設定屬性（properties）值，最後再使用SetInfo方法儲存到Metabase當中。語法如下，其中IIsObj為上面所定義的位址物件名稱：

IIsObj.Put "屬性名稱", 設定值

IIsObj.SetInfo

譬如「IIsObj.Put "AccessRead", False」，表示設定AccessRead 屬性的設定值為False。

以下整體的.vbs碼，表示先設定AccessRead 屬性的設定值為False，再讀取AccessRead 屬性的設定值，儲存到gRead的變數當中。如下：

Set IIsObj = GetObject("IIS://LocalHost/W3SVC/1/Root")

IIsObj.Put "AccessRead", True

IIsObj.SetInfo

gRead = IIsObj.Get("AccessRead")

WScript.Echo gRead

其中「WScript.Echo gRead」，表示將所讀取AccessRead 屬性的值gRead，顯示於DOS指令行下的畫面。

若我們將以上五行的碼使用記事本儲存成admobj1.vbs檔案，並於DOS指令行下執行下列指令：

C:\> cscript admobj1.vbs

於DOS指令行下，將顯示：

C:\> cscript admobj0.vbs

Microsoft (R) Windows Scripting Host Version 5.0 for Windows

Copyright (C) Microsoft Corporation 1996-1997. All rights reserved.

-1

其中-1表示True（0表示False）。

除了使用「GET」和「PUT」指令以讀取和設定屬性（properties）值外，您也可以使用「.」以讀取和設定屬性（properties）值。

譬如「IIsObj.AccessRead」表示讀取AccessRead屬性值，「IIsObj.AccessRead = True」表示設定AccessRead屬性值為True。以下的碼與上例結果相同：

Set IIsObj = GetObject("IIS://LocalHost/W3SVC/1/Root")

IIsObj.AccessRead = True

IIsObj.SetInfo

gRead = IIsObj.AccessRead

WScript.Echo gRead

最大頻寬限制設定範例

如何透過瀏覽器使用ASP、或於MS-DOS 指令行和於Windows環境下使用WSH，呼叫IIS Admin Objects，來設定最大頻寬限制呢？

方法為使用「GET」和「PUT」指令，以讀取和設定最大頻寬限制「MaxBandwidth」的屬性值。

看個例子就知道最大頻寬限制如何設定了！

於DOS指令行下執行cscript admobj3.vbs，將顯示：

c:\asp>cscript admobj3.vbs

Microsoft (R) Windows Scripting Host Version 5.0 for Windows

Copyright (C) Microsoft Corporation 1996-1997. All rights reserved.

-1

204800

以上將整個IIS伺服器的最大頻寬，限制為204800 bits（200 KBytes）。

-1表示原本未限制最大頻寬。

設定後，若要確認設定是否已經改變，方法為在 [Internet 服務管理員] 上，在IIS伺服器的電腦上按一下滑鼠右鍵，再選取 [內容]，可看到已經啟用頻寬節流設定，並將最大網路限制設定為200 KBytes。

讓我們看看如何做到，使用記事本開啟admobj3.vbs，內容如下：

‘首先定義位址物件，「IIS://LocalHost」，表示為整個IIS伺服器。

Set IIsObj = GetObject("IIS://LocalHost")

‘使用「GET」以讀取最大頻寬限制「MaxBandwidth」的屬性值。

MaxBW = IIsObj.Get("MaxBandwidth")

‘或使用「.」以讀取最大頻寬限制「MaxBandwidth」的屬性值。

'MaxBW = IIsObj.MaxBandwidth

‘用「WScript.Echo …」，將讀取的顯示於DOS指令行下的畫面。因此顯示-1表示原本未限制最大頻寬。

WScript.Echo MaxBW

'接著使用「PUT」指令以設定最大頻寬屬性值，設定MaxBandwidth為200 KBytes。

IIsObj.Put "MaxBandwidth", 200 * 1024

'最後再使用SetInfo方法儲存到Metabase當中。

IIsObj.SetInfo

'再使用「.」以讀取剛剛寫入的MaxBandwidth屬性設定值並顯示出。

MaxBW = IIsObj.MaxBandwidth

WScript.Echo MaxBW

簡單嗎？

啟動、暫停、繼續、或停止Web站台範例

如何透過瀏覽器使用ASP、或於MS-DOS 指令行和於Windows環境下使用WSH，呼叫IIS Admin Objects，來啟動、暫停、繼續、或停止一個Web站台、FTP站台、或伺服器呢？

為達到這個功能，您必須使用IIS Admin Objects所提供的方法（methods），如下：

  • Continue：繼續一個Web站台、FTP站台、或伺服器。

  • Pause ：暫停一個Web站台、FTP站台、或伺服器。

  • Start：啟動一個Web站台、FTP站台、或伺服器。

  • Stop ：停止一個Web站台、FTP站台、或伺服器。

首先先定義待設定的Web站台、FTP站台、或伺服器等路徑，語法如下，其中IisObj為定義的路徑物件名稱：

Set IIsObj = GetObject("IIS://電腦名稱/服務/站台編號")

譬如：

  • IIS://LocalHost：表示為整個IIS伺服器。

  • IIS://LocalHost/W3SVC/1：表示「預設的 Web 站台」。

  • IIS://LocalHost/MSFTPSVC/1：表示為「預設的 FTP站台」。

看個例子就知道如何使用方法（methods）了！

讓我們看看如何啟動一個Web站台，使用記事本開啟startweb.vbs，內容如下：

'「 IIS://LocalHost/W3SVC/1」，表示為預設的Web站台。

fullPath = " IIS://LocalHost/W3SVC/1”

On Error Resume Next

‘開啟此路徑。

Set objServer = GetObject(fullPath)

If Err <> 0 Then

'若Err <> 0表示錯誤則跳出

WScript.Echo Now & ". 錯誤碼: " & Hex(Err) & " - " & "無法開啟 " & fullPath & "."

WScript.Quit

End If

'使用Start啟動(開始)站台。

objServer.Start

If Err <> 0 Then

WScript.Echo Now & ". 錯誤碼: " & Hex(Err) & " - " & "無法啟動(開始)Web站台 " & fullPath & "."

else

WScript.Echo "已經啟動(開始)Web站台 " & fullPath & "."

End If

程式當中，首先使用Start方法（methods），啟動預設的Web站台（IIS://LocalHost/W3svc/1）。

若直接於Windows 環境下執行startweb.vbs，將顯示「已經啟動(開始)Web站台 IIS://LocalHost/W3svc/1.」的對話盒。

或於DOS指令行下執行cscript startweb.vbs，將顯示：

c:\ntopsamp> cscript startweb.vbs

Microsoft (R) Windows Scripting Host Version 5.0 for Windows

Copyright (C) Microsoft Corporation 1996-1997. All rights reserved.

已經啟動(開始)Web站台 IIS://LocalHost/W3svc/1.

我們啟動了預設的 Web 站台。

同理，將以上程式中的「objServer.Start」，若改變為「objServer.Pause」即可暫停一個Web站台；改變為「objServer.Continue」可繼續一個Web站台；改變為「objServer.Stop 」則可以停止一個Web站台。

　

啟動、暫停、繼續、或停止FTP站台範例

將以上程式中路徑更改為「msftpsvc」，就可以控制FTP站台了！

讓我們看看如何停止一個FTP站台，使用記事本開啟stopftp.vbs，內容如下：

'「 IIS://LocalHost/msftpsvc/1」，表示為預設的FTP站台。

fullPath = "IIS://" & ComputerName & "/msftpsvc/" & ServerNo

'開啟此路徑。

Set objServer = GetObject(fullPath)

If Err <> 0 Then

WScript.Echo Now & ". 錯誤碼: " & Hex(Err) & " - " & "無法開啟 " & fullPath & "."

End If

'使用Stop停止站台

objServer.Stop

If Err <> 0 Then

WScript.Echo Now & ". 錯誤碼: " & Hex(Err) & " - " & "無法停止FTP站台 " & fullPath & "."

else

程式當中，首先使用Stop方法（methods），停止預設的FTP站台（IIS://LocalHost/ msftpsvc /1）。

同理，將以上程式中的「objServer.Stop」，若改變為「objServer.Pause」即可暫停一個FTP站台；改變為「objServer.Continue」可繼續一個FTP站台；改變為「objServer.Start 」則可以啟動一個FTP站台。

　

建立虛擬目錄範例

如何透過瀏覽器使用ASP、或於MS-DOS 指令行和於Windows環境下使用WSH，呼叫IIS Admin Objects，來建立虛擬目錄呢？

方法為使用「Create("IISWebVirtualDir","虛擬目錄名稱")」方法，以建立虛擬目錄，並設定以下屬性：

  • Path：設定虛擬目錄的實際物理路徑。

  • EnableDirBrowsing：設定虛擬目錄是否允許瀏覽目錄。

  • AccessRead：設定虛擬目錄是否允許讀取。

看個例子就知道建立虛擬目錄如何設定了！

於DOS指令行下執行cscript admobj6.vbs，將顯示：

admobj6.vbs於DOS指令行下執行時將顯示：

c:\asp>cscript admobj6.vbs

Microsoft (R) Windows Scripting Host Version 5.0 for Windows

Copyright (C) Microsoft Corporation 1996-1997. All rights reserved.

讓我們看看如何做到，使用記事本開啟admobj6.vbs，內容如下：

'首先定義位址物件，「 IIS://LocalHost/W3SVC/1/Root」，表示於預設的 Web 站台的主目錄下建立虛擬目錄。

Set ServiceObj = GetObject("IIS://LocalHost/W3SVC/1/Root")

'使用「Create("IISWebVirtualDir","虛擬目錄名稱")」方法，以建立虛擬目錄asp5。

Set newVirDir =ServiceObj.Create("IISWebVirtualDir","asp5")

'由Path屬性設定虛擬目錄的實際物理路徑c:\temp。

newVirDir.Path = "c:\temp"

'由EnableDirBrowsing屬性設定虛擬目錄是否允許瀏覽目錄。

newVirDir.EnableDirBrowsing = True

'由AccessRead屬性設定虛擬目錄是否允許讀取。

newVirDir.AccessRead = True

'最後再使用SetInfo方法儲存到Metabase當中。

newVirDir.SetInfo

設定後，若要確認設定是否已經改變，方法為在 [Internet 服務管理員] 上，在於預設的 Web 站台下可看到建立虛擬目錄asp5。再按滑鼠左鍵選取 [內容]，可看到已經允許讀取，以及允許瀏覽目錄。

安裝MTS套件

透過瀏覽器使用ASP、或於MS-DOS 指令行和於Windows環境下使用WSH，使用CreateObject("MTSAdmin.Catalog.1") 用來與MTS（Microsoft Transaction Server）溝通，可以做到的譬如安裝和解除安裝元件（DLL）、安裝和解除安裝套件等功能。

讓我們看看如何安裝一個套件，使用記事本開啟instpak.asp，內容如下：

'套件檔案

pakPath = "d:\jack.pak"

'套件名稱

packageName = "jack"

'首先產生catalog object

Dim catalog

Set catalog = CreateObject("MTSAdmin.Catalog.1")

'然後取得packages collection

Dim packages

Set packages = catalog.GetCollection("Packages")

'重新整理

packages.populate

'使用Remove刪除已經安裝的相同名稱套件

For i = packages.Count - 1 To 0 Step -1

If packages.Item(i).Value("Name") = packageName Then

packages.Remove (i)

End If

Next

'儲存所做的刪除動作

packages.SaveChanges

'使用InstallPackage從一個.pak檔案安裝套件

Dim util

Set util = packages.getUtilInterface

util.InstallPackage pakPath, "", 0

若直接於Windows 環境下執行instpak.asp，或於DOS指令行下執行cscript instpak.asp，將於在Transaction Server Explorer中顯示所安裝的套件。
