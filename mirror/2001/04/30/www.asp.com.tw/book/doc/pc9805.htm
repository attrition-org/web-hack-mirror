建構IIS網站

計算網站所須頻寬

建構一個IIS網站，首先讓我們先計算網站所須頻寬。

您所建構的Internet網站，需要多少的頻寬（每秒的連線速度）才夠呢？64 Kbps？128 Kbps？512 Kbps？或T1的1,54 Mbps？

您的Internet 網站所需的連線頻寬，決定於：

  • 使用者所要求傳送的資料大小，即網頁檔案的大小。

  • 可以同時服務的人數，即同時有幾個人上網。

  • 使用者願意等候接收檔案的時間。

若沒有足夠的頻寬以供應進來網站的人數使用，將發生嚴重的滯延或失敗。

一但決定Web或 FTP伺服器所需的頻寬，您也必須考慮到其它服務所吃掉的頻寬。這些服務包括電子郵件（e-mail）、新聞（news），或即時傳送的聲音影像資料流。您必須提供足夠的頻寬，以供Web、 FTP、和其它各種服務使用。

願意等候接收檔案的時間

您必須考慮傳送網站網頁檔案至使用者的速度，檔案傳送的快慢決定於連線的速度及傳送檔案的大小。

對於使用者願意等候接收檔案的時間而言，一般準則為傳送一頁的文字不超過「五秒鐘」，之後再傳送非文字的其他檔案（包括圖形、聲音、及視訊等）部份，非文字部份不應超過「30 秒鐘」。

換句話說，當使用者使用瀏覽器光臨貴寶號的網站時，最好於「五秒鐘」內於瀏覽器顯示出網頁的文字部份，其他的圖形等部份於瀏覽器等候的時間不要超過「30 秒鐘」，否則可能遭受使用者因沒有耐心而放棄。

每天可以提供的連線數目

讓我們計算每天可以提供的連線（可被查閱）數目。

以T1連線而言，每秒可以傳輸125.3 KBytes，計算方式為1,540,000 bps 連線速度除以12 個位元（因每傳送8個位元的資料，就會浪費處理4個位元的時間），求出每秒傳輸 125.3 KBytes。

每天可以傳輸10,825,920 KBytes（125.3 Kbytes * 60 * 60 * 24）。

假設網頁（文字加圖形）的平均檔案大小為 25 KBytes，則每天可被查閱人數為 433,037（10,825,920 KBytes / 25 KBytes）。

64K專線夠用嗎？

若您租用64K專線，若一頁網頁（文字加圖形）的平均檔案大小為 5 KBytes，等待的時間為5秒鐘，則每秒傳送速度為5 KBytes，每秒傳送網頁數為1頁，可以同時上線的使用者人數為5人，每天可查閱人數為92,160。

天啊！一個網頁大小為 5 Kbytes時，64K專線可以同時上線的使用者人數為5人，一個網頁隨便加個幾張各數Kbytes的圖形，譬如總計30KBytes，即使只讓一個人連線，由於每秒傳送速度為5 KBytes，等待的時間也須6秒鐘！

那麼，您想想每月花費兩萬多租用的64K專線，夠用嗎？

當然，當使用者再次光臨您網站時，瀏覽器可使用前一次暫存的圖形檔案，真的幫了不少頻寬的忙。

建構IIS網站可以採用的連線方式

若您想要建構一個IIS的網站，以便執行IIS的ASP程式和資料庫應用程式，可以採用的連線方式，如下，前三種為Internet網站，第四種為Intranet企業內部網站：

  • 租用專線：自己添購和維護伺服器電腦（安裝Windows NT Server和IIS）和路由器等設備，拉一條專線。

  • 虛擬主機：租用ISP（Internet 服務提供者）主機的硬碟空間，須找「可執行ASP程式和資料庫」的IIS虛擬主機，以及較大的硬碟空間，以裝日益膨脹的資料庫。

  • 主機代管（co-location）：自己準備一台Windows NT Server和安裝IIS的伺服器電腦，放置於ISP的機房。

  • 區域網路：準備一台Windows NT Server和安裝IIS的伺服器電腦，連接到區域網路，做為Intranet的企業內網站。

租用專線

建構一個Internet網站的傳統方式，為租用專線，向中華電信租用一條專線，由您的網站伺服器連接到ISP的機房，以連線上Internet。

您的網站伺服器透過網路卡、路由器（router）等裝置與 Internet 連線。接線的方式，為專線接到DSU裝置後，接到路由器，再接到區域網路的HUB，網站伺服器透過網路卡接到HUB。

Internet專線費用，包括Internet連線費用與中華電信的專線電路費，譬如64 Kbps專線，以Hinet而言，月租費為每個月兩萬多，128 Kbps專線為五萬多，T1專線為35萬多。

虛擬主機

除了「租用專線」方式外，另外一個方法為採用「虛擬主機」方式，租用ISP主機的硬碟空間。

「租用專線」方式必須自己添購和維護設備，設備包括伺服器電腦和路由器等，約須十幾萬。「虛擬主機」方式，由於運用ISP的設備，可節省設備費用與維護的人力。

「虛擬主機」方式，費用一般為每個月數千元，譬如Hinet每月4,400（15MB），其他ISP約每月2,000~4,000。須注意的是，是否對頻寬有所限制，許多ISP將「虛擬主機」的頻寬限制為64K或128K，有的不限制但是也不保證。

大部份ISP提供的「虛擬主機」，由於大都是UNIX主機，只能執行使用Perl寫的CGI程式，都不能執行IIS的ASP程式和存取網站資料庫的網站應用程式。

國內有些ISP，提供「可執行IIS的ASP程式和資料庫」的IIS虛擬主機，有50MB的硬碟空間，解決了不能執行IIS的ASP程式和存取網站資料庫的問題，費用為每個月數千元，詳情請上網www.asp.com.tw查詢。

主機代管（co-location）

除了「租用專線」、「虛擬主機」方式外，國外流行另外一個方法，叫做「主機代管」（co-location）的方式，自己準備一台伺服器電腦，放置於ISP的機房，向ISP租用放置空間，與其大頻寬的網路連線。

好處為可分享ISP的大頻寬，而費用較便宜。國內的ISP有數家提供「主機代管」（co-location）的服務，費用約為每月一萬多，詳情請上網www.asp.com.tw查詢。

「主機代管」（co-location）方式，也解決執行IIS的ASP程式和存取網站資料庫的問題。

建立Web站台

若要快速建立一個Web站台，您可以使用 IIS 內含的 [預設 Web 站台] 來建立站台。[預設 Web 站台] 可於 [Internet 服務管理員] 找到。

首先建立 Web 站台的首頁，將首頁的檔名命名為 default.htm。再將您的首頁檔案複製到 IIS 的預設Web站台主目錄\Inetpub\Wwwroot下。

如果您的網路有名稱解析系統，則訪客只要在瀏覽器的位址列輸入http:// 加上您電腦的名稱，譬如http://nt，便可以連線到您的Web站台，瀏覽首頁的資訊。如果您的網路沒有名稱解析系統，則訪客必須輸入您電腦的IP 位址。

如果您想使用其它檔名做為首頁，您必須啟動 [Internet 服務管理員] ，選擇 [預設 Web 站台] ，按滑鼠右鍵選擇 [內容]，然後選擇 [文件] 頁，按下 [新增] 按鈕以新增一個新的首頁檔名，並將此新增首頁檔名移到 [啟用預設文件] 的預端。

如此一來，當訪客在瀏覽器的位址列輸入您電腦的名稱或IP 位址時，便可以瀏覽首頁index.htm檔。

建立FTP站台

若要快速建立一個FTP站台，您可以使用 IIS 內含的 [預設FTP 站台] 來建立站台。[預設FTP 站台] 可於 [Internet 服務管理員] 找到。

將您的檔案複製或移動到 [預設 FTP 站台] 的主目錄\Inetpub\Ftproot下。

　

管理IIS網站

整合的管理介面

IIS 4.0使用一個單一整合的Microsoft Management Console（管理控制台，簡稱MMC），來管理Internet Service（WWW、FTP、NNTP、SMTP）、Microsoft Transaction Server、和Microsoft Index Server。

MMC使用一個相同的整合控制介面畫面，即可做所有的管理，可以節省用戶學習多個管理介面的時間。

MMC並且整合Windows NT的一些管理工具，包括識別碼管理員（Key Manager）、伺服器管理員（Service Manager）、用戶管理員（User Manager）、事件檢視器（Event Viewer）、效能監視器（Performance Monitor）、和效能監視器（Performance Monitor）等。

數種管理方式

IIS 4.0提供數種管理網站伺服器的方式，包括：

  • Internet服務管理員：使用上述單一整合的Microsoft Management Console（MMC）控制介面來管理。

  • Internet服務管理員（HTML）：使用瀏覽器，透過Internet或Intranet，於遠端來管理WWW、FTP站台。

  • Windows Scripting Host：可以使用DOS指令行的方式，來管理WWW、FTP，可以產生、啟動、停止、暫停WWW或FTP網站，產生、移除虛擬目錄，控制目錄存取的設定，以及顯示搜查設定的樹狀結構等。

頻寬限制

管理者可以將放置於一台伺服器的每一個網站，做分配頻寬（allocate bandwidth）的動作，限制一個網站最多的頻寬，以保證所有網站都可以享有頻寬，不會被一個熱門的網站吃掉所有的頻寬。

各自管理網站

放置於同一台伺服器的每一個網站之管理者，可以各自管理其網站，而不互相影響。譬如每一個網站之管理者，可以設定其網站的存取權限、啟動記錄、改變預設文件表尾、設定內容有效期間、設定內容分級等。每一個網站之管理者，並沒有改變IIS設定、Windows NT Server設定、或網路設定的權限。譬如每一個網站之管理者無權設定匿名用戶名稱密碼、設定頻寬限制、改變網路設定、伺服器用戶設定等。

效能監視

IIS讓管理者能夠針對一個網站伺服器中各自的網站，做效能的監視。這個表示管理者能夠得知那個網站使用最大的頻寬。

使用Windows NT的效能監視器（Performance Monitor），可以監視網站伺服器的效能。效能監視器使用計數器（counters），以監視每個時段的檔案傳輸率、頻寬使用率、連線率等。

詳細的記錄

IIS 4.0提供兩種新的記錄（log）方式，如下：

  • Customized Extended Logging：使用W3C的Extended Logging標準格式。這個格式是一個可以客戶化的ASCII格式，讓管理者可以移除不必要的欄位，以限制記錄（log）的檔案大小。這些記錄（log）的欄位約有20個，包括日期、時間、IP Address、和瀏覽器種類等。

  • Resource Logging：大部份IIS 4.0的記錄（log），可以設定到每個檔案的方式。管理者可以選擇那些資源需要做記錄，減少記錄（log）的檔案大小，以增進執行的效率和容易分析記錄。譬如您可以將所有的影像圖片儲存到一個目錄下，並且選擇此目錄不要做記錄（log）。

用戶端認證

使用用戶端認證（Client Certificate），讓使用者自動登入伺服器。

IIS 4.0的Certificate Server，使您公司能夠自行建立一個認證中心（Certificate Authority），自己核發用戶端認證給所有用戶，讓用戶安全地存取伺服器的資訊。

IIS 4.0並且提供用戶端認證與Windows NT user account的對照表，以用戶端認證方式自動登入伺服器的用戶，依照此對照表，即享有所對應到user account的權限。

此對照表有兩種對照方式，包括一個用戶端認證對照到一個user account的一對一方式，以及多個用戶端認證對照到一個user account的多對一方式。

多對一的對照方式，管理者可以定義對照原則，譬如由定義某個認證中心所核發的所有用戶端認證，都對照到同一個user account，而非一個對一個。

SSL安全協定

使用SSL（Secure Sockets Layer）安全協定的方式，用戶與伺服器可以建立安全的連線，傳送前先將資料加密，以於網路上傳送加過密的資料，接收時再將資料解密以獲得原來的資料。

IIS 4.0支援SSL 2.0和3.0的標準規格。

使用SSL前，伺服器必須先取得伺服器認證（Server Certificate）。於SSL連線時，伺服器將其伺服器認證傳送給用戶，讓用戶驗證伺服器的身份。

伺服器也可以要求用戶傳送其用戶端認證，讓伺服器驗證用戶的身份，但是並不是需要如此。

如此建立SSL連線後，用戶與伺服器即可以安全地傳送資料。

Microsoft Site Server Express

Windows NT Option Pack所包括的Microsoft Site Server Express，為Microsoft Site Server的精簡版本，不包括Microsoft Site Server所提供電子商務的功能。

Microsoft Site Server Express提供：

  • 用法分析（usage analysis）的功能。

  • 站台分析（content analysis）的功能。

  • 發行（publish）內容的功能。

每一次使用者登入網站時，就會於log記錄一行的資料，這些資料不經過分析不易讀取。

Microsoft Site Server Express的用法分析（usage analysis）功能，透過Usage Import和Report Writer的模組，分析log記錄的資料，統計使用的資訊。

於一個組織內，由於有愈來愈多的單位提供同一個網站內網頁的內容，讓網站變得複雜而不易管理。Microsoft Site Server Express的站台分析（content analysis）功能，提供網站內所有網頁內容的分析，包括超連結是否存在、物件是否存在等。

Microsoft Index Server

Windows NT Option Pack所包括的Microsoft Index Server，提供中文全文檢索的功能。

用戶使用瀏覽器，輸入關鍵字，即可以搜尋網站的內容。

Microsoft Index Server，提供的中文全文檢索的功能，包括：

  • 支援Active Server Pages（ASP）：使用ASP即可以設計搜尋表單、和搜尋網站內容的結果顯示格式，以及產生複雜精確的搜尋。

  • 支援SQL查詢：除了使用標準的搜尋表單，另外一個全文檢索的方式，為於ASP和ADO的應用程式當中，使用SQL 查詢（SQL query）。

  • 容易管理：使用Microsoft Management Console（管理控制台，簡稱MMC）管理Microsoft Index Server。一個網站或虛擬目錄，若要加入全文檢索的搜尋範圍，只要於MMC中選擇按滑鼠右鍵，選擇內容（property）即可設定是否加入全文檢索的搜尋範圍。

  • 支援中文等多種語文。
