微軟Web產品與技術有那些呢？

微軟公司的三層式（3-tier）Web架構，包括使用者介面層（presentation tier）、商業邏輯層（business logic tier）、和資料服務層（data services tier）。

微軟提供的Web產品與技術，如下：

使用者介面層的技術

使用者介面層（presentation tier），負責接收使用者的資料輸入，和將結果顯示出來， Web的應用指的使用者介面層就是瀏覽器。

於使用者介面層，微軟所提供的Web產品與技術為：

  • Microsoft Internet Explorer 4.0（或以後版本）瀏覽器的「動態HTML」（Dynamic HTML）。

商業邏輯層的技術

商業邏輯層（business logic tier），作為使用者介面與資料庫的橋樑，負責資料處理、網站伺服器（web server）等商業法則（business rules）的工作，譬如使用IIS（Internet Information Server）的網站伺服器，採得ASP（Active Server Pages）撰寫程式，並透過MTS（Microsoft Transaction Server）管理元件，再透過ODBC或OLE DB與各種資料庫相連接。

於商業邏輯層，除了IIS網站伺服器外，微軟所提供的Web產品與技術為：

  • MTS（Microsoft Transaction Server）：提供Transaction（交易或異動）與元件管理的功能。

  • SMTP Service：提供傳送e-mail的功能。

  • Index Server ：提供中文全文檢索的功能。

  • Certificate Server：提供譬如SSL安全傳輸的伺服器認證功能。

  • ADO（ActiveX Data Object）/ RDS（Remote Data Service）：提供資料庫連結的功能。

以上的功能都包括於Windows NT Option Pack 的光碟片當中。

資料服務層的技術

資料服務層（data services tier），負責資料庫或訊息的處理，譬如使用SQL Server資料庫的預儲程序（stored procedures），或使用MSMQ做非同步訊息的處理等。於資料服務層，微軟所提供的Web產品與技術為：

  • SQL Server 7.0：資料庫。

  • MSMQ（Microsoft Message Queue Server）：提供非同步訊息的功能。包括於Windows NT Option Pack 的光碟片當中。

相關的Web技術

於程式開發方面，微軟所提供的Web產品與技術為：

  • 開發語言：使用ASP（Active Server Pages）撰寫Web應用程式，使用Visual Basic或Visual C++等語言來開發元件（component）。

  • 開發工具：Visual InterDev 6.0 。

於管理方面，微軟所提供的Web產品與技術為：

  • IIS（Internet Information Server）管理：可以使用ASP（Active Server Pages）、WSH（Windows Scripting Host）、Visual Basic等語言，透過程式來管理IIS網站伺服器，譬如新增加一個虛擬目錄等。

  • MTS（Microsoft Transaction Server）管理：可以使用ASP、WSH、Visual Basic等語言，透過程式來管理MTS，譬如新增加一個元件等。

　

如何製作一個「動態」的網站呢？

您的網站還是「靜態」的嗎？

目前Internet上許多的網站，仍然提供「靜態」（static）的網頁內容。

所謂「靜態」，指的就是網站的網頁內容是「固定不變」地。

當瀏覽器透過Internet的HTTP（Hypertext Transport Protocol）協定，向網站伺服器（web server）要求提供網頁的內容時，網站伺服器收到要求後，傳送原已設計好靜態的HTML的檔案資料給瀏覽器。

「傳統」的網站是屬於「靜態」的，僅僅使用標準的HTML碼，頂多加上GIF89A格式的圖片，產生幾隻小貓小狗跑來跑去的動畫效果，若要更新網頁的內容，必須手動地來更新其HTML的檔案資料。「靜態」網站的一個大缺點就是不容易維護，為了不斷更新網頁內容，必須不斷地製作新的HTML檔案資料。

「傳統」的網站伺服器並沒有做甚麼事，僅僅將使用者所要求的HTML檔案，傳送到瀏覽器而已。

甚麼是「動態的網站」？

一個「動態」的網站，網站的網頁內容是「動態」地。

所謂「動態」的網站，指的是：

  • 「內容會變」：網頁內容會依照不同時間、不同使用者，而提供不同的網頁內容。

  • 「自動更新」：網頁內容會自動更新。

  • 「互動性」：網頁內容會依照使用者的需要或選擇而改變。

如何製作一個「動態的網站」？

將原本「靜態」的網站，變成一個「動態」的網站，方法如下：

  • 將網站「asp化」：所謂「asp化」，即採用於IIS網站伺服器執行的asp碼，使用程式控制，「動態」地產生網頁內容。

  • 將網站「資料庫化」：所謂「資料庫化」，即使用資料庫管理整個網站，只要更新資料庫的內容，網站的內容將自動更新。

  • 將網站「元件化」：撰寫元件，採用MTS多層式架構，多人上網共享資源，提供執行效能。

將網站asp化、資料庫化、和元件化，筆者稱為「網站3化」。

將網站asp化、資料庫化、和元件化的好處在那裡？

將網站「asp化」的好處，為以程式控制網站、個人化、增加互動性、設計迅速與維護方便。

將網站「資料庫化」的好處，為以程式控制網站、網頁自動更新、各種Web資料庫應用、直接使用瀏覽器管理與維護。

將網站「元件化」的好處，為可以重覆使用、執行速度增快10~30%、保護原始碼、多人使用時共享資源。

將網站「asp化」的好處在那裡？

「動態」的網站伺服器，必須執行一些程式，然後將執行的結果輸出成標準的HTML格式檔案，傳送到使用者的瀏覽器。

所謂將網站「asp化」，就是於IIS或PWS網站伺服器執行asp程式，使用程式來控制網站。

asp（Active Server Pages）為於IIS或PWS網站伺服器執行的script語言，使用VBScript或JScript來撰寫程式。

將網站「asp化」的好處在那裡？好處如下：

  • 以程式控制網站：每次內容不同。

  • 個人化：因人因時內容不同。

  • 增加互動性。

  • 設計迅速與維護方便：asp不須compile編譯，使用記事本即可編輯設計。

將網站「資料庫化」的好處在那裡？

甚麼是「資料庫化的網站」？所謂將網站「資料庫化」，指的是使用「資料庫」管理整個網站，只要更新資料庫的內容，網站的內容將自動更新。

採用資料庫，將網站「資料庫化」的好處在那裡？好處如下：

  • 訪客自由搜尋：將網站的內容儲存於資料庫當中，可以搜尋網站的資料。

  • 網頁自動更新：只要更新資料庫的資料，網頁內容會自動更新。

  • 各種Web資料庫應用。

  • 管理方便：直接使用瀏覽器管理與維護。

讓我們看幾個例子。安裝所附的『網站熱門應用精選』光碟，就可以看到將網站「資料庫化」的好處。

將網站「資料庫化」的第一個好處，就是使用「資料庫」管理整個網站，只要更新資料庫的內容，網站的內容將自動更新。譬如「新消息」、「輪播廣告圖片」、「檔案下載」、「相關網站」等常見的網站功能，就可以使用「資料庫」管理整個網站的內容，自動更新網頁。

「新消息」真的是「新的消息」嗎？

當連上網際網路瀏覽各網站時，偶而會看到「過期的新消息」，甚至去年的消息仍然列為「新消息」，這時候您就可以寫封信給站長，來嘲笑其新消息「過期」了。

其實這不是站長的錯，日期一天天的過，總不能要求站長每天都去檢查每一則「新消息」是否過期。

但是如果站長將網站「資料庫化」，使用「資料庫」來管理網站的「新消息」，只要更新資料庫的內容，網站的「新消息」將自動更新。

譬如筆者網站「www.asp.com.tw」中的「新消息」部份，每一則「新消息」都於下圖的「資料庫」設定其「開始日期」和「結束日期」。當使用者瀏覽此網頁時，將自動比較當天日期是否介於資料庫中「開始日期」和「結束日期」之間，只將符合此條件的「新消息」顯示出來，顯示出來的「新消息」保證是「新的消息」。

加入「新消息」時，只要於資料庫新增一筆資料，於資料庫設定「新消息」的「開始日期」和「結束日期」即可，如此一來，網站的「新消息」內容將自動地更新，「新消息」永遠是「新的消息」，站長可以安心地睡個好覺，不用擔心半夜有訪客會寫信來嘲笑「新消息」過期了。

下次您看到「過期」的新消息時，仍然可以寫封信給站長，不是嘲笑其新消息「過期」了，而是嘲笑其網站仍未「資料庫化」。

加入「新消息」時，只要使用瀏覽器於資料庫新增一筆資料。

『網站熱門應用精選』於每天第一個網站訪客進來時，自動讀取「新消息」資料庫以產生新的網頁內容。每天只讀取資料庫一次，不須每次訪客進來時都讀取資料庫，可以加快多人瀏覽的速度。如果您的「新消息」是隨時變動的，當然您也可設定為「即時」更新新消息。

您可以設定是否顯示消息簡介或只顯示標題，也可超連結到詳細說明的html檔案。

如果當天沒有最新的「新消息」時，不用擔心，『網站熱門應用精選』將自動傳送EMAIL給維護人員以告之。

對於不同種類的「新消息」，您可以做分類，不同的分類可以設定不同的標題圖片。

可以使用資料庫管理「輪播廣告圖片」嗎？

『網站熱門應用精選』讓您使用瀏覽器來管理資料庫中的「廣告圖片播放期間」，於每天第一個網站訪客進來時，自動讀取「廣告」資料庫以產生新的輪播圖片內容，自動輪流顯示，並自動記錄廣告圖片被按到的次數。

「檔案下載」可以使用資料庫管理嗎？

『網站熱門應用精選』也提供使用資料庫管理「檔案下載」的功能，訪客可下載檔案，自動顯示檔案的大小，以及記錄檔案下載的次數。

訪客可依照分類與關鍵字搜尋檔案。

「相關網站」呢？

『網站熱門應用精選』使用資料庫管理「相關網站」，只要於瀏覽器更新資料庫，自動產生新的相關網站網頁內容。

訪客可以「搜查」產品/校園/便民資訊嗎？

將網站「資料庫化」的第二個好處，就是將網站的內容儲存於資料庫當中，訪客可以自由搜尋網站的資料。譬如「產品/校園/便民資訊」、「FAQ」（常見問題與解答）等常見的網站功能，將網站的資訊放到「資料庫」堶情A就可以提供訪客搜查資訊的功能。

大部份現有的網站都無法提供讓訪客搜查資訊的功能，這是因為沒有使用資料庫來管理產品/校園/便民資訊的緣故。

『網站熱門應用精選』，讓您公司自行於瀏覽器更新資料庫的產品/校園/便民資訊，不用修改網頁，自動產生新的產品/校園/便民資訊網頁。訪客可依照分類與關鍵字來搜尋有興趣的資訊。

有提供「FAQ」的功能嗎？

有的，『網站熱門應用精選』也提供使用資料庫管理「FAQ」（常見問題與解答）的功能，只要於瀏覽器更新資料庫，自動產生新的FAQ網頁內容。

因為使用資料庫來管理FAQ的緣故，訪客自可依照分類與關鍵字搜尋常見問題解答。

開發各種Web資料庫的應用

將網站「資料庫化」的另一個好處，就是可以開發出各種Web資料庫的應用。

使用者使用瀏覽器，透過網際網路或intranet內部網路，來存取Web資料庫的資料，可以使用於學校教學、醫院、商業、銀行、股市、和運輸旅遊等各種Web資料庫的應用，譬如銀行餘額線上查詢、線上機票訂位、線上醫院預約掛號、網路下單、線上股市買賣交易、線上學校註冊選課、線上擇友、線上購物等各種應用。

Web-based應用軟體

將目前各種網路上的應用軟體改變為「Web資料庫」的應用，使用「瀏覽器」做為使用者介面，是必然的趨勢。

以往每個使用者都必須去學習每一種應用軟體的操作方式。Web資料庫應用使用「瀏覽器」做為使用者介面，對於使用者的好處是只要學會瀏覽器的操作方式，譬如看到有超連結的部份就按下去，如此可以操作各種軟體應用，大幅降低訓練的成本。

以往每當修改一個bug時，就須為所有使用者去更新一個新的版本。Web資料庫應用對於設計者的好處是，新版本的軟體只要更新一次網站的軟體就可以了。

基本上，所有的網路上的應用軟體，都可以改變為Web資料庫的應用，使用「瀏覽器」做為使用者介面。

　

將網站「元件化」的好處在那裡？

所謂將網站「元件化」，指的是使用Visual Basic或Visual C++等語言來開發元件（component），再使用ASP（Active Server Pages）來呼叫所撰寫的元件。

元件（component），為ActiveX DLL，副檔名為.dll。

將網站「元件化」的好處在那裡？好處如下：

  • 可以重覆使用。

  • 執行速度增快10~30%。

  • 保護您的原始碼。

  • 多人使用時共享資源。

元件就像一個零件一般，第一個好處就是可以重覆使用。

由於元件是經過compile過的，執行速度依據筆者的測試結果，大概可以增快10~30%。

也由於元件是經過compile過的，所以可以保護您的原始碼。不像ASP（Active Server Pages），原始碼可以一覽無遺。

撰寫的元件，使用MTS（Microsoft Transaction Server）來管理，就可以享受到MTS多人使用時共享資源的功能，多人使用時效率將提高。
