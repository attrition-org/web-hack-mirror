Internet Information Server

您想要在Windows NT或 Windows 95/98上，迅速架設一個「超級」的Internet、或Intranet網站伺服器嗎？

需要選擇什麼樣的軟體呢？不用再考慮了！「俗又大碗」的唯一選擇就是Microsoft公司的「Windows NT Option Pack」中文版軟體，包括：

  • Internet Information Server（IIS） 4.0：Intranet 與Internet的 Web 伺服器。安裝於Windows NT Server 4.0 或 Windows NT Workstation 4.0。

  • Personal Web Server 4.0：安裝於Windows 95/98的Web 伺服器。

  • Index Server 2.0：中文全文檢索。

  • Certificate Server 2.0：作為認證中心（Certificate Authority），以管理數位認證的授與、廢除及更新，數位認證用於公開金鑰加解密之應用，譬如SSL的伺服器認證與用戶端認證，讓公司透過Intranet或 Internet 安全地傳送資料。

  • Message Queue Server 2.0：讓不同時間執行的多個應用程式，能夠相互交換資料訊息。透過訊息的傳送與接收，使得與其他應用程式之間的通訊變得快速、可信及非同步。

  • Transaction Server 2.0：交易（transaction）功能。

  • Site Server Express：提供用法分析（usage analysis）、站台分析（content analysis）、發行（publish）內容的功能。

  • Data Access Components 1.5 資料庫元件：包括ActiveX Data Objects （ADO） 及 Remote Data Service（RDS）、Microsoft OLE DB Provider for ODBC 以及 Open Database Connectivity（ODBC），以存取網站伺服器資料庫的資料。

  • Microsoft Script Debugger： 可以對包括VBScript 、JScript、Java 的ASP程式碼做除錯的動作。可以使用 Script Debugger 來設定中斷點、一次執行一條程式、檢視與更新變數的值等。

  • Internet Explorer 4.01：IE 4.0瀏覽器的更新版本。

Microsoft公司這次推出的「Windows NT Option Pack」中文版軟體，幾乎將Microsoft開發的所有Web伺服器都頃巢而出，涵蓋了Web伺服器所需的各種完整功能。以筆者的角度來看，「Windows NT Option Pack」將大幅提升使用Windows NT作為Web伺服器的市場佔有率，加上是免費下載的（CD版本為NT$990），殺傷力之大，可預見使用UNIX作為Web伺服器的市場，以及Web伺服器的廠商如Netscape公司，將大受影響！

IIS到了4.0版本，功能增加了「非常地多」，功能變成「非常地強」，包括於Windows NT Option Pack或Windows NT Server 5.0當中。新出貨的Windows NT Server 4.0當中，也包括Windows NT Option Pack。

管理方面新增功能

<特異功能1> 多種網站管理方式

IIS 4.0使用一個單一整合的Microsoft Management Console（管理控制台，簡稱MMC），叫做Internet服務管理員，來共同管理Internet Service（WWW、FTP、NNTP、SMTP）、Microsoft Transaction Server、和Microsoft Index Server。

MMC使用一個相同的整合控制介面畫面，即可做所有的管理，可以節省用戶學習多個管理介面的時間。

除了MMC的Internet服務管理員外，IIS尚提供兩種管理方式：

  • Internet服務管理員（HTML）：使用瀏覽器透過Internet或Intranet，於遠端來管理WWW、FTP站台。

  • Windows Scripting Host：可以DOS指令行的方式，來管理WWW、FTP站台。

<特異功能2> 詳細的記錄

IIS 4.0提供兩種新的網站記錄（log）方式，如下：

  • 使用W3C的Extended Logging標準格式。這個格式是一個可以客戶化的ASCII格式，讓管理者可以移除不必要的欄位，以限制記錄的檔案大小。

  • 大部份IIS 4.0的記錄（log），可以設定到每個檔案的方式，可以選擇那些資源需要做記錄，減少記錄的檔案大小。

安全方面新增功能

<特異功能3> 驗證用戶身份

當用戶連線到伺服器時，有時候伺服器需要驗證用戶的身份。Windows NT Server和IIS 4.0提供三種驗證用戶身份的方式，如下：

  • 基本驗證（Basic Authentication）：使用傳統上輸入用戶名稱和密碼的方式登入伺服器，未加密的密碼會於網路上傳送，因此可能會被截取盜用。

  • Windows NT Challenge/Response（NTLM）：使用加密技術以驗證密碼，實際的密碼並不會於網路上傳送，因此不會被截取盜用。但僅適用於使用IE瀏覽器。

  • 用戶端認證（Client Certificate）：用戶端認證的方式，讓用戶登入伺服器時，不須輸入用戶名稱和密碼。

<特異功能4> 用戶端認證（Client Certificate）

用戶端認證一般由認證中心（Certificate Authority）所核發， IIS 4.0的Certificate Server，使您公司能夠自行建立一個認證中心，自己核發用戶端認證給所有用戶，讓用戶安全地存取伺服器的資訊。

IIS 4.0並且提供用戶端認證與Windows NT user account的對照表，以用戶端認證方式自動登入伺服器的用戶，依照此對照表，即享有所對應到user account的權限。

<特異功能5> SSL安全協定

使用SSL（Secure Sockets Layer）安全協定的方式，用戶與伺服器可以建立安全的連線，傳送前先將資料加密，以於網路上傳送加過密的資料，接收時再將資料解密以獲得原來的資料。

IIS 4.0支援SSL 2.0和3.0的標準規格。

使用SSL前，伺服器必須先取得伺服器認證（Server Certificate）。於SSL連線時，伺服器將其伺服器認證傳送給用戶，讓用戶驗證伺服器的身份。

伺服器也可以要求用戶傳送其用戶端認證，讓伺服器驗證用戶的身份，但是並不是需要如此。

　

標準方面新增功能

<特異功能6> 頻寬限制

管理者可以將放置於一台伺服器的每一個網站，做分配頻寬（allocate bandwidth）的動作，限制一個網站最多的頻寬，以保證所有網站都可以享有頻寬，不會被一個熱門的網站吃掉所有的頻寬。

<特異功能7> 多個網站共用一個IP Address

一台伺服器放置多個網站的缺點，為每一個網站都需要一個單獨的IP Address。IIS 4.0採用HTTP 1.1標準的主機標題（Host Header）方式，以解決這個問題，使多個網站能夠共用一個IP Address。

<特異功能8> 各自管理網站

放置於同一台伺服器的每一個網站之管理者，可以各自管理其網站，而不互相影響。譬如每一個網站之管理者，可以設定其網站的存取權限、啟動記錄、改變預設文件表尾、設定內容有效期間、設定內容分級等。

<特異功能9> 全權控制

因為IIS提供全權控制的功能，管理者能夠更有彈性地，管理整個網站伺服器以及各自的網站。

這些全權控制的功能，可針對一個網站伺服器、各自的網站、虛擬目錄、檔案等層次，做一些設定，這些設定包括記錄、預設文件、安全等。

<特異功能10> 效能監視

IIS讓管理者能夠針對一個網站伺服器中各自的網站，做效能的監視。

使用Windows NT的效能監視器（Performance Monitor），可以監視網站伺服器的效能。效能監視器使用計數器（counters），以監視每個時段的檔案傳輸率、頻寬使用率、連線率等。

<特異功能11> 支援HTTP 1.1標準協定

當用戶連線到伺服器要求傳送一個網頁時，若此網頁包括十張圖片，需要建立十次不同的連線。建立連線和關閉連線的動作，花了許多時間和耗盡更多用戶、伺服器、網路的資源。使用IIS 4.0所支援HTTP 1.1標準協定的持續連線法（或稱為keep alives），用戶於指定的時間內所提出的每個HTTP request，可以與伺服器一直保持連線，不需要重新建立連線的動作。持續連線法可以提高整體的效能。

<特異功能12> 傳送電子郵件功能

原本於IIS 3.0的版本，若要提供傳送電子郵件的功能，必須另外安裝third party的ActiveX伺服器元件。

到了IIS 4.0，內建的SMTP Service提供傳送電子郵件的功能。您可以結合ASP程式碼和資料庫，傳送所須內容的電子郵件給不同的用戶。

<特異功能13> 新聞討論群組功能

IIS 4.0，內建的NNTP Service提供新聞討論群組的功能。由於使用標準的NNTP協定，任何符合標準的討論群組使用者，都可以參予新聞討論群組的討論。

這個NNTP Service 功能，並未提供新聞回覆（news feeds）或複製（replication）的功能。這些功能可以藉由加裝Microsoft Exchange Server來達到。

開發方面新增功能

<特異功能14> Active Server Pages（ASP）

Active Server Pages（動態伺服器網頁，簡稱ASP）的程式碼，是一種於IIS 3.0或IIS 4.0網站伺服器執行的語言，讓您結合HTML、VBScript/JScript等script語言、和ActiveX Server component（ActiveX伺服器元件），以快速開發出各種網站應用。

ASP與瀏覽器無關，由於ASP係先於IIS 3.0或IIS 4.0的網站伺服器執行後，產生標準的HTML碼傳送到用戶端的瀏覽器，因此於各種瀏覽器均可執行。即使ASP採用只有IE瀏覽器支援的VBScript語言，由於VBScript語言的程式碼已經於網站伺服器執行，傳送到不支援VBScript語言的瀏覽器譬如Netscape的只剩下標準的HTML碼，因此Netscape瀏覽器也可以執行包括VBScript的ASP程式碼。

<特異功能15> Microsoft Script Debugger

IIS 4.0和IE 4.0所包括的Microsoft Script Debugger，提供於伺服器端和用戶端執行的VBScript/JScript等script語言的偵錯功能，包括逐步執行與設定中斷點等。

<特異功能16> 多人使用（multi-user）管理

因為IIS 4.0整合Microsoft Transaction Server，提供多人多工處理管理（automatic thread and process management）的功能， ASP程式碼自動於多人使用（multi-user）的環境下執行。

不需要強迫開發者考慮多人使用環境時的特殊設計，開發者只要於單人使用的環境下開發完成，就可以使用於多人使用的環境。

<特異功能17> 重新載入元件

對於發展網站應用的開發者而言，有一個困擾，就是每次改變一個元件（component），就需要重新啟動應用程式和伺服器。

IIS 4.0提供一個叫做「out of process」的獨立執行功能，當您重新載入元件（component load/unload）時，不須重新啟動伺服器。

<特異功能18> 當機保護

網站應用程式是於多人使用（multiuser）的環境下執行，需要一個高度可靠的伺服器，一個應用程式當機時，不能影響到同一台伺服器其他應用程式的執行，或者甚至讓整台伺服器當機。

IIS 4.0提供當機保護（crash protection）的功能，一旦一個ASP或ISAPI應用程式當機時，僅僅此「out of process」的應用程式當機，整台伺服器和其他應用程式仍然執行不受影響。IIS將於下一個對此應用程式提出執行request時，自動重新啟動當機的應用程式。

<特異功能19> 支援交易（transaction）功能

所謂交易（transaction）功能，指的是於一個交易（transaction）當中，所有的動作（譬如訂單、庫存、付款等）都同時一起成功、或一起失敗。

交易（transaction）功能十分重要，譬如從一個銀行（如甲銀行）帳戶匯款到另一個銀行（如乙銀行）帳戶時，當甲銀行帳戶已經扣除匯款，若此時於乙銀行帳戶的加入匯款動作發生錯誤，則交易（transaction）功能將自動恢復（rollback）原甲銀行帳戶的變動。

IIS 4.0整合Microsoft Transaction Server，提供的交易（transaction）功能。

<特異功能20> 訊息佇存（message queuing）

商業上，公司需要執行於不同系統的多個獨立應用程式，即使於不同時間執行，也能夠相互交換資料訊息。

Microsoft Message Queue Server，讓不同時間執行的多個應用程式，能夠相互交換資料訊息。發展者能夠由ASP傳送非同步的資料訊息（asynchronous message），給遠端者以延期處理（deferred processing）。

<特異功能21> 支援伺服器端Java功能

IIS 4.0包括一個新的Java virtual machine，可以支援伺服器端Java（server-side Java）的功能，於伺服器執行Java component。

Java component可以包含於ASP程式碼當中。

資料連結方面新增功能

<特異功能22> ActiveX Data Objects

ActiveX Data Objects（簡稱ADO），提供高效率的ODBC資料庫、或OLE-DB資料來源的連接功能。

透過ODBC驅動程式，ADO可以存取大部份資料庫的資料，包括非資料庫型態（如試算表和文字檔案）的資料。

ADO與ASP結合，執行SQL命令，讓用戶使用瀏覽器來存取網站伺服器的資料庫資料。只要更換ODBC驅動程式，即可更換為另外一種資料庫，ASP程式碼不需求修改。

ADO與ASP結合Microsoft Transaction Server，可以提供交易（transaction）的功能。

<特異功能23> Remote Data Service（RDS）

Remote Data Service（RDS），原來叫做ADC，是一個用戶端的資料存取技術。配合ActiveX data control（譬如grids、list boxes、combo boxes等），以顯示查詢和更新的結果。

所有的資料更動，都先儲存於用戶端，然後傳送到伺服器以更新資料庫的資料。

RDS適合用戶更新或查詢大量的資料之用。

<特異功能24> Open Database Connectivity（ODBC）

Open Database Connectivity（ODBC），提供與超過55種資料庫的連接標準介面。

ODBC讓發展者不需要針對一種資料庫做設計，只要於程式碼加入SQL命令，透過ODBC驅動程式，即存取超過55種資料庫的資料。

<特異功能25> 加強Oracle資料庫連接性

IIS 4.0包括了Microsoft Oracle ODBC Driver 2.0，讓發展者使用Oracle資料庫開發出支援ODBC的應用程式。

這個版本也提供額外的性能和控制功能，包括存取PL/SQL packages和XA/DTC整合。

<特異功能26> 主機資料整合

使用SNA Server 4.0，可以連線到AS400和mainframe的資料庫。

內容管理方面新增功能

<特異功能27> 站台分析

Windows NT Option Pack所包括的Microsoft Site Server Express，為Microsoft Site Server的精簡版本，不包括Microsoft Site Server所提供電子商務的功能。

Microsoft Site Server Express提供用法分析（usage analysis）、站台分析（content analysis）、和發行（publish）內容的功能。

<特異功能28> 中文全文檢索

Windows NT Option Pack所包括的Microsoft Index Server，提供中文全文檢索的功能。

用戶使用瀏覽器，輸入關鍵字，即可以搜尋網站的內容。

<特異功能29> 可自行定義錯誤訊息

除了使用預設的錯誤訊息外，IIS 4.0讓您可以自行定義顯然於用戶瀏覽器的錯誤訊息。

<特異功能30> 設定HTTP header

設定HTTP header，可以做：

  • 內容分級（content rating）：由RSAC組織所發展的網頁分級制度PICS（Platform for Internet Content Selection），將網頁分為暴力、裸體、性、攻擊語言等分級。

  • 內容保留期限（content expiration）：設定網頁內容於用戶瀏覽器的保留期限

  • 特殊設定：譬如SET藉由HTTP header的設定，啟動用戶的電子錢包。

  • 使用document footers，管理者可以設定網站伺服器自動插入一個logo圖片。
