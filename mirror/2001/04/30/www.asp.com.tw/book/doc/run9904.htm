將網站「資料庫化」

所謂將網站「資料庫化」，指的是使用「資料庫」管理整個網站，採用資料庫，將網站「資料庫化」的好處在那裡？好處如下：

  • 訪客自由搜尋：將網站的內容儲存於資料庫當中，可以搜尋網站的資料。

  • 網頁自動更新：只要更新資料庫的資料，網頁內容會自動更新。

  • 各種網站資料庫應用。

  • 管理方便：直接使用瀏覽器管理與維護。

如何透過瀏覽器存取網站資料庫的資料呢？答案是：

  • ADO（ActiveX Data Object）

  • RDS（Remote Data Service）

ASP（Active Server Pages）程式呼叫ADO或RDS物件，就可以存取網站資料庫的資料。

上一期介紹ADO網站資料庫的設計技巧。本期將介紹RDS網站資料庫的設計技巧，包括RDS資料庫存取基本技巧（功能、與GRID結合、與HTML結合）、RDS資料庫存取進階技巧（顯示上下筆、顯示上下頁、修改資料）等單元。

RDS（Remote Data Service）

RDS（Remote Data Service）是甚麼東東呢？RDS提供一個資料處理的ActiveX Control，以存取支援ODBC（Open Database Connectivity）的資料庫。

一般的網站資料處理技術，譬如ADO、IDC等，資料查詢的動作，是在伺服端的網站伺服器運作的，每次查詢或更新資料的要求，都須由用戶端發起送到伺服端，伺服器運作查詢或更新資料庫後，再傳回用戶端的瀏覽器顯示出來。這一來一往，對頻繁的資料查詢動作，效率較差。

RDS提供的用戶端資料暫存（Client-Side Data Cache）功能，將大量的資料暫存於用戶端的電腦。RDS將伺服器資料庫可高達數千筆的資料，先下載到用戶端，在用戶端的瀏覽器直接做頻繁的資料查詢動作，大部份時間不須再與伺服器連線，效率較佳。

RDS與其他網站資料處理技術，有一個很大的不同點，就是RDS的資料查詢或更新動作，是在用戶端的瀏覽器上運作的，於瀏覽器更新多筆資料後，再一次送到伺服器更新資料庫。

這個資料查詢就是由安裝於用戶端的RDS ActiveX Control處理的。

RDS將資料庫的大量資料暫存於用戶端（前端），直接於用戶端的瀏覽器查詢資料，節省前端（用戶端）和後端（伺服端）之間資料往返所須的時間。

RDS適合於Internet、Intranet、或LAN上，做頻繁的資料庫資料存取查詢動作。

RDS存取資料庫的功能呢？

RDS（Remote Data Service），提供了以下數個新的功能：

  • Client Cursor Engine

  • 於用戶端排序和搜尋資料（Sorting and Filtering）

  • 與IE 4.0/5.0整合

  • 將一個Recordset轉換成MIME字串

  • 於用戶端產生Recordset物件

簡介如下：

Client Cursor Engine

Client Cursor Engine提供用戶端游標（client-side cursor）的功能。

Client Cursor Engine讓您以非同步的方式，開啟和存取一個未連線的Recordset。

能夠提供這項功能的主要原因，是因為提供了下列的屬性（properties）和方法（methods）：

  • ExecuteOptions屬性：執行為同步方式或為非同步方式，是否同步存取下一個Recordset。即是否使用獨立的thread（separate thread），非同步方式來執行DataControl物件的Refresh方法。

  • FetchOptions屬性：設定讀取下載資料的方式，可以設定是否必須等到將所有的查詢資料都讀取下載到Recordset後，才可以做其他的動作。

  • ReadyState屬性：傳回RDS的狀態，譬如已經開啟RDS的Recordset但尚無資料、Recordset正在接受資料當中、Recordset已經接受完資料等狀態。

  • Cancel方法：取消正在進行的非同步（asynchronous）執行或讀取動作。

Remote Data Service的Client Cursor Engine，讓您可以更新許多的資料庫，譬如Microsoft SQL Server、Microsoft Access、Oracle等，以前只能更新Microsoft SQL Server 資料庫。

於用戶端排序和搜尋資料

RDS提供於用戶端，對於暫存（cache）的資料，進行排序和搜尋資料的功能。

由於提供下列的屬性（properties）和方法（methods），您可以選擇待排序和搜尋資料的欄位名稱、排序的方向、搜尋的原則：

  • SortColumn屬性：設定待排序的欄位名稱。

  • SortDirection屬性：設定待排序的方向，True 表示正向排序（Ascending order），False表示反向排序（Descending order）。

  • FilterColumn屬性：設定待搜尋的欄位名稱。

  • FilterCriterion屬性：設定待搜尋的原則。

  • FilterValue屬性：設定待搜尋的比較值。

  • Reset方法：開始於用戶端執行排序和搜尋的動作。

與IE 4.0/5.0整合

Internet Explorer 4.0/5.0瀏覽器，已經包括Remote Data Service用戶端執的元件，您不需要於RDS ActiveX Control使用CODEBASE 參數來下載Remote Data Service .cab檔案。

最大的好處，是RDS存取資料庫所下載到瀏覽器的資料，可以與IE 4.0/5.0大部份的HTML標記相結合，以顯示或更新資料。

使用IE 4.0/5.0的HTML標記，以顯示或更新RDS資料，就不需要使用DataControl物件了。

將一個Recordset轉換成MIME字串

ConvertToString方法，將一個Recordset轉換成MIME字串。

於用戶端產生Recordset物件

CreateRecordset方法，讓您直接於用戶端產生Recordset物件。

DataControl物件提供唯讀的Recordset和唯寫的SourceRecordset屬性。

RDS執行環境

於伺服器端所須的RDS執行環境，如下：

  • Windows NT Server version 4.0包括Service Pack 3或以後的版本。

  • Internet Information Server（IIS）3.0或以後的版本，包括安裝Active Server Pages。

  • Microsoft SQL Server或其他支援ODBC Level 2的資料庫。

於用戶端所須的RDS執行環境，如下：

  • Windows 95、Windows NT Workstation 4.0 、Windows NT Server 4.0 包括Service Pack 3或以後的版本。

  • 支援ActiveX的瀏覽器，譬如Microsoft Internet Explorer 3.0以上版本等。

三層式架構與RDS

若要善用RDS（Remote Data Service）技術，必須了解甚麼是「三層式架構」的應用程式（Three-Tiered Applications）。所謂「三層式架構」，指得是以下三層：

  • 用戶端層（Client tier）：為一台local電腦，提供使用者顯示介面，以及處理遠端資料來源的資料。一般為使用瀏覽器（Web-based應用程式）、或使用一個單獨執行經編譯過的應用程式（stand-alone compiled front-end application），做為使用者介面。

  • 中間層（Middle tier）：為一台提供商業法則（business rules）的Windows NT Server電腦，Web-based應用程式為執行ASP（Active Server Page）語言的IIS（Internet Information Server），非Web-based應用程式則執行一個經編譯過的可執行應用程式。

  • 資料來源層（Data source tier）：為一台提供資料庫管理系統（database management system，簡稱DBMS）的電腦，譬如Microsoft SQL Server 資料庫。

「兩層式架構」與「三層式架構」主要相異處，為中間層（Middle tier）與資料來源層（Data source tier）被合併為一層。

「三層式架構」指得是邏輯上的三層，實際上，這「三層」並不需要區分為網路上的「三台」電腦，譬如您可以使用兩台電腦，一台Windows 95/98電腦執行瀏覽器，另外一台Windows NT Server電腦執行執行IIS和Microsoft SQL Server。您可以根據效能和管理來決定要使用幾台的電腦。

當您開發Remote Data Service的應用程式時，您可以區分應用程式為三個邏輯層。

中間層（Middle tier）執行IIS。用戶端層（Client tier）一般為使用Internet Explorer瀏覽器，使用HTTP協定與伺服器通訊，由於Internet Explorer 4.0或以後版本已經包含RDS client端元件（client-side RDS components），因此可以簡化您的開發工作。

一旦使用者從用戶端層（Client tier）要求查詢資料，經過RDS client端元件，和RDS 伺服器端元件（server-side RDS components），將查詢需求傳送到IIS Web伺服器和DBMS。

RDS 伺服器端元件將DBMS 將所執行結果的資料傳送到一個ADO Recordset物件，然後經過網路送回到用戶端電腦。

用戶端電腦可以使用一個data-aware control，譬如一個text box 或grid，來顯示資料，管理資料的是RDS.DataControl物件。一個RDS.DataControl物件可以管理數個data-aware controls。

RDS提供的用戶端資料暫存（Client-Side Data Cache）功能，將大量的資料暫存於用戶端的電腦，可以減少與Web的連線數目，以提高效率。

　

RDS存取資料庫如何與GRID結合呢？

RDS將存取查詢資料庫的結果顯示出來，有兩種方法：

  1. GRID ActiveX controls：將RDS存取查詢資料庫的結果Recordset，連接到一個或多個的GRID ActiveX controls，於用戶端的瀏覽器顯示出來，譬如Apex公司的Datagrid、和Sheridan公司的DropDown list等。

  2. 您也可使用IE 4.0/5.0的大部份HTML標記，以顯示或更新資料。

RDS資料庫設計要訣

RDS資料庫設計要訣的步驟，如下：

  1. 將RDS ActiveX Control，使用<OBJECT>...</OBJECT>標記插入HTML碼當中。

  2. 於HTML碼要顯示的地方，插入顯示查詢結果GRID ActiveX Control或使用IE 4.0/5.0的大部份HTML標記。

  3. 於「rds.Server」設定IIS網址，「rds.Connect」設定資料庫連線，和「rds.SQL」設定SQL指令，再下個「rds.Refresh」指令，即可開始執行存取查詢資料庫的動作。

RDS資料庫設計要訣的第一步驟，就是將RDS ActiveX Control，使用<OBJECT>...</OBJECT>標記插入HTML碼當中，如下：

<OBJECT classid="clsid:BD96C556-65A3-11D0-983A-00C04FC29E33"

ID=rds HEIGHT=1 WIDTH = 1>

<PARAM NAME="BINDINGS" VALUE="GRID1;">

</OBJECT>

用戶端的瀏覽器一旦瀏覽此HTML網頁，將自動下載和安裝RDS ActiveX control於用戶端的硬碟當中，只要第一次下載和安裝即可，不必每次瀏覽時都下載。IE 4.0/5.0已經包括這個RDS ActiveX control。

「BINDINGS」設定顯示查詢結果GRID ActiveX control的名稱（ID）。

第二步驟於HTML碼要顯示的地方，插入顯示此查詢結果GRID ActiveX Control的<OBJECT>...</OBJECT>標記，如下：

<OBJECT ID="GRID1" WIDTH=600 HEIGHT=200 Datasrc="#rds"

CODEBASE="http://<%=Request.ServerVariables("SERVER_NAME")%>/MSADC/Samples/ssdatb32.cab"

CLASSID="CLSID:AC05DC80-7DF1-11d0-839E-00A024A94B3A">

</OBJECT>

顯示查詢結果GRID ActiveX Control的相關檔案，為「CODEBASE」所設定的ssdatb32.cab檔案，將自動下載和安裝於用戶端的硬碟當中。

第二步驟也可使用IE 4.0/5.0的大部份HTML標記，不一定要使用GRID ActiveX Control。

第三步驟於「rds.Server」設定IIS網址，「rds.Connect」設定資料庫連線，和「rds.SQL」設定SQL指令，再下個「rds.Refresh」指令，即可開始執行存取查詢資料庫的動作。如下：

<SCRIPT LANGUAGE="VBScript">

SUB Window_OnLoad

rds.Server = "http://<%=Request.ServerVariables("SERVER_NAME")%>"

rds.Connect = "Driver={Microsoft Access Driver (*.mdb)};DBQ=<% = Server.MapPath("ntopsamp.mdb") %>"

rds.SQL = "Select * from 著作"

rds.Refresh

End Sub

</SCRIPT>

看個例子就可知曉。

RDS ActiveX control設定

第三步驟於「rds.Server」設定IIS網址，「rds.Connect」設定資料庫連線，和「rds.SQL」設定SQL指令的部份，您也可以於RDS ActiveX control的參數當中直接設定，執行結果與上例相同。

程式碼部份，與上例的差異處為：

<OBJECT classid="clsid:BD96C556-65A3-11D0-983A-00C04FC29E33"

ID=rds HEIGHT=1 WIDTH = 1>

<PARAM NAME="BINDINGS" VALUE="GRID1;">

RDS存取資料庫如何與HTML結合呢？

HTML碼如何顯示執行RDS查詢資料庫資料的結果呢？

與HTML標記相結合的技巧如下：

  1. 於HTML標記的DATASRC參數，設定資料來源。

  2. 於HTML標記的DATAFLD參數，設定資料欄位名稱。

程式碼部份，首先使用RDS的「rds3.Server」連結到Access資料庫ntopsamp.mdb，「rds3.SQL」下SQL指令，「Select * from 著作」查詢資料表「著作」的所有資料，將查詢的結果傳回到rds3的ActiveX control當中：

<SCRIPT LANGUAGE="VBScript">

SUB Window_OnLoad

rds3.Server = "http://<%=Request.ServerVariables("SERVER_NAME") %>"

rds3.Connect = "Driver={Microsoft Access Driver (*.mdb)};DBQ=<% = Server.MapPath("ntopsamp.mdb") %>"

rds3.SQL = "Select * from 著作"

rds3.Refresh

End Sub

</SCRIPT>

<OBJECT classid="clsid:BD96C556-65A3-11D0-983A-00C04FC29E33"

ID=rds3 HEIGHT=1 WIDTH=1>

</OBJECT>

其他html標記

與文字輸入的HTML碼Text相結合的ASP程式碼如下：

<input type=text size=30 DataSrc="#rds3" DataFld="書名">

其中「DataSrc="#rds3"」設定資料來源為rds3的ActiveX control，「DataFld="書名"」將文字輸入預設為欄位「書名」的資料。

其中「DataSrc="#rds3"」設定資料來源為rds3的ActiveX control，「DataFld="書名"」將多行文字輸入預設為欄位「書名」的資料。

與勾選的HTML碼Checkbox相結合的ASP程式碼如下：

<input type=checkbox DataSrc="#rds3" DataFld="新書">新書

其中「DataSrc="#rds3"」設定資料來源為rds3的ActiveX control，「DataFld="新書"」將檢查「著作」資料表中欄位「新書」資料的值，若為True則勾選（Checked）。

與圖片的HTML碼<IMG…>相結合的ASP程式碼如下：

<img DataSrc="#rds3" DataFld="圖片">

其中「DataSrc="#rds3"」設定資料來源為rds3的ActiveX control，「DataFld="圖片"」將欄位為「圖片」的資料顯示為圖片。

與超連結的HTML碼<A HREF…>相結合的ASP程式碼如下：

「DataSrc="#rds3"」設定資料來源為rds3的ActiveX control，「DataFld="簡介"」將欄位為「簡介」的資料顯示為超連結。

與下拉式選單的HTML碼<SELECT>相結合的ASP程式碼如下：
