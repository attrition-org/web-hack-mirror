Actif KM Builder 2001
喬篷知識管理建構系統

簡介

於知識經濟與e化的時代中，為何要建構一個知識管理入口網站(KM Portal)呢？
知識管理入口網站讓所有內部人員快速得到個人專屬的資訊，以及增加群組協同作業的效率，更重要的是讓內部的知識經驗傳承累積成為企業的重要知識資產。

您想快速建立內部知識管理的入口網站嗎？
Actif KM Builder喬篷知識管理建構系統，為企業、政府、學校內部知識管理入口網站的快速建構軟體，善用Microsoft Exchange 2000 Server、Windows 2000最新的Web Storage、XML、Active Directory、與數位儀表板等技術與功能，於瀏覽器管理包括文件、流程、資訊、應用、訊息、資源、檔案等所有知識。
個人化與統一管理的數位儀表板、40個隨拖拉即用的常用知識功能模組、及規則設定管理(Rule-based Management)，讓您迅速於內部建立群組中心、資訊中心、資源中心、流程中心、搜尋中心等知識管理的入口網站。

Actif KM Builder 2001版新增加「規則設定管理」模組與多語版功能。
規則設定管理(Rule-based Management)：「規則設定管理」(Rule-based Management) ，依照使用者資料設定各個樹狀節點數位儀表板、及文件的使用權限規則。
多語版：一台伺服器同時提供中繁、中簡、英文等多語版，管理者自行設定。

 八大特點：

1. 個人化與統一管理的數位儀表板  
2. 「規則設定管理」(Rule-based Management)
3. 群組中心 (群組工作區)  
4. 資訊中心 (All-In-One樹狀知識匣)  
5. 資源中心  
6. 流程中心  
7. 搜尋中心 (全文檢索 + 動態分類)  
8. 40個常用知識功能模組隨拖拉即用

 

安裝

 Actif KM Builder必須安裝於Windows 2000 Server中文版電腦上。請先確定您的電腦是否已經安裝了Windows 2000 Server中文版，檔案格式必須為NTFS。
你可以直接開始安裝, 或是查閱安裝詳細說明。

若您的電腦或網路上其他電腦已經安裝了Exchange 2000 Server時，則可以使用：
1. 個人訊息、與資源中心的功能。
2. 群組中心、資料中心可以多使用郵件、記事、日誌、工作、行事曆、連絡人等訊息功能。

安裝WebFlow後即可使用流程中心的功能(WebFlow可安裝於同一台電腦或網路上其他電腦上)。

WebFlow、Exchange 2000 Server、SQL Server 7.0/2000可安裝於同一台電腦或網路上其他電腦上。

若您的電腦或網路上其他電腦尚未安裝了SQL Server時 ，則可以選擇Access做為資料庫(Access僅供測試使用，正式使用時建議採用 Microsoft SQL 7.0/2000資料庫)。若您的電腦或網路上其他電腦已經安裝了SQL Server 7.0或SQL Server 2000 時，安裝時請選擇SQL Server。

您可選擇使用Windows 2000 Server的Active Directory或Windows NT 4.0 Server的PDC來管理使用者帳號及群組。  

安裝注意：
請注意安裝者所登入的帳號必須為Windows 2000 Server 本機Administrator。
1. 安裝前注意務必不要開啟元件服務、Internet服務管理員、Microsoft SQL Enterprise Manager 等畫面。
2. 安裝後請複製 \KMBuilder\KMFolders2\所有檔案 到Exchange 2000 Server /public/KMFolders2/ 。
執行http://localhost/exchange： 
於 /public/KMFolders2/資源管理中心/會議室/ 選 "約會項目" 新增 "會議室101" "會議室201" 資料夾。
於 /public/KMFolders2/資源管理中心/投影機/ 選 "約會項目" 新增 "投影機1" "投影機2" 資料夾。
3. 請於Windows 2000 Server 程式集/系統管理工具/服務啟動Indexing Service。
4. 安裝Actif KM Builder時將自動安裝WebGenie，WebGenie不需再安裝。
建議安裝WebFlow。

軟體需求
1. 伺服器端：
Microsoft Windows 2000 Server中文版，檔案格式必須為NTFS
Microsoft Exchange 2000 Server (使用郵件、記事、日誌、工作、行事曆、連絡人等訊息功能時)
Microsoft SQL Server 7.0/2000 (使用WebFlow或使用佈告欄等功能時)

2. 用戶端：
Windows 95/98/2000
瀏覽器（IE 5.0或以上版本）

3. 網路：
Internet網際網路、或intranet內部網路

　

常見問題與解答

Q: 安裝本軟體的最小軟體需求?
A: 必須安裝於Windows 2000 Server中文版，檔案格式必須為NTFS。安裝時選擇Access做為資料庫。
Exchange 2000 Server、或Microsoft SQL 7.0/2000資料庫可以於以後視需要再安裝。
安裝本軟體的電腦記憶體最好128MB(含)以上。

Q: 本軟體的功能與軟體需求的關係?
A:
1. 僅安裝於Windows 2000 Server中文版(NTFS檔案格式)的電腦：
可使用個人化與統一管理的數位儀表板、群組中心、企業園地、資訊中心、搜尋中心等功能。

2. 僅安裝於Windows 2000 Server中文版(NTFS檔案格式)與WebFlow的電腦：
可使用個人化與統一管理的數位儀表板、群組中心、企業園地、資訊中心、搜尋中心、流程中心等功能。
WebFlow可安裝於同一台電腦或網路上其他電腦上。

3. 電腦或網路上其他電腦安裝Exchange 2000 Server與WebFlow：
可使用個人化與統一管理的數位儀表板、群組中心、企業園地、資訊中心、搜尋中心、流程中心、個人訊息區、資源中心等功能。
若您的電腦或網路上其他電腦已經安裝了Exchange 2000 Server時，則可以多使用：
* 個人訊息、與資源中心的功能。
* 群組中心、資訊中心可以多使用郵件、記事、日誌、工作、行事曆、連絡人等訊息功能。

Q: 是否必須於安裝電腦或網路上其他電腦已經安裝了Exchange 2000 Server時才可以安裝本軟體?
A: 可以後再安裝Exchange 2000 Server。
未安裝Exchange 2000 Server前，您仍可使用個人化與統一管理的數位儀表板、群組中心、企業園地、資訊中心、搜尋中心、流程中心等功能。
若您的電腦或網路上其他電腦已經安裝了Exchange 2000 Server時，則可以多使用：
1. 個人訊息、與資源中心的功能。
2. 群組中心、資訊中心可以多使用郵件、記事、日誌、工作、行事曆、連絡人等訊息功能。

Q: 是否一定要安裝AD(Active Directory)?
A: 不需要。
您可選擇使用Windows 2000 Server的Active Directory或Windows NT 4.0 Server的PDC來管理使用者帳號及群組。 

Q: 非Administrator登入時出現「HTTP 錯誤 401 401.1 未授權：登入失敗」的錯誤訊息：
A: 若本軟體安裝在Domain Controller時，未將 [Domain Users] 賦予可以登入本機權利時，非Administrator將無法登入IIS。 
將 [Domain Users] 賦予可以登入本機權利的方法，為於 [開始/程式集/系統管理工具] 執行 [Active Directory 使用者及電腦]，於 [Domain Controllers] 按右鍵選 [內容]，於 [群組原則] 頁按下 [編輯] 以開啟 [群組原則]，於 Active Directory之Domain Controllers Policy，選擇 [電腦設定/Windows設定/安全性設定/本機原則/使用者權派]，於右邊畫面 [本機登入] 快速連按兩下滑鼠左鍵，將 [Domain Users] 加入 [本機登入] 的權利中，過約數分鐘後即可生效，或於 [開始/執行] 執行 secedit.exe /refreshpolicy machine_policy 即可完成設定。

Q: 安裝時在"Exchange Server設定"的地方裡,"URL"應該填什麼?
A: 填Exchange Server Name或IP，有無http://均可，建議使用者安裝時填 Server Name。

Q: 安裝時在IIS網址設定的地方要填什麼?
A: 本機IIS的IP、Server Name、或localhost，有無http://均可，建議使用者安裝時填 Server Name。

Q: WebFlow、Exchange 2000 Server、SQL Server 7.0/2000是否要與本軟體安裝在同一台電腦上?
A: 不需要。
WebFlow、Exchange 2000 Server、SQL Server 7.0/2000可安裝於同一台電腦或網路上其他電腦上。

Q: Exchange 2000 Server安裝於本電腦或網路上其他電腦時，安裝本軟體後要做甚麼事?
A: 若Exchange 2000 Server安裝於網路上其他電腦時，安裝後請先複製 \KMBuilder\KMFolders2\所有檔案 到Exchange 2000 Server /public/KMFolders2/ 
於瀏覽器執行http://ExchangeServerIP/exchange： 
於 /public/KMFolders2/資源管理中心/會議室/ 選 "約會項目" 新增 "會議室101" "會議室201" 資料夾。
於 /public/KMFolders2/資源管理中心/投影機/ 選 "約會項目" 新增 "投影機1" "投影機2" 資料夾。
請 於 [Internet服務管理員] 確認 Exchange 2000 Server 公用資料夾之 /public/虛擬目錄目錄安全設定為基本驗證、整合的Windows驗證。

Q: Actif KM Builder數位儀表板的功能與Microsoft Digital Dash Board 2.01的差異點?
A: 
比較如下：
　 Actif KM Builder 2001 Digital Dash Board 2.01
數位儀表板製作 Yes (加強製作功能) Yes
完全支援中文 Yes  No (部份中文亂碼)
「個人化」 Yes (個人數位儀表板、我的最愛)  No
「統一管理」 Yes (公用數位儀表板、推薦)  No
「規則設定管理」 Yes (數位儀表板及文件的使用權限)  No
KM Portal所需「內容」 Yes (40個常用KM Portal模組)  No (自行製作)
支援多語版 Yes (中繁、中簡、英文等多語版) No

說明：
Microsoft Digital Dash Board提供數位儀表板「製作」的功能，但未提供「個人化」、「統一管理」、及知識管理入口網站之所需要的「內容」(需自行製作Web Part)功能，造成對資訊管理者而言，要幫全公司每一個人都建立一個數位儀表板(1000人的公司要製作1000個數位儀表板)，並且無法管理，造成企業未能成功的導入「數位儀表板」。
Actif KM Builder提供「個人數位儀表板」、「我的最愛」讓數位儀表板可以個人化，「公用數位儀表板」、「推薦」讓數位儀表板又可以統一來管理。
Actif KM Builder整合Microsoft Digital Dash Board 2.01的功能，提供個人化與統一管理的數位儀表板、40個隨拖拉即用的常用知識功能模組、及規則設定管理(Rule-based Management)，讓您迅速於內部建立個人訊息區、群組中心、企業園地、資訊中心、資源中心、流程中心、搜尋中心等知識管理的入口網站。


Q: Actif KM Builder與X-OA差異點?
A: 
比較如下：

　 Actif KM Builder 2001 X-OA 1.0
「個人化」數位儀表板 Yes (個人數位儀表板、我的最愛) No
「統一管理」數位儀表板  Yes (公用數位儀表板、推薦)  No
「規則設定管理」 Yes (數位儀表板及文件的使用權限)  No
KM Portal所需「內容」 Yes (40個常用KM Portal模組) No
群組協同作業 Yes (群組工作區)  No
資訊中心  Yes (All-In-One樹狀知識匣)  No
資源中心 Yes (資源行程查詢)  No
支援多語版 Yes (中繁、中簡、英文等多語版) No
動態分類搜尋 Yes (WebGenie動態分類) No
企業園地
(企業園地：佈告欄、資訊區、討論區、問題解答區、意見箱、卷調查、訂閱訊息、活動報名) 		Yes No

說明：
Actif KM Builder與X-OA是不同層級的產品，Actif KM Builder所提供的功能，X-OA幾乎都為未提供。
針對X-OA所提供的功能分析如下：
1. 企業入口網站模組(數位儀表板)部份
功能上有以下限制：
(1) 不易導入數位儀表板：X-OA僅將Microsoft Digital Dash Board 2.01英文訊息翻譯成中文(只要翻譯XML檔案即可)，未提供「個人化」、「統一管理」功能，造成企業不易成功的導入「數位儀表板」。
Actif KM Builder提供「個人數位儀表板」、「我的最愛」讓數位儀表板可以個人化，「公用數位儀表板」、「推薦」讓數位儀表板又可以統一來管理，40個常用KM Portal模組讓MIS迅速於內部建立知識管理入口網站。
(2) 安全性：X-OA其所提供的個人、部門、公司入口網站等元件，須考慮是否夠安全，會不會讓不應該看的人看到不應該看的資訊呢？
Actif KM Builder與Windows 2000「安全權限」整合，即所有All-In-One樹狀知識匣、群組工作區、公用數位儀表板、推薦的權限控制與Windows 2000安全權限完全整合。

2. 電子表單中心模組部份
X-OA架構於Exchange Workflow Designer，未使用資料庫，架構與功能上有以下限制：
(1) 速度太慢：填寫與處理表單速度太慢 (約數十秒以上)。
(2) 無法累積大量：申請累積到數百個表單後，系統執行速度十分慢 (如查詢進度約須數分鍾以上)。
(3) 表單格式變動後無法管理：使用Excel範本、Word範本做為表單，申請者在填寫表單之前，每次都需先從伺服器下載最新的表單範本來填寫內容，表單格式變動後，若未下載最新的表單範本，將造成有的人仍使用舊表單。
(4) 無法規劃流程圖：申請與處理者自由決定流程，無法依照流程規劃圖行走。
(5) 不易與現有資料庫整合。
Actif KM Builder搭配WebFlow提供表單製作容易、流程規劃簡單、與現有資料庫快速結合之特點。

3. 文件管理中心模組部份
僅提供簡易文件摘要與全文檢索功能，未提供文件管理所需之文件版本控制、Check-in/out、發行、訂閱等功能。
Actif KM Builder未提供文件管理的功能，但提供更多的文件摘要(如自定摘要表單)、與全文檢索(如動態分類搜尋)、文件規則設定管理功能。
Actif DM Builder將提供完整的文件管理功能。

4. 網路學習中心模組部份
純粹為Access資料庫應用，與Exchange Server無關。


Q: Actif KM Builder是否提供文件管理功能?
A: 
Actif KM Builder未提供文件管理的功能，但提供更多的文件摘要(如自定摘要表單)、與全文檢索(如動態分類搜尋)、文件規則設定管理功能。
Actif DM Builder將提供完整的文件管理功能，包括文件版本控制、Check-in/out、發行、訂閱、查閱記錄、統計報表等功能。

Q: Actif KM Builder與Exchange 2000 Server整合的部份?
A: 
Actif KM Builder善用Microsoft Exchange 2000 Server最新的Web Storage等技術與功能。
Web Storage主要是將Exchange 2000 Server的訊息功能(郵件、記事、日誌、工作、行事曆、連絡人)轉為Web-based，可於瀏覽器存取6個訊息功能。
Actif KM Builder善用Web Storage 6個Web-based訊息功能，若您的電腦或網路上其他電腦已經安裝了Exchange 2000 Server時，則可以多使用：
* 個人訊息、與資源中心的功能。
* 群組中心、資訊中心可以多使用郵件、記事、日誌、工作、行事曆、連絡人等訊息功能。
* 一般全文檢索與動態分類搜尋功能，可以搜尋到Exchange 2000 Server公用資料夾的資料。
