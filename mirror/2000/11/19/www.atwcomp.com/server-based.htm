TERMINAL

SERVERS

TERMINAL SERVERS are the latest technology for the multi-user environment.  In a traditional computer network, a file server sends programs and data to powerful client computers for processing.  Because the client computers actually do the processing, they have to be very capable.  Also, a high bandwidth connection is required because a lot of data has to flow between the server and client workstations. 

Using the new TERMINAL SERVER (SERVER-BASED COMPUTING) technology, the server actually executes the programs and the only things transmitted between the server and the client are the keystrokes, mouse movements and screens.  The client computers can now be "thin clients" -- relatively low-powered, inexpensive computers or the new "Windows Terminals".

This is all possible using the new Microsoft Windows 2000 Server and Citrix Metaframe.

                     +  

Combining the low cost of a thin clients with the benefits of a Windows-based environment, offers the same low-cost, centrally-managed environment of the traditional mainframe computer using text terminals, but adds the familiarity, ease of use, and breadth of applications support offered by the Windows operating system platform.

Additionally, you can easily provide computer access to your branch offices and remote users as Terminal Servers do not require high bandwidth connections -- even regular phone lines work fine.

There are some very good reasons to consider Terminal Servers:

RAPID APPLICATION DEPLOYMENT - Applications can be installed once on the server and are accessible to any user.  Nothing has to be done on the client side computer.  This saves countless hours of administrative and technical services time.

HIGH PERFORMANCE REMOTE ACCESS - Because Server-based, thin client computing requires minimal bandwidth, remote users can now access the computer network and use applications at nearly the same speed as local users.  Even dial-up connections using 28.8 modem work fine.  This is the only remote access solution that works at an acceptable level of speed.  Now branch offices, the outside sales team and home users can all operate on the same network.

LOWER TOTAL COST OF OWNERSHIP (TCO) - Consider that legacy equipment such as older Pentium and 486 computers can still serve as thin clients.  Also, consider that these older computers, when used as thin clients, pose no Y2K problems.  You can slow the obsolescence cycle and save money.  Additionally, the only configuration required on the local computer is the one-time installation of a small terminal client program.  Applications are installed only once, on the server.  Any changes are performed on the server and network administration can be done from a remote location as easily as in front of the server console.  Combined, these factors make Server-Based/Thin Client Computing the most efficient solution and results in a lower cost of ownership and greater return on your computing dollar.
