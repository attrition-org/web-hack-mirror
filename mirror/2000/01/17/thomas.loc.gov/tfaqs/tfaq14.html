Q14. Why is the THOMAS server not working properly, and why am I getting error messages?

If the problem is ...

  • Slow response

    Action(s):

    * If this occurred between 10-11 a.m. or 4-5 p.m. e.d.t. during a week day, this is our busiest usage time. Try THOMAS later.

    * If your browser will allow it, turn off your graphics function. Try THOMAS again.

    * Clear your browser by turning it off and then on. Try THOMAS again.

    * If your system has a "ping" command, you might try seeing what your path is to http://thomas.loc.gov or if and where it hangs up.

  • Received "Fatal Error 500"

    Action::

    * This is a universal transient error on the internet. Try THOMAS again.

  • Received "Server Error" message attempting a GPO PDF copy

    Action:

    * This pdf function extracts documents from the GPO site. An error message here indicates the bill which you are attempting to copy has not been completed for pdf copying by GPO. THOMAS has no control over this availability. Try later.

  • Can't connect to the House link

    Action:

    * Try other non-THOMAS web sites. At the bottom of the THOMAS homepage go to the U.S. Legislative Branch web site. Or try the U.S. House of Representatives homepage.

  • Unable to find retrieved information in display of the Congressional Record or a committee report

    Action:

    * When a retrieved document is so large that it cannot be entirely displayed on a single web page, the Forward option is highlighted (located near the bottom as well as near the top of this display screen). This indicates to the user that the document exceeds more than the current web page and that after scrolling to the bottom of the current portion of the retrieved document, the next portion can be viewed by clicking the Forward button.
