DoS Attack Methods

Since network communications consist of several layers with the greatest concentration of traffic toward the bottom in the telecommunications layer, interruption of the telecom layer can bring about a denial of service. Among the most frequent attacks used by attackers are: *Host system hogging; *Mail bombs; *Rogue applets; *SYN flood; *Ping of death; *Exploitation of size between a DNS query and a DNS response; and *Accidental DoS.

Host system hogging involves running a program on the attacked system that ties up the CPU on the system. This generally results in the operating system crashing, which takes down the entire system.

Mail bombs consist of sending very large volumes of mail to a mail server. Since many mail servers can't handle the volume of mail sent, legal users will be denied service and in some cases mail may even be lost.

Rogue applets are used to attack users rather than servers. Hostile applets embedded in Web pages, when downloaded and run, can put the user's system into an infinite loop that requires a restart to end the looping. Browsing Web pages with automatic running of applets turned off is a preventive measure often overlooked by users. Security levels of pages can be changed to run applets when a source is trusted.

SYN flood is specifically geared to use up all new network connections at a site and prevent legal users from being able to connect. SYN (Synchronize Sequence Number) is the first of the requests to connect sent to a system. SYN contains an ID to which the receiver is supposed to respond. When a fake ID is in the packet, the receiving system never gets a connection acknowledgement. Eventually, the connection will time out and the incoming channel on the receiver will become available again and normally handle another request. During a SYN Flood, many requests are sent containing false IDs and the incoming connections are continuously tied up waiting for acknowledgements.

Ping of Death is related to the Packet Internet Groper (Ping) that tests a TCP network by sending an echo request, expecting a reply. The Ping of Death DoS attack uses a test packet larger than allowed. This can result in problems with network programs running on the targeted computer, or a crash.

Exploitation of size between a DNS Query and a DNS response is one of the newer forms of DoS and may allow remote DoS attacks against target hosts whose IP address is spoofed in the DNS query. During this attack, small DNS queries are sent from the attacker to each of the servers. The queries contain the spoofed IP address of the target. The DNS servers respond to the small queries with large responses. The responses are routed to the target, causing link congestion and possible denial of Internet connectivity.

Accidental DoS may happen due to the complexity of the Internet and the lack of a single, central point of control for everything. These types of denial of service are infrequent but when they occur they can cause inaccessibility to servers and systems.

For deliberate DoS attacks, hackers install Distributed Denial of Service (DDoS) tools on various computers. Often, computer owners are unaware of the presence of this software. By installing the DDoS software on several computers, a hacker creates a network that allows him or her to launch a coordinated attack against a server.

Generally speaking, there's little that can be done. Once a server is attacked, the administrator must take appropriate action in order to restore service. But the administrator is hampered, heor she doesn't know the location the attack is coming from, and if several servers are involved, the originating server may cease attacking and another server may take over the attack

Sniffer and filter software can help protect a server. Special scanners such as CyberCop Scanner by Network Associates, Inc. (NAI) can help detect whether a computer has DDoS tools installed on it. The FBI provides a tool on its Web site (www.fbi.gov) that can be downloaded to help determine if DDoS tools are present on a server.

It seems unlikely that DoS attacks will ever be eradicated. Because of the vastness of the Internet and the ever-increasing number of servers and users, it's doubtful any single group can exercise enough vigilance to prevent such attacks. The best we can do is to be aware by reviewing current information (www.cert.org), impose security practices on our own servers and search for improper DDoS software when possible.
