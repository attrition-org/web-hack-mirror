Web & File Server specifications

    This section contains the product specifications for the AppleShare IP Web & File Server for both file sharing and Web service.

      File sharing specifications
    Web service specifications

 
File sharing specifications
 

    • Maximum number of connected users (depending on your license agreement): 10, 50, or 500 over TCP/IP; 10, 50, 250 over AppleTalk
    Note: If you expect your network usage to exceed 250 connected users, Apple recommends a minimum of 128 MB of RAM and a Macintosh Server G3 or Power Macintosh G3 as the server.
    • Maximum number of unique open files: approximately 364
    • Maximum number of all open files: 3000 forks (data and resource)
    • Maximum number of physical volumes: 50
    • Maximum number of share points: 100
    • Maximum number of users and groups in the Users & Groups Data File: 10,000
    Note: If you expect to have more than 5000 users, Apple recommends a Macintosh Server G3 or Power Macintosh G3 as the server.
    • Maximum number of group memberships per user: 42
    • Maximum number of simultaneous launches: up to 250 (depending on the program and number of connected users)
    • Maximum volume size: 2 terabytes (TB)
    • Maximum number of multihomed ports (AppleTalk only): 4
    • AppleShare file sharing TCP port number: 548
    • FTP port numbers: 20 and 21
    • Windows file sharing TCP port number: 139
    • Windows network neighborhood UDP port number: 138
    • Windows name service UDP port number: 137

 
Web service specifications
 

    • Maximum number of connected users: 150
    Note: A computer connected through the AppleShare IP Remote Admin program is included as one connection.
    • Supported standards: HTTP 1.1
    • Idle connection timeout: 60 seconds
    • CGI timeout: 60 seconds
    • Keep alive: Supported
    • WebSTAR API: 1.2 Support
    • Web service port number: 80
    • Remote Admin port number: 311

    You can see the default MIME (Multipurpose Internet Mail Extensions) type mappings supported by the Web & File Server by choosing "Web MIME Types" from the Web & File Admin program's Server menu.
