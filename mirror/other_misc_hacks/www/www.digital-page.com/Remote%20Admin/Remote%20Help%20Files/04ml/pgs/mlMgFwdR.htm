Forwarding and relaying mail

    You can forward mail to a different user or group. If you've installed the Mail Server on multiple computers, you may also want to set up mail forwarding so that users can send e-mail to anyone on your network without entering the Mail Server's domain name.

    If you're connected to an external network, such as the Internet, you may also need to set up your Mail Server to relay mail outside your domain. You can set up mail relay on a global or host-specific basis.

    For more information, see How mail service works.

    This section tells you how to
      Forward mail addressed to local users
    Set up mail forwarding for mail addressed to local users
    Forward mail addressed to unknown users
    Set up mail forwarding to users who are not in your local Users & Groups List
    Relay all outgoing mail
    Relay outgoing mail via a specific server
    Relay outgoing mail addressed to a specific host
    Send mail addressed to a specific host via a specific server

 
Forwarding mail addressed to local users
 

    Mail forwarding can redirect mail sent to a user or group to another user or group.

    To set up mail forwarding for local users, follow these steps:
    1 Log on to AppleShare IP Remote Admin.
    2

    Click the Users button.


       
    3 Click the user whose mail settings you want to edit.
    4 Click Mail Settings.
    5 To set up mail forwarding, select the "Forward the user's mail to" button.
    To turn off mail forwarding, either set up mail for this user (by clicking the "User can receive mail via" button) or turn off mail for this user by clicking "User has no mail." Then skip to step 11.
    6 Choose the type of address you're fowarding mail to by clicking Internet or AppleTalk Address.
    If you chose AppleTalk, go to step 7. If you chose Internet, skip to step 10.
    7 In the User Name text box, type the name of the user or group that will receive the forwarded mail.
    The name must be in the Users & Groups List on the computer to which the mail is to be forwarded.
    8 Enter the zone of the Mail Server you're forwarding to.
    9 Type the name of the Mail Server you're forwarding mail to.
    10 In the Internet Address text box, type the e-mail address of the user who will receive the mail.
    11 Click the Save button.
    The Mail Server will forward the mail over TCP/IP or AppleTalk to the appropriate user or Mail Server.

 
Forwarding mail addressed to unknown users
 

    You can forward incoming mail addressed to a user who does not exist in your local Users & Groups List.

    When this feature is set up, mail that is misaddressed can be forwarded to someone who is likely to be able to resolve the problem. If you don't turn on this feature, misaddressed mail is returned to the sender.

    To forward mail addressed to unknown users, follow these steps:
    1 Log on to AppleShare IP Remote Admin.
    2 If the Mail Server is not running, click the Start Mail Server button.
    3

    Click the Mail server button.


       
    4 Click the Server Settings button.
    5 Click Mail In.
    6 Select the "Forward mail addressed to unknown local users" checkbox.
    7 Type the name of the user or group who will receive the incoming mail.
    Type the name exactly as it appears in the Users or Groups list.
    8 Click the Save button.

 
Relaying all outgoing mail
 

    If you have a connection to an external network, such as the Internet, you may need to set up your Mail Server to send (or "relay") outgoing mail to a mail server that can communicate with the external network. You may also want to relay mail through a specific server to improve performance or get around network configuration limitations. See How mail service works.

    To relay mail from one mail server to another, follow these steps:
    1 Log on to AppleShare IP Remote Admin.
    2 If the Mail Server is not running, click the Start Mail Server button.
    3

    Click the Mail server button.


       
    4 Click the Server Settings button.
    5 Click Mail Out.
    6 Click the "Allow outgoing mail" button.
    7 Select the "Relay all SMTP mail via" checkbox.
    SMTP (Simple Mail Transfer Protocol) is the industry-standard protocol for sending mail.
    When this box is checked, all mail sent by users who connect to this server for mail service is relayed through the SMTP server address you enter.
    To turn off this feature, deselect the "Relay all SMTP mail via" checkbox. All mail is delivered to the SMTP mail server specified in the message's "to," "cc," or "bcc" recipient field.
    8 Type the domain name or IP address of the SMTP mail server on your network through which you want to relay outgoing mail.
    See The Mail Server and DNS.
    9 Click the Save button.
    Your changes take effect immediately. Messages that are not addressed to the AppleShare IP Mail Server will be relayed to the specified SMTP mail server over a TCP/IP connection.

 
Relaying outgoing mail addressed to a specific host
 

    You can choose to relay outgoing mail addressed to a specific host to another mail server. A "host" is any mail server to which your users have sent mail, or from which they have received mail. See How mail service works.

    To forward mail addressed to a specific host to another mail server, follow these steps:
    1 Log on to AppleShare IP Remote Admin.
    2 If the Mail Server is not running, click the Start Mail Server button.
    3

    Click the Mail server button.


       
    4 Click the name of the host whose mail you want to relay.
    The Host List includes each host to which users of your Mail Server have sent mail, or from which your users have received mail.
    5 In the Host window, click "Edit Host settings."
    6 Click Mail To.
    7 Make sure the "Allow mail to host" button is selected.
    8 Select the "Relay all SMTP mail via" checkbox. Then type the DNS name of the SMTP server on your network through which you want to relay outgoing mail.
    To turn off this feature for the specific host, deselect the "Relay all SMTP mail via" checkbox. All mail sent by users who connect to this host for mail service is stored on the current server until it's delivered.
    9 Click the Save button.
