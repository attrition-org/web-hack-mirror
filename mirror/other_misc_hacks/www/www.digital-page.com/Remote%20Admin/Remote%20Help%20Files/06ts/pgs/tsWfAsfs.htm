Web & File troubleshooting–AppleShare file sharing administration

    The problems you might have while administering the AppleShare IP Web & File Server usually fall into one of two categories: general problems, or problems with access privileges, security, and passwords.

    General
      You're having trouble creating drop boxes
    Too many people have access to the Web & File Server

The access privileges you assigned to a single folder are lost.
 

    If you backed up a folder and then retrieved the backup copy, the access privileges for the backup copy may disappear. Some backup programs don't save access privileges; see your backup program's documentation for more information.

    Make sure that the owner of the folder (or another administrator) did not change its access privileges.

    To restore access privileges for the backup copy, you need to reassign them. For instructions, see Setting access privileges.

 
You or other users see a message that a duplicate serial number has been found on the network.
 

    • When the Web & File Server starts up, it checks the network for other computers using the same serial number. If the same serial number is detected, an unauthorized copy of AppleShare IP software may be running in violation of the Apple software licensing agreement.
    • Routers on your network may be caching serial numbers in the router table. Ask your network administrator to prevent the routers from caching serial numbers.
    • Determine which computer is running software in violation of the agreement and cease using that copy of the software. For guidance, see "Problems with your Serial Number" in AppleShare IP Help.
    • If these solutions don't help, you may have a damaged Users & Groups Data File. Go to the server and replace the file with your backup copy. If you don't have a backup copy of your Users & Groups Data File, use the AppleShare IP First Aid program in the AppleShare IP First Aid folder on your hard disk to repair the damaged file. See the Read Me in the AppleShare IP First Aid folder for instructions. If the First Aid program doesn't repair the file, you'll need to reenter your users and groups. For instructions, see Creating users.

     

 
Users have access to your personal files.
 

    • Keep personal files in a separate folder that is not shared.
    • On the Disks & Share Points page, open the Privileges page for the folder and check the folder's access privileges. Make sure you are the owner of the folder.
    • Remove any access privileges assigned to the User/Group and Everyone categories. Finally, make certain that no one but you has the "Allow user to administer server" box checked on the User page for this share point. If other users have this privilege, they will be able to see all your shared files.
    For detailed instructions, see Creating share points and Setting access privileges.

     

 
Too many people have access to the Web & File Server.
 

    • Turn off guest access. For instructions, see Allowing and denying guest access.
    • Limit guest access to folders only, not entire hard disks, and make sure that access privileges for Everyone are enabled for only those folders that require guest access. Decrease the maximum number of guests allowed. (See Lowering concurrent connections.)

 
You're having trouble creating drop boxes.
 

    Drop boxes must be created inside a share point. Assign the Read access privilege to the share point and assign Write Only access to folders inside the share point that are to be used as drop boxes.

    For instructions, see Creating share points and Setting access privileges.

 
You can't share an item.
 

    You may be trying to share an item that is contained by a share point. To set explicit access privileges for the item, see Setting explicit access privileges.
