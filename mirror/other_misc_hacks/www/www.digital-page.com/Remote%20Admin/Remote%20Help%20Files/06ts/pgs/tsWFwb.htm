Web & File troubleshooting–Web services

    This section helps you address problems providing Web services.

    This section tells you what to do if

      Users say that they cannot connect to your Web site
    A Web plug-in is not working as expected
    An alias cannot find the original item
    You see "low plug-in memory" warnings
    You're not able to upload files to the Web Server

 
Users say they cannot connect to your Web site.
 

    • Make sure that Web service is turned on in the Web & File Server Settings page.
    • Verify that the correct folder is selected as the default Web folder on the Web Server Settings page. Make sure that the correct HTML file is selected as the home page.
    • For the default Web folder, make sure that the Read Only access privilege is set for the Everyone category.
    • Verify that users' computers are configured correctly for TCP. Check the users' TCP/IP control panel. If there don't appear to be problems with the TCP/IP settings, use a "pinging" utility that allows you to check network connections.
    • If the users are not registered, make sure guest access is enabled.
    • Verify that the problem is not a DNS problem by specifying the IP address of the Web Server instead of its DNS name.
    • Make sure users are entering the correct URL to connect to the Web Server.
    • If you have set up multiple Web sites, make sure you have configured the Multi Domain Config file correctly and performed any other necessary steps. Go to the server and see AppleShare IP Help for more information.
    • Make sure your DNS server's entries for the Web & File Server's IP address and domain name are correct.
    • Make sure your users have access privileges to your Web site, and/or that guest access is enabled. See Accessing your Web site.
    • If users are seeing a Java error and using Internet Explorer, tell them to make sure that Java Virtual Machine is selected in the Java section of their browser's Preferences window.

 
A Web plug-in is not working as expected.
 

    • Check the documentation that came with the Web plug-in to make sure it's installed correctly and compatible with AppleShare IP 6.1. You can also look there for troubleshooting information.
    • Increase the memory allocated to plug-ins on the Web Server Settings page. If plug-in memory is set too high, you'll see a warning message when you restart, but this will not prevent the plug-ins from working. See Setting aside memory for plug-ins.

 
An alias cannot find the original item.
 

    Make sure the original item is located in a share point or in the appropriate Web folder. If it's not, share the item that contains the original item and alias, or move the original and the alias to a shared area. See Setting access privileges.

    Additional troubleshooting help on this topic is available on the server, in AppleShare IP Help.

 
You see "low plug-in memory" warnings.
 

    Increase the memory allocated to plug-ins on the Web Server Settings page. If plug-in memory is set too high, you'll see a warning message when you restart, but this will not prevent the plug-ins from working. See Setting aside memory for plug-ins.

 
You're not able to upload files to the Web Server.
 

    • Make sure the Enable Plug-Ins checkbox is selected on the Web Server Settings page.
    • Make sure the path to the Plug-Ins folder (the URL) is correct. For instructions, see Uploading files to the Web server.
    • Make sure the person trying to upload files has Read/ Write access privileges for the Web folder that contains the plug-ins.
    • Make sure the plug-ins file is not in use by another user.
