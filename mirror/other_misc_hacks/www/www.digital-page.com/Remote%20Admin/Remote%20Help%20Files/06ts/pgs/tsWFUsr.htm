Web & File troubleshooting–AppleShare file sharing–Mac OS clients

    This section tells you what to do if your Mac OS file sharing clients are having problems with access privileges, passwords, connections, and so on.

    Note: If your Windows clients are having file sharing problems, see Web & File troubleshooting–Windows file sharing.

 
A user has forgotten his or her password.
 

    Give the user a new password. See Creating a new user.

 
A user cannot find a file or folder.
 

    A user must have Read access privileges to the volume and to each folder in the path to the file.
    • At the user's computer (logged on to the Web & File Server as an administrator), locate the item using the Find command (in the File menu). Then choose the Sharing command from the File menu to examine the access privileges.
    For information about how AppleShare IP access privileges look on a user's computer, see What users see.
    • If necessary, change the access privileges to the share point. See Setting access privileges.

 
The AppleShare icon does not appear in the Chooser on a user's computer.
 

    Make sure that the AppleShare extension is turned on in the Extensions Manager on the client's computer. If it's not turned on, click the checkbox to the left of the extension under the On/Off column. Then restart the computer.

    If you can't find the AppleShare extension in the Extensions folder, you need to reinstall the AppleShare Client software on the user's computer. For instructions, see the "Getting Started" manual or "Upgrading" document that came with your software.

 
Users can't find the Web & File Server in the Chooser.
  

    • Make sure that AppleTalk is active on the computer that is running the Web & File Server.
    • On the user's computer, make sure that AppleTalk is active and that the correct network connection type is specified in the "Connect via" pop-up menu in the AppleTalk control panel.
    • Make sure that the Web & File Server is running.
    • Make sure users are looking for the correct name and AppleTalk zone (if any). You can find the name you assigned to the Web & File Server under Server Settings.
    • Verify the network connections between the server and the users' computers.

 
Users are disconnected unexpectedly.
 

    • Verify that the users were not disconnected intentionally by an administrator.
    • Make sure that the volume to which users were connected is mounted.
    • Verify the network connection of each user's computer.
    • Make sure that the Disconnect Idle Users checkbox is not checked on the File Server Settings page.

    Additional troubleshooting help on this topic is available in AppleShare IP Help on the server.

 
Users can't log on to the Web & File Server.
 

    • Make sure the user has entered the correct name and password. The user name is not case-sensitive, but the password is. See Creating a new user.
    • Make sure "User may log on" is selected on the User page for that user.
    • Make sure you have shared a disk or folder and you have assigned appropriate access privileges.
    • Make sure the maximum number of concurrent file server and FTP client connections has not been exceeded.
    • Check to see if users can connect to the server via AppleTalk (the Chooser). Then check to see if they can connect via TCP/IP (for example, by opening a Web browser program and logging on to the AppleShare IP Web site). If they can connect over one network but not the other, there may be a problem with their AppleTalk or TCP/IP settings.

 
A user's password doesn't work.
 

    • Check whether the password has expired. If it has, assign a new one.
    • Change the "Failed attempts before log on disabled" attribute to a higher number on the Web & File Server Settings page.
    • These are the messages users see when their passwords have expired:
      User's computer   Message
    Mac OS with version 3.6.1 or later of the AppleShare Client software

    "Your password has expired. Please set a password."

    or

    "Your password must be changed. Please set a new password."
    Mac OS with AppleShare 2.0 Client software "The attempted connection to the server's volume has failed. Please contact the server's administrator."
    Windows "Error -5019 has occurred."
       
    • If your network has several computer types or AppleShare versions, you can sometimes avoid password problems by turning off the following two attributes: "Require new password on next log on" (on each user's User page) and "Days until password expires" (on the File Server Settings page).
    • If none of the above solve the problem, you may need to delete and recreate the users who can't log on.

 
Users receive a -1072 error message.
 

    The users were disconnected from the server. Tell users to click the OK button and disregard the message. If any files are open on the client computer, tell the user to save the files to his or her hard disk.

 
Users receive error messages when logging on.
 

    Make sure the "User may log on" checkbox is selected on the user's General Information page.

 
The access privileges a user sets keep changing.
 

    • The user may be setting an invalid combination of access privileges. See What users see.
    • There may be a problem with the AppleShare PDS file. Use the AppleShare IP First Aid program to repair the file. (The AppleShare IP First Aid program is in the AppleShare IP First Aid folder inside your AppleShare IP 6.1 folder on the server's hard disk.)

 
A user can't log on to the Web & File Server via TCP.
 

    • Make sure the user's TCP/IP settings are correct for your network.
    • On the Web & File Server, make sure "Enable File Server Clients to Connect over TCP" is selected on the File Server Settings page. See Enabling or disabling AppleShare over TCP/IP.
    • Make sure the user's computer has version 3.8.1 or later of the AppleShare Chooser extension.
    • Use a TCP/IP utility that tests network connections by "pinging" devices on the network.

 
Users can't access a CD-ROM disc.
 

    Make sure you've made the CD-ROM disc a share point and set access privileges for it.

    See Creating share points and Setting access privileges.

 
Users who can administer AppleShare IP servers have forgotten their passwords.
 

    • If there is more than one administrator for the computer, one who has not forgotten the password can reset the passwords of other administrators. See Creating a new user.
    • Go to the server and use the Extensions Manager control panel to turn off the AppleShare IP Registry extension. Restart your computer. When your computer restarts, open the Sharing Setup control panel and type a new password in the Password text box. Then use the Extensions Manager control panel to turn the AppleShare IP Registry extension back on. Restart your computer. When you open an AppleShare IP administration program, enter the owner's name in the Name box and the owner's new password in the Password box.

 
Users' accounts become disabled for no known reason.
 

    The limit on failed log-on attempts may have been exceeded for one of the following reasons:
    • Someone may be trying to guess users' passwords.
    • Users' computers may be set up to log on automatically with an incorrect user name.
    • Users may be choosing Recent Servers from the Apple menu and using the wrong name to log on. By default, the most recently used name is placed in the Name text box in the log-on dialog box.

    Reset the users' accounts and/or turn off or change the limit on failed log-on attempts.

 
Some users see a volume size that is different from the volume's size displayed on the Web & File Server.
 

    Users who have versions of system software earlier than System 7.5 cannot see large volumes. Update the client computers to Mac OS 8.5 and AppleShare Client 3.8.1. For instructions, see the "Getting Started" manual or "Upgrading" document that came with your software.
